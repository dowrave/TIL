# 실시간 처리
#실시간처리

1. **크기가 무한**한 데이터셋
2. **유효 시간** 내에 데이터 처리 필요 (EX : 환자의 심박수)

### 실시간 처리 방식 유형
1. 마이크로 배치 처리 (`Spark`)
- 배치처리의 단위를 작고 빠르게
	- 이 단위는 고정(`Fixed Window`).

2. 스트림 처리
- 데이터 도착 즉시 처리, 한 번에 1건 처리
- 스트림 처리의 종류
	1. Time Agnostic
	- 들어오는 모든 데이터를 처리
	2. Filtering
	- 들어오는 모든 데이터 중 처리하고자 하는 데이터를 필터링하여 처리

### 실시간 처리에 필요한 기술적 요소
1. 데이터의 유실 방지
	- 데이터의 생성 속도는 처리 속도보다 빠르다
	- 이를 위한 임시 데이터 저장 공간인 `버퍼 모듈`이 필요함. -> 카프카
2. 데이터의 일관성 보장
	- 시스템 장애 발생 상황에도!
