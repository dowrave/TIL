#FLStudio 

## Arranging
- 아이디어나 비트를 곡에 녹여내는 일
- 곡에 새로운 요소를 추가하거나 바꾸거나 등등

- `intro - verse - chorus - verse - chorus - breakdown(bridge) - another chorus or double chorus - Outro` 같은 식으로 구성한다고 함
	- 물론 구성은 본인 마음임 : `intro`를 뺀다든가.
	- `verse`와 `chorus`의 악기 구성을 다르게 해서 `chorus`를 강조할 수 있음
	- 코러스는 훅이나 메인 멜로디가 놓이는 곳
	- 여기선 **모두 (보컬 없이) 악기로 구성하고, 코러스는 리드 멜로디 파트**
	- `riser`나 `reverse`로 코러스 전의 빌드업을 구성할 수 있음
	- 코러스 진입 직전의 `crash`도 방법

... 말로 풀면 장황해지지만 사실 다 어디선가 들어봤을 거임

- 각 파트의 길이
	- `verse` : 12 ~ 16마디
	- `chorus` : 8 마디
	- `breakdown` : 원하는 만큼 - 빌드업 길이에 의존
	- `bridge` : 8마디
		- 일렉트로닉에서 `bridge`나 `breakdown`은 8마디 이상이 되기도 함

- **`Playlist`에서 마디 표시하는 부분을 클릭하고, `alt + T`를 누르면 마커를 달아놓을 수 있다.**

- 플레이리스트의 각 트랙을 구성할 때 `Auto name Clips`라는 게 있다. 해당 `Track`의 이름과 그 안에 들어간 패턴들의 이름을 바꾸는 것으로 보임

### Arpeggiator

- 아르페지오는 일렉트로닉 댄스, 혹은 일렉트로닉 장르 자체에서도 많이 쓰임

- 기존 `Lead` 패턴을 `Chorus`로 밀고 
- `Arpeggio`라는 패턴, 악기`Flex - Pure Feather`, 트랙, 믹서 등등에 이름 & 색상 설정해서 추가

- 기존 코드 진행 : Am - G - F - G

- 이번에는 **`기준음 + 5번째 위 + 한 옥타브 위의 음`을 사용**해볼 것
	- 메이저인지 마이너인지 판단할 수 없음
	- `A2 E3 A3` - `G2 D3 G3` - `F2 C3 F3` - `G2 D3 G3`
	- 예시 : `A2(1)` `B2(2)` `C3(3)` `D3(4)` `E3(5)` `F3(6)` `G3(7)` `A3(8)`

![[Pasted image 20260119230411.png]]

- 이 상태에서 노트들을 다 잡고`Ctrl + A` 
- **`Alt + A`로 `Arpeggiator`를 켤 수 있음**

![[Pasted image 20260119230501.png]]
- `Pattern` : `Default`로 지정
- `Time Mul` : 왼쪽으로 돌리면 노트를 더 잘게 쪼갬, 오른쪽으로 돌리면 덜 쪼갬
- `Gate` :  `Time Mul`로 쪼개진 각 노트의 길이를 더 수정함. `Time Mul`을 설정한 상태에서의 노트 자체의 길이 설정.

- 강의에선 아르페지에이터를 끄고 직접 노트를 찍었다.

## Countermelody 추가
- `Countermelody` : 리드 멜로디만큼 선명하진 않은 2번째 멜로디. 그 자체로도 멜로디로서 기능을 하고, 리드 멜로디를 보충해준다. 

- 멜로디를 짤 때는 피아노`FL Keys`부터 시작한다. 
	- 코드가 선명하게 들리고, 신스로 바꾸는 것도 쉽기 때문임.

- `Keys`에 있는 코드 진행을 복붙해서 가져온다.
	- `Keys` 패턴 클릭 - Piano Roll에 패턴이 뜸 - `Ctrl + A` - `Ctrl + C` - `Countermelody`로 패턴 변경 - `Ctrl + V`으로 붙여넣기
	- **조심** : 피아노 롤에서 원래 악기로 붙여넣기 될 수 있음. `FL Keys` 체크할 것.

- 정상적으로 다 가져왔다면 이런 상태 : `Lead + Keys`
![[Pasted image 20260119235743.png]]

- 가져온 코드들은 **한 옥타브 낮춰줌.**

- 노트들 분석
- 앞 4마디는 무브먼트가 많고, 뒤 4마디는 롱노트가 많음

- **카운터멜로디는 그 반대로 구성해주면 됨** 
	- 앞 4마디는 롱노트가 많게, 뒤 4마디는 무브먼트가 많게. 
- 그래서 카운터멜로디는
	- 메인 멜로디를 잘 관찰해야 함
	- 리드 악기 or 보컬 뒤에서 연주하는 악기들을 생각해보면 됨 
	- "대조적", 혹은 "정반대"의 구성도 ㅇㅋ
	- `call and response, question and answer`로 생각해도 됨.

- 대조를 주는 뚜렷한 방법은 **두 멜로디의 옥타브를 다르게** 해주는 것임
- 이외에도 바로 위 예시처럼 **무브먼트가 많다면 롱노트를, 혹은 그 반대로.**

- 이제 가져왔던 코드와 메인 멜로디는 모두 뮤트 처리.
- 악기를 바꾼다. `FLEX - Chariot Horn`
	- 설정 
	- `Master Filter`의 `CutOff`을 살짝 줄여서 고음을 죽임
	- `Delay` 증가 - 리드 멜로디와의 거리감 형성
	- 리버브도 살짝 증가.
- 옥타브도 바꾼다. 
	- 가져온 멜로디와 코드를 한 옥타브 내렸으므로 원래 멜로디와 카운터 멜로디가 충돌하는 상황임
	- `Countermelody`을 한 옥타브 내려줌. 다른 노트들을 뮤트처리했으니까 전체 노트를 잡고 내려주자.
