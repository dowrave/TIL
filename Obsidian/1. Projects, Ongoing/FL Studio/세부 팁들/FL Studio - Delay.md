- **딜레이의 역할** : 소리를 작아지게 + 반복해서 들려줌. 노래방의 에코 생각하면 비슷한가?

> 참고 : 노브를 `Alt + 클릭`하면 디폴트 설정으로 돌아감

## 기능 
- `Input`
	- `Pan` : 딜레이된 시그널의 패닝. 
		- 즉, 원본의 소리를 조절하진 않는다. 원본 소리는 항상 중앙에서 들림.
	- `Vol` : 딜레이된 시그널의 볼륨
		- 볼륨이 0이면 딜레이가 들리지 않음
		- `Max`일 경우 1번째 딜레이된 소리는 원본보다 큼 

- `FeedBack`
	- `Vol` : 딜레이된 시그널의 피드백 
		- `0`이면 딜레이가 1번만 들림 (즉, 원본까지 총 2번 들림)
		- `Max`면 계속 딜레이가 들림
	- `Cut` : 딜레이된 시그널의 EQ. 
		- 노브값이 낮아질수록 고주파부터 빨리 사라지는 경향이 있는 듯
		- `0`일 경우 딜레이 시그널이 들리지 않음
	- 모드 : 패닝에 영향을 받음
		- `Normal` : 딜레이된 시그널이 패닝 방향을 따라감
		- `Invert` : 딜레이된 시그널이 패닝 역방향으로 따라감
		- `P(ing) Pong` : 딜레이된 시그널이 패닝 방향부터 시작해서 좌우 교차로 들림

- ★`Time`
	- `Time` 
		- 왼쪽으로 돌리면 딜레이된 시그널이 빠르게 들림. 오른쪽은 느리게 들림.
		- **프로젝트의 템포값을 따라간다.** 프로젝트의 템포에 맞춰서 얼마나 오래 울릴지 결정할 수 있음.
			- `우클릭 - Set - N Steps`
			- **`2 steps`으로 설정하는 걸 추천.**
	- `OFS(Offsets)`
		- 좀 신기한 효과인데, 일종의 스테레오 효과가 난다. 
		- 값이 너무 커지면 소리가 좌/우에서 따로따로 들린다. 살짝만 줘도 자체로 스테레오 효과가 남(의도적으로 좌/우에서 딜레이 시그널을 내는 원리인 듯? 살짝 번갈아가면서 내는 느낌이 있다.)

- `Dry`
	- 이전에 리버브에서도 다뤘듯, 원본의 소리.
	- 디폴트 설정보다 키우면 원본보다 더 큰 소리가 난다. 

딜레이도 **이전의 리버브처럼 별도의 딜레이 채널을 두고 관리하는 걸 추천**한다. 이 때 해당 플러그인의 `Dry`를 0으로 놓는 것도 잊지 말자. (원본의 소리를 내보내는 다른 채널이 있다는 가정 하에)
