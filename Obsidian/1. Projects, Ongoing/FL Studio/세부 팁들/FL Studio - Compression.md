#FLStudio 

## 정의

- `컴프레션` : 오디오의 `다이내믹 레인지`를 줄이는 것
	- `다이내믹 레인지` : 한 오디오 소스에서 가장 작은 소리와 가장 큰 소리 사이의 차이
		- 넓다 : 극적인 대비가 느껴짐(교향곡), 
		- 좁다 : 전체적인 볼륨이 일정하게 느껴짐(댄스 음악, 광고 방송)

컴프레션을 걸면 소리의 대비(=다이내믹 레인지)가 줄어들고 전체적인 소리는 훨씬 안정적이고 크게 들리게 됨(큰 소리를 깎고 전체적인 볼륨이 올라감)

## FL Studio - Fruity Limiter 플러그인
- 믹서의 플러그인에 있는 `Fruity Limiter`
	- `Fruity Compressor`도 있지만 시각적으로는 `Limiter`가 더 좋음

![[Pasted image 20260101204320.png]]
> `Comp` 탭을 눌렀음

- `Thres` 
	- 기본으로 흰색 선에 설정되며 노브를 왼쪽으로 돌리면 파란 선이 나타나며 높이가 낮아짐
	- 말 그대로 컴프레서를 적용하는 기준선. 소리 크기가 파란 선에 닿으면 적용되기 시작하는 듯
- `Knee`
	- 컴프레셔가 부드럽게 작동하는 정도
	- `Hard Knee` : 빠르게 적용됨. 드럼 등에 적합. 오른쪽으로 돌릴 경우.
	- `Soft Knee` : 더 부드럽게 적용됨. (?) 왼쪽으로 돌릴 경우(근데 왼쪽으로 돌리면 아예 반대 방향으로 솟구치는데, 오른쪽으로 덜 돌리는 게 맞는 것 같기도.)
- `Ratio`
	- 컴프레션이 적용되는 정도. 
	- 노브를 돌린 채 좌측 상단을 보면 `Comp Ratio : n : 1`이라고 나온다. 
		- 초과한 `n` 데시벨을 `1` 데시벨로 바꿔서 적용하겠다는 의미.
	- 왼쪽으로 돌리면 컴프레션이 적용되는 영역의 소리가 커지고 오른쪽으로 돌리면 작아진다.

- `Knee vs Ratio` 
	- `Knee`는 컴프레션의 변화 속도를 나타냄
	- `Ratio`는 컴프레션의 변화량을 나타냄

![[Pasted image 20260101204907.png]]
> 이런 비율이라면 소리가 파란 선을 초과하는 영역, 즉 소리가 큰 영역에서 컴프레서가 걸려서 소리가 더 작게 나온다. 
> - 참고) 컴프레서를 건 채로 다시 재생시키면 컴프레서가 적용된 파형이 나오므로 위와 같은 형태는 아님. 위 예제는 컴프레서를 걸지 않은 상태에서 멈춘 상태.


- `ATT(어택)` : `Threshold`를 넘었을 때 볼륨을 줄이기 시작하는 시간
	- 기본 0초.  큰 소리를 바로 줄인다. 
	- 1초까지 설정할 수 있다. 값이 클수록 누르지 않은 음이 들리기에 타격감을 살릴 수 있지만, 컴프레서의 기능(소리 죽이기)을 못할 수도 있다.

- `Rel(릴리즈)` : 컴프레서의 동작으로 인해 소리가 작아졌을 때 원래 볼륨으로 되돌리는 시간
	- 기본 0.25초. 
	- 값이 작다면 소리가 작아지자마자 볼륨을 확 키운다. 에너지가 넘치게 되지만, 너무 빠르면 `왜곡Distortion`이 생길 수 있다. 
	- 크다면 볼륨을 천천히 올린다. 소리가 부드러워지지만 다음 소리가 나올 때까지 볼륨이 계속 눌려 있어서 전체적인 에너지가 줄어든다.

- 비교
- 예시 1) 베이스
	- 빠른 어택 & 빠른 릴리즈 : 빠르게 소리가 줄어들고, 빠르게 회복됨
	- 느린 어택 & 빠른 릴리즈 : 느리게 소리가 줄어들고, 빠르게 회복됨
	- 빠른 어택 & 느린 릴리즈 : 빠르게 소리가 줄어들고, 천천히 회복됨
	- 느린 어택 & 느린 릴리즈 : 느리게 소리가 줄어들고, 천천히 회복됨

> 개인적으로 특히 눈에 띄는 부분은 느린 어택이다. 첫 타격음이 귀에 거슬릴 정도로 세게 들리기 때문.

- 예시 2) 드럼
	- 빠른 어택 & 릴리즈 : 소리가 탁하고 디스토션이 많이 들림.
	- 느린 어택 & 빠른 릴리즈 : 감소 속도가 느려서 타격음이 선명하지만 빠른 릴리즈로 인한 디스토션이 있음
	- 빠른 어택 & 느린 릴리즈 : 소리가 탁하지만 느린 릴리즈로 인해 디스토션이 없음
	- 느린 어택 & 느린 릴리즈 : 첫 킥이 강하게 들리지만 이후 소리들은 작게 들림 - 릴리즈 시간이 느리기 때문
![[Pasted image 20260101213845.png]]
> - 흰색 영역이 결과 아웃풋. 왼쪽의 피크가 킥인데, 1번째 피크보다 2번째 피크가 확연히 작은 걸 볼 수 있다.
> - `Threshold`를 넘는 소리가 나올 때마다 컴프레서가 적용된다고 생각하면 될 듯.
> - 이런 식으로 소리가 줄어드는 전반적인 현상을 `Gain Reduction`이라고 함

그래서 이 드럼 예시에서는 어택과 릴리즈 모두 극단적으로 돌리지 말고 `Gain Reduction`이 일어나지 않는 선 정도로 조절해주면 됨

- `Gain` : 소리 전체의 크기를 조절한다고 이해하면 편함
	- (정확히 말하면 볼륨과는 다른 개념으로 알고 있긴 함)

컴프레서가 적용된 곳은 윗쪽의 흰 선이 아래로 내려오는 정도로 알 수 있음

- **일반적인 작업**
1. **컴프레서로 피크 부분의 소리를 죽임**
	- 그래서 `Threshold`가 너무 낮으면 안됨. 
	- 소리가 너무 튀는 부분을 찾아 `Threshold`를 설정해주는 게 좋다.
2. **전체적인 볼륨이 죽기 때문에 게인을 올려서 전체 소리를 보정함**

강의에서 보컬 + 기타로 된 예제를 들려주는데, 보컬이 힘있게 뱉는 부분에서 귀가 불편할 정도로 소리가 튄다는 느낌이 듦

![[Pasted image 20260101211230.png]]
- 왼쪽 아이콘들(위부터)
	- 인풋(보라색 영역)
	- 아웃풋(초록색 영역)
		- 두 영역이 겹친 정도가 흰색으로 표시됨. 
		- 보라색 영역과 흰색 영역이 같이 나타나는 부분은 컴프레서가 적용된 영역.
	- 게인 적용된 정도
	- Makeup Gain (= 기본 소리 레벨? 정도로 생각하면 될 듯?)

- **프리셋**
	- 우측 상단에 프리셋이 있음
	- 좌측 상단 화살표로 현재의 세팅을 프리셋으로 저장할 수도 있음


## 예제
- 기타, 보컬 예시를 강의에서 줌
- **튄다고 생각되는 영역**을 플레이리스트에서 지정해서 루프를 반복하게 만듦
	- 시간 부분을 Ctrl + 드래그 해서 지정할 수 있음
	- `Ratio`의 시작 추천 값은 `4.0 : 1`
	- `Knee`도 오른쪽으로 쫙 돌리고 시작

1. `Threshold`, `Ratio` 값 설정
- 위 상태에서 설**정한 `Gain Reduction`을 확인하기 위해, 다른 부분을 지정해서 들어본다.**
	- **다른 튀는 부분**
	- **조용한 부분**
	- **더 튀는 부분도 있음**

- 위 까지는 소리를 듣는 것 + 시각적으로 `Threshold`가 그래프에 잘 적용되는지를 체크하면 됨
	- 조용한 부분은 `Threshold`가 높아서 `Gain Reduction`이 적용되지 않게
	- 더 소리가 큰 부분은 `Gain Reduction`이 잘 적용되는지

2. `Attack, Release` 값 설정
이렇게 적절한 `Threshold` 값을 찾으면, 다음은 `Attack, Release` 값을 설정

- `Att`
	- 복습) 소리가 줄어드는 데 걸리는 시간이라고 했음
		- 너무 느리면 타격감이 강해짐, 소리를 줄인 효과가 나지 않을 수 있음
	- 컴프레서 효과가 나면서도 보컬이 강조되는 적절한 지점이 있음. 강의에선 6ms 정도로 줬다. 
- `Release`
	- 복습) 줄어든 소리가 복구되는 시간
		- 값이 너무 크다면 컴프레서 효과가 오래 지속되는 것이므로 `Gain Reduction` 효과가 강해짐
		- 값이 너무 작다면 소리가 들쭉날쭉해지는 게 느껴짐 : `Pumping Effect`
	- 이것도 마찬가지로 적절한 지점을 찾아주면 됨 - 소리가 너무 탁하지 않게, 그러면서도 들쭉날쭉이 느껴지지 않는 지점.
- `Knee` 값도 살짝 줄여서 소리를 부드럽게 만들 수 있다
	- 차이는 미묘한데 부드러운 정도에서 차이가 있다고 하긴 함

2. `Gain` 설정
이러한 **세부 사항들(Threshold, Ratio, Knee, Attack, Release)을 다 조율하고 나면 마지막으로 `Gain`을 만져준다.**

예제의 경우 전체 소리를 크게 줄인 게 아니기 때문에 게인 값도 크게 만질 필요는 없음
게인을 만질 때 참고할 점 : **게인 설정은 작게 진행**

> 이렇게 **큰 부분의 소리를 줄이고, 전체적인 게인을 올리면 가장 조용한 부분의 소리도 가장 큰 부분의 소리와 어느 정도 매칭이 된다.**

## Sustain 다이얼
- 컴프레서의 작동 후, 릴리즈 단계로 넘어가기 전까지 `Compress` 상태를 얼마나 유지할 것인가를 결정한다.
- 노브를 올리면 큰 소리가 기준선 아래로 떨어진 후에도 바로 볼륨을 키우지(=릴리즈하지) 않고 설정된 시간만큼 압축 상태를 `유지Sustain`한다
- 릴리즈가 너무 빨리 일어나서 소리가 떨리는 현상`Distortion`을 방지하거나, 볼륨이 너무 급격히 변하는 걸 막아 소리를 더 탄탄하게 일정하게 잡아주는 역할을 한다.

- 기본값 : `0.1초`
	- 긴 서스테인은 보컬 등의 자연스러운 소리에 어울림
	- **서스테인을 키울수록 컴프레서는 더 긴 신호에 반응함**
		- 즉 순간의 소리보다는 전체적인 소리의 흐름에 반응한다고 보면 될 듯
	- 반대로 서스테인을 줄일수록 짧은 신호에도 컴프레서가 반응함
	- 추천 예시
		- 드럼 : 짧은 서스테인 추천
		- 보컬, 베이스 : 자연스러운 소리를 위한 긴 서스테인 추천

## Curve Control
![[Pasted image 20260101224750.png]]
- `Attack, Release`에 연결된 값
- 숫자가 
	- 낮다면 볼륨 변화가 딱딱하고 
	- 크다면 천천히에서 시작해서 빠르게 변화함
- 값에 따른 어택 / 릴리즈의 변화 형태를 바꾸는 건데, 약간의 **부차적인 요소이므로 어택 / 릴리즈 / 임계값`threshold` / 비율`ratio` 값을 바꾸는 데 시간을 쓰는 게 훨씬 낫다.** 짬이 좀 차면 건드리는 걸 추천


## SideChain
- 사전 준비가 필요함 
	- `SideChain`으로 사용할 다른 채널을 믹서에 지정
	- `SideChain`이 적용될 채널(=`Fruity Limiter`가 있는 채널)로 위의 채널에서 연결함
		- 연결 과정은 `믹서 채널 아래에 있는 화살표 -> SideChain to this track`

- 예를 들어서 일정하게 들리는 `Bass`가 `SideChain`이 적용될 채널, 킥 드럼이 `SideChain`으로 연결되었다고 하면, **`Bass`를 재생하면 킥 드럼이 함께 재생되면서 킥 드럼이 나오는 부분은 순간적으로 `Bass`의 소리가 작아졌다가 커지는 현상이 발생**함
	- 이것도 일종의 `Pumping Effect`

## Saturation
- 디지털보다는 아날로그 같은 효과를 낼 떄 사용함.
