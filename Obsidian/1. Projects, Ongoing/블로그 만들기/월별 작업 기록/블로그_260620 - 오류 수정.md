
## 오류 수정
### 발생 중인 오류
- `ReviewModal`의 수정 버튼을 누르면 팝업을 닫으려는 동작이 이뤄짐
	- 다른 탭에 다녀와도 해당 동작이 발생함
- 이거 생각보다 복잡한 듯?
### 해결
- 이거저거 찾아 헤맸음.
- 역시 디버깅의 출발점은 **"추가한 뭔가를 제거했더니 발생하지 않는다"에서 시작**하는 듯. 이걸 발견하는 게 막연하게 질문하는 것보다 빠른 것 같다. 헤매는 과정에서 적용해도 제대로 되지 않는 답변 2개를 받았음.

---
- `ReviewModal`에는 이런 코드가 있었다.
```tsx
useBackHistory(!!reviewId, isEditing ? handleAttemptClose : onClose);
```

- 얘를 주석처리하니까 더 동작하지 않는 걸 봐선 얘가 문제인 듯.
- 근데 얘는 "뒤로 가기"를 클릭했을 때만 동작시키려는 훅이었다. 왜 지금 동작할까?

- AI 답변
	- 수정 버튼을 클릭하면 `ReviewModal.IsEditing`의 상태가 바뀌고, `useBackHistory`에 들어가는 함수가 `handleAttemptClose`로 변경된다. **`useEffect`가 재실행, `pushState`가 재호출되고, 기존 `popState` 핸들러가 제거되고 새 핸들러가 등록**된다.
	- 다만, `pushState` 호출 자체가 일부 브라우저에서는 `popState`을 트리거하는 경우가 있다. 이 때 새로 등록된 핸들러가 `event.state?.historyId`를 체크하는데, 타이밍에 따라 조건을 통과해서 `onClose()`인 `handleAttemptClose`가 호출될 수 있다.
	- `isStatePushed.current`가 `true`일 때는 `pushState`를 막지만, `onClose`가 바뀌면 `useEffect`의 재실행으로 인해 클린업 -> 재등록 사이클이 돌기에 이러한 체크가 의미없어진다.
	- 따라서, **`onClose()`를 `useEffect()` 의존성에서 제거하고, `ref`로 관리**하면 된다.

```tsx
export const useBackHistory = (isOpen: boolean, onClose:() => void) => {
    const onCloseRef = useRef(onClose);
    
    // onClose가 바뀌어도 useEffect이 재실행되지 않게 ref로 최신값을 유지함
	useEffect(() => {
		onCloseRef.current = onClose;
    }, [onClose]);
    
    
	useEffect(() => {
		// ...
		
		const handlePopState = (event: PopStateEvent) => {
			
			// 뒤로 가기를 해서 도착한 히스토리의 상태가 내 ID와 같다면, 
			// 아직 내 히스토리 안에 있다는 뜻이므로 무시
			if (event.state?.historyId === instanceId.current ) return;
	
			isClosedByPopState.current = true; // 뒤로 가기 플래그 기록
			onCloseRef.current(); // onClose에서 변경
		}
		
	}, [isOpen]) // onClose 의존성 제거
```
> 수정 내용
> - 기존 : 하단 `useEffect` 문에 `onClose`가 있었고, 내부에도 `onClose`를 실행하는 함수가 있었음. 
> - 수정 : `onCloseRef`을 별도로 관리. 함수를 실행시키지 않고, `onClose`가 다른 것으로 바뀌었다는 내용만 저장함.

- `handleAttemptClose`가 동작하는 다른 상황으로, 다른 탭에 다녀올 때도 그런 일이 있었다. 이것도 함께 해결된 이슈이긴 한데, 얘도 원인이 궁금해짐.
	- AI 답변 
		- 탭 전환 시 일부 브라우저는 `popstate` 이벤트를 발생시킨다. 이후의 디테일한 설명은 위와 동일. 클린업 -> 재등록 사이클 과정에서 `pushState`와 `popstate` 핸들러 등록 타이밍이 어긋났음.

## 기타 수정 사항
1. 수정 상태에서 "뒤로 가기"로 `handleAttemptClose`가 동작할 때, 취소를 누르는 경우 현재 수정 상태가 유지됨. 이 상태에서 다시 뒤로 가기 버튼을 누르면 `handleAttemptClose`가 동작하는 게 아니라 **그냥 모달이 꺼져버림.**
2. `Post` 글 쓰기 / 수정에서 링크를 벗어나려고 할 때 확인 질문이 나타나지 않음

- 읽는 경우는 아예 고려하지 않겠음. 나가지면 다시 돌아오면 되니까.

- 글을 쓰거나 수정하는 경우가 문제가 됨. 정보가 다 날아갈 수 있기 때문.

중요한 건 **글을 쓰는 중에 창을 닫거나 이동하려고 할 때 곧바로 돼선 안된다**는 것.

- 네이버에서 메일을 쓰는 부분이나, 카페에 글을 쓰는 부분을 확인해봤음
	- 메일 : **뒤로 가기를 누르면 일시적으로 뒤로 갔다가 앞으로 돌아오는 것으로 보임.** 이 돌아온 창에 팝업이 나타나며, 확인을 누르면 뒤 페이지로 나가지고 취소를 누르면 현재 페이지에 남음.
		- 바로 나가지는 경우도 종종 있음
	- 카페 : **아예 새로운 탭에 글을 쓰게 되어 있음.** 이 탭을 닫으려고 하면(탭을 닫든, 브라우저를 끄든) 나갈지 말지를 물어봄.

- 이외에도 글을 작성 / 수정하는 페이지도 그냥 나가지는 현상이 있음. (이건 왜 갑자기 그런지 모르겠다. 이전에 이상해서 수정한 기억이 있는데..)

- 아래 파일들을 수정함. 내용을 다 옮기진 않겠음. 구현 원리보다는 "구현했다"에 집중하고 있는 프로젝트라서.
	- `useBackHistory.ts`
	- `useLeaveGuard.ts`
	- `ReviewModal.tsx`
	- `ReviewBoardPage.tsx`
	- `useBeforeUnloadGuard.tsx`

- 수정한 사항
	- 페이지 나갈 때(뒤로 가기 및 창을 닫는 상황) 물어보는 팝업 띄우기
	- 리뷰 모달 수정 중에 **뒤로 가기 시 수정 상태만 바뀌고 나가지지 않음**
	- 리뷰 모달 수정 중에 창을 닫는 상황 시 물어보는 팝업 띄우기

- 더 확인해야 할 사항(AWS에 올린 후에)
	- `Post` 작성하는 로직이 제대로 동작하는가? : O

- 추가로 발견한 이슈
	- 글 작성 중에 창을 벗어날 때 취소 버튼이 가끔 잘 클릭되지 않음
	- ReviewModal을 수정 중일 때, 뒤로 가기는 `IsEditing`을 바꾸지만 닫기 버튼이나 바깥 레이어는 그냥 모달이 꺼짐. 세 동작 모두 물어보는 팝업은 잘 나옴.
		- 이게 자연스러운 것 같아서 냅둠.
