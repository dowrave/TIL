## 도메인 이전 과정(등록 과정과 크게 다르지 않을 듯?)
- 작성 일자는 26년 5월 1일.

1. 도메인 구매 사이트(내 경우는 반값도메인을 사용 중)에서 도메인 구매

2. AWS의 `Route 53`에서 **호스팅 영역**에서 구매한 도메인 이름으로 호스팅 영역을 만든다. `public`임.

3. 그러면 `유형 : NS`에 해당하는 `값/트래픽 라우팅 대상`에 4개의 URL이 나타난다. 이걸 도메인 구매 사이트의 `네임 서버 변경`에 적어넣는다.
	- 각 네임서버 정보에는 `HOST`와 `IP`가 있는데, 2년 전에도 `IP`를 적어넣지는 않았으니 `HOST` 값만 넣으면 될 듯. 실제로도 `IP` 값이 따로 나타나진 않았다.
	- 이메일이나 휴대폰 인증을 요구함

4. AWS의 **`Certificate Manager(=ACM)`에서 인증서를 요청**한다. 내 경우 클라우드 프론트를 쓰고 있는데, 이 떄 **ACM의 서버는 `미국(버지니아 북부), us-east-1`** 이어야 한다.
	- **인증서의 상태가 발급 대기 중일 때, 들어가서 `Route 53에서 레코드 생성` 탭을 눌러서 빠르게 처리할 수 있다.**
	- 도메인이 외부에서 관리된다면 CNAME 이름, 값을 복붙해서 네임서버에 등록하는 방식으로 작업하지만, 이 과정에서는 `Route 53`에서 해당 도메인을 관리하도록 구성했기 때문에 이 과정을 훨씬 빠르게 처리할 수 있음.

5. 클라우드프론트의 설정에서 `사용자 정의 SSL 인증서` 및 `대체 도메인 이름`을 위에서 발급받은 SSL 인증서와 도메인 이름으로 바꿔주면 됨.

## 추가 이슈 - 글에 들어간 이미지가 깨지는 현상
- **글에 들어가 있는 이미지의 링크가 클라우드 프론트 기반으로 연결되어 있다.** 그런데 연결된 도메인이 업데이트되더라도 텍스트값 자체는 여전히 `htlee-blog.com`으로 되어 있어서 이미지들이 모조리 깨지고 있음
- 반면에 리뷰에서 표지로 사용하고 있는 이미지는 잘 업데이트되었음

- 텍스트 수정
	- 크게 DB 자체에서 수정하는 방법과 Django를 이용해 수정하는 방법이 있음
	- `LightSail`의 콘솔에서는 한글이 한글로 나타나지 않고 유니코드로 나타난다.

- 백엔드 컨테이너로 접근해서 쉘 돌리는 방법 적용함.
```bash
docker exec -it <장고_컨테이너_이름> python manage.py shell
```

```python
from your_app.models import Post  # 'your_app'과 'Post'를 실제 이름으로 수정

# 1. 수정 대상이 있는지 먼저 확인
targets = Post.objects.filter(content__contains='htlee-blog.com')
print(f"총 {targets.count()}개의 글을 수정합니다.")

# 2. 반복문을 돌며 안전하게 치환
for post in targets:
    # 기존 내용을 변수에 담아 치환
    old_content = post.content
    new_content = old_content.replace('htlee-blog.com', 'waltwaez.com')
    
    # 실제 값이 바뀌었을 때만 저장 (불필요한 DB 접근 방지)
    if old_content != new_content:
        post.content = new_content
        post.save()
        print(f"ID {post.id}번 글의 도메인 치환 완료")
```
> 접근할 모델이 크게 블로그 글 / 리뷰 글 2개가 있었음

이거 수정하니까 이미지가 있는 리뷰 글의 수정 시간까지 모조리 바뀌었다.
