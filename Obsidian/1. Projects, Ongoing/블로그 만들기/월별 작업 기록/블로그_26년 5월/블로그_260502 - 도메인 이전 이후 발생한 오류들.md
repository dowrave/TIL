## 메인 화면에서의 오류

```
django.core.exceptions.DisallowedHost: Invalid HTTP_HOST header: 'api.htlee-blog.com'. You may need to add 'api.htlee-blog.com' to ALLOWED_HOSTS.
Invalid HTTP_HOST header: '{IP 주소}'. You may need to add '{IP 주소}' to ALLOWED_HOSTS.
```

- 원인
	1. ip 주소의 경우 `ALLOWED_HOSTS`에 `,`가 안 찍혀 있었음;;
		- 로컬에서 파이썬을 지웠나? 보다? 그래서 오류 체크가 확실하게 되지 않은 듯.
	2. `api.htlee-blog.com`이 들어오는 현상은 확실히 이상하다. 프론트엔드에서 이미지 업로드 부분을 살펴보니 `cloudfrontUrl`이라는 변수명으로 이전 블로그 도메인을 사용하고 있었음.

- 프론트엔드, 백엔드 다 문제가 있어서 수정하고 테스트해봄

- 여전히 둘 다 안됨

1. Post로 작성하는 부분은 잘 동작하는데, MyDoing은 잘 동작하지 않는다.

- 프론트 : 주소는 명시적으로 적어둠(백엔드로 요청이 가는 거는 맞아서 이거 이슈는 아니지만 고쳐둠)
```tsx
  if (isNewContent) {
	await axios.post(`${backend}api/blog/mydoing/`, 
	  formData,
	  { withCredentials: true }
	);
  } else {
	await axios.put(`${backend}api/blog/mydoing/`, 
	  formData,
	  { withCredentials: true }
	);
  }
```

- 백엔드
```python
class MyDoingView(APIView):

	# 데코레이터들을 클래스 레벨로 옮김
    authentication_classes = [JWTAuthentication]

    def get_permissions(self):
		# ...

    # @authentication_classes([JWTAuthentication])
    def post(self, request, *args, **kwargs):
		# ...
	
    # @authentication_classes([JWTAuthentication])
    def put(self, request):
		# ...
```
- 기존에 데코레이터로 달아뒀는데, 이 데코레이터는 FBV 전용이란다. CBV에서는 무시되므로 `post/put` 요청이 `JWTAuthentication` 없이 들어온다고 한다.

- 그래서 클래스 레벨의 `authentication_classes`으로 옮겼음.

