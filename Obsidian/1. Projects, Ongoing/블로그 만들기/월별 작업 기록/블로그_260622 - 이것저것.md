## 1. 제목 구조 수정
	- `react-helmet` 대신 `react-helmet-async`을 사용, 공통적으로 각 페이지의 활성화 요소 뒤에 블로그 제목이 오게끔 설정함.
	- `Section`, `Subsection`의 제목에 오는 영어 단어의 첫글자는 대문자로 표기되도록 수정. `Utils/stringUtils.ts`에 구현해놨다.

- `App.tsx`
```ts
<Helmet 
	defaultTitle="Waltwaez's Blog" 
	// 구분선은 -이랑 |이 있는데, 제목에 일반적으로 |을 잘 안 쓰니까 |으로 사용함
	titleTemplate="%s | Waltwaez's Blog" 
/>
```

- `Main.tsx`
```tsx
ReactDOM.createRoot(document.getElementById('root')!).render(
  <Provider store={store}>
    <QueryClientProvider client = {queryClient}>
      <HelmetProvider> {/* 추가 */}
        <App /> 
      </HelmetProvider>
    </QueryClientProvider>
  </Provider>
)
```

- 이제 각 페이지에서 아래처럼 넣으면, `제목`에 해당하는 부분이 `%s`에 들어가게 됨.
```tsx
<Helmet>
	<title>제목</title>
</Helmet>
```

	- 지금은 수동으로 적어왔음
	- `models.py`의 `MyDoing`에 `updated_at` 필드 추가, `Serializer.py`에서도 `fields`에 `updated_at` 추가

- 프론트에서는 우측 상단에서 해당 값을 받도록 구현

- 메인 페이지의 레이아웃을 바꿈
	- 원래는 최근 게시물 -> 현재 진행 중 -> 각 게시판의 게시물 -> 액티비티 였다면
	- 지금은 현재 진행 중 -> 액티비티 -> 최근 게시물 + 각 게시판의 게시물로 수정
	- 최근 게시물의 헤더 부분만 좀 더 색을 진하게 했음

- 메인 페이지 한정으로 상단의 `Navbar`의 토글 전환 버튼을 누르면 바뀌도록 구현... 해봤는데 같은 버튼이 상황에 따라 동작이 달라지는 것이니까 별로 좋은 기능은 아닌 것 같다.
	- 대신 조건문을 없애서 **토글 전환 버튼을 다른 카테고리의 메인 페이지로 이동시키도록 함.**
