## BGM

### Export할 때(FL Studio 기준)
- **확장자명 : `ogg` 추천(별개로 원곡은 `wav`로 저장해둘 것)**
- **채널 : `stereo`**

- 확장자명 - OGG : 이유 
	- BGM에서 중요한 건 용량과 루프
	- 용량 : `mp3`보다 작고 음질이 좋음
	- 루프 : `mp3`는 파일 구조 상 앞뒤에 미세한 공백이 발생할 수 있음, `ogg`는 시작과 끝의 부드러운 연결이 가능
	- 범용성 : 모바일, PC 모두 안정적으로 작동

- `mp3`는 호환성이 좋지만 루프가 그렇게 중요하지 않을 때만 사용함

### 유니티에 넣을 때
- **`Load Type` : `Streaming`**
	- 파일이 큰 BGM은 실시간으로 읽어들여야 렉이 발생하지 않음
- **`Compression Format` : `Vorbis`**
	- `OGG`를 위한 효율적인 압축방식
- `Quality`를 조절해서 더 용량을 줄일 수도 있음. `70~80` 정도로도 괜찮음.

## 효과음(SFX)
### Export할 때(FL Studio 기준)
- **확장자명 : `wav` 추천**
- **채널 : `mono(merged)`**
- **`Tail : Cut Remainder`**

- 이외에도 효과음은 잛기 때문에  `Sample Rate`을 굳이 `44100hz`로 잡지 않고 `22050hz`로 내보내도 음질 차이는 거의 없으면서 용량을 줄일 수도 있음

- 확장자명 : `wav`인 이유
	- 효과음에서 중요한 건 레이턴시가 없고, 가벼워야 함
	- `mp3, ogg`는 CPU가 압축을 푸는 시간이 필요하지만 **`wav`는 즉시 재생 가능**
		- 압축을 푸는 데 드는 CPU 부하가 없음
	- **음질 손상 없음**
### 유니티에 넣을 때 
- `Force to Mono` 체크
- `Load Type` : `Decompress On Load`
- `Compression Format` 
	- `PCM` : 압축하지 않은 원음. 아주 짧은 소리에 사용
	- `ADPCM` : `PCM`보다 용량은 작으면서 CPU 부담이 적어 효과음에 가장 많이 쓰인다
