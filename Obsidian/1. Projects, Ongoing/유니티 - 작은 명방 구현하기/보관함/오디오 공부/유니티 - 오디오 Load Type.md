#유니티 #오디오 

| Load Type            | 메모리    | CPU | 적합한 용도   |
| -------------------- | ------ | --- | -------- |
| Decompress on Load   | 많이 사용  | 낮음  | 짧은 효과음   |
| Compressed in Memory | 적게 사용  | 중간  | 중간 길이 음원 |
| Streaming            | 거의 안 씀 | 중간  | 긴 배경음악   |

- 3가지 `Load Type`
	- `Decompress on Load`
		- 파일을 메모리에 올릴 때 즉시 압축 해제해서 통째로 올림
		- 재생 시 CPU 부하는 없지만, 메모리를 많이 사용함
		- **짧은 효과음**
			- 파일이 작아 메모리에 올려도 부담 없고, 즉각적인 재생이 중요하며, 동시에 여러 개가 터질 수 있다.
			- 재생 요청이 오는 순간 바로 재생된다. 재생 시점에 CPU가 일을 하는 것도 아니니까 동시에 여러 개가 터져도 문제 없다. 
	- `Compressed in Memory`
		- 압축된 상태로 메모리에 올린 다음, 재생할 때마다 압축 해제
		- 메모리를 절약하지만 재생 시마다 CPU 부하 발생
		- **중간 길이 음원(캐릭터 보이스 등)**
	- `Streaming`
		- 메모리에 올리지 않고 파일에서 실시간으로 읽어서 재생함
		- 메모리를 거의 쓰지 않지만 디스크 I/O발생
		- **긴 배경 음악**
			- 파일이 크고, 1~2개만 재생되며, 끊김 없이 지속되는 게 중요하다.
			- 용량이 생각보다 크기 때문에 메모리에 올려두면 낭비가 심하다. `Streaming`은 재생할 부분만 잘라서 떼오는 방식이므로 메모리를 거의 안 쓰고, BGM은 어차피 1~2개만 재생되니 부하도 문제 없다.

> 캐릭터 보이스를 `Streaming`으로 하지 않는 이유 
> - **동시에 여러 음원을 스트리밍하면 디스크 읽기 부하가 누적**된다. (보이스는 겹칠 수 있기 때문. 컷씬 말고 여러 NPC가 말하는 상황을 생각해보자)
> - BGM 등은 1~2개만 재생되므로 크게 상관이 없지만, 여러 개가 동시에 발생할 수 있는 음원에 사용되면 재생 지연이나 끊김이 발생할 수 있음.

>[!note]
>- 각 용도와 그 이유를 생각해보면, **파일 크기와 동시 재생 수**가 기준이 된다.
>- `Decompress On Load` : 작고 많이 터지는 것 - 미리 많이 올려두기
>- `Streaming` : 크고 하나만 재생되는 것 - 그때그때 읽어오기
>- `Compressed In Memory` : 압축된 채로 올려두기
