#오디오

- 세 방식의 가장 큰 차이는 "압축 방식"이다.

| 포맷  | 압축 방식 | 특징                          |
| --- | ----- | --------------------------- |
| WAV | 무압축   | 원본 그대로, 용량 큼                |
| MP3 | 손실 압축 | 용량 작지만 품질 손상                |
| OGG | 손실 압축 | MP3와 비슷하나 품질 대비 용량 효율이 더 좋음 |

- `손실 압축` : 사람이 잘 못 듣는 주파수 대역을 버려서 용량을 줄임.

- 게임에서의 사용
	- `WAV` : 편집/제작 단계에서의 원본 보관용 / 짧은 효과음
	- `OGG` : 게임 빌드에 들어가는 파일 (배경 음악, 긴 음원)
	- `MP3` : 유니티에서 지원하지만 OGG보다 효율이 낮아서 잘 안 씀

- 유니티에서의 실제 흐름
	- `import` 시점에 유니티 내부 자체 포맷으로 변환한다. 어떤 포맷으로 넣든 최종 빌드 품질은 유니티 설정에 따라 결정된다.
	- 따라서 실무에서 추천하는 흐름은
		- `WAV` 작업 -> `WAV`로 넣기 -> 압축은 유니티 `Load Type`에서 처리
