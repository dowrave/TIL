
- "처리 속도"에 관한 이야기는 일반적으로 CPU에 데이터가 드나드는 속도 차이라고 생각하면 됨
- 모든 처리 작업은 CPU에서 이뤄짐

- CPU로 데이터를 가져오는 속도
	- 캐시(L1, L2, L3) : `수 ns`
	- RAM : `수십 ~ 수백 ns`
	- SSD : 수십 마이크로초
	- HDD : 수 밀리 초

>[!question]
>왜 속도 차이가 나는가? 
-> 물리적인 이유다. **속도와 용량은 트레이드오프 관계**

| 종류  |     위치     | 용량  |
| --- | :--------: | :-: |
| 캐시  |  CPU 칩 내부  | MB  |
| RAM | 보드에 별도로 장착 | GB  |
| 디스크 | 보드에 별도로 장착 | TB  |

>[!Question]
>속도를 중시한 소자와 용량을 중시한 소자의 구조는 어떻게 다른가?
>1. 전기적 접근 vs 물리적 동작 : **전기적 접근이 빠름**
>2. 단순한 구조 vs 복잡한 구조 : **비트를 저장하는 소자가 단순할수록 집적도가 높아지지만, 속도를 위한 부가 장치가 줄어든다**

## SRAM vs DRAM
- 캐시와 RAM이 차이나는 이유

### SRAM(캐시)
- 트랜지스터 6개로 1개의 비트를 저장함
- 전원이 있는 한 데이터가 유지됨
- 빠르지만 트랜지스터가 많이 필요하므로 집적도가 낮고 비쌈

### DRAM(RAM)
- 트랜지스터 1개 + 커패시터 1개로 1개의 비트를 저장함
	- 여기서의 트랜지스터는 스위치 역할
- 커패시터는 서서히 전하가 빠져나가서 주기적인 재충전`Refresh`이 필요함
	- `Refresh`가 속도를 늦추는 원인 중 하나
- 구조가 단순하므로 집적도가 높고 저렴함

> 같은 면적에 트랜지스터가 6개 vs 1개인 셀들을 비교하면, 후자가 6배 많은 비트를 넣을 수 있다. 트랜지스터 하나하나가 물리적 공간을 차지하기 때문이다.

## 디스크

### 낸드플래시SSD
- 플로팅 게이트에 전하를 가둬두는 방식
- 비휘발성
- 읽기, 쓰기 단위가 크고 컨트롤러를 거쳐야 해서 레이턴시 발생함
### HDD
- 자기 디스크를 물리적으로 회전시켜서 헤드가 읽는 방식
- 기계적인 동작이 들어가는 순간 ms 단위로 느려짐
- 원하는 위치로 헤드가 이동하는 시간 Seek Time이 병목
