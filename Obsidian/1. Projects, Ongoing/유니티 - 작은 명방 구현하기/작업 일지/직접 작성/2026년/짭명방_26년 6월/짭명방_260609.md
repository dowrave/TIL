### `StageResultPanel`의 결과 패널 텍스트가 클릭 가능하게 된 후에 바뀌는 현상
```cs
private IEnumerator BlockingClick()
{
	yield return new WaitForSecondsRealtime(blockDurationTime);
	blockingPanel.gameObject.SetActive(false);
	returnToLobbyText.text = "로비로 돌아가기"; // 여기가 이슈
}
```
> - 저 `returnToLobbyText`라는 부분은 `Localize String Event`로 관리되던 부분임
> - 스크립트 단위에서는 Awake에서 ""으로 처리된 다음 `BlockingClick`에서 일정 시간 후에 값이 들어가는 방식임
> - 근데 **그냥 Awake에서 비활성화하고 저 부분에서 활성화하는 것만으로 충분**해 보임.

#### 코드 단위에서 구현하기 vs 에디터에서 구현하기
- "런타임에서 **상태에 따라 내용이 바뀔 수 있는 경우라면 스크립트 단위에서 구현해야 한다**"로 알고 있었음.
- 어떤 구조를 가져갈지는 상태의 갯수로 판단하면 될 듯.
	1. **상태가 많지 않은 경우** : 같은 구조의 TMP를 여러 개 배치하고, 그 중 하나만 스크립트 단위에서 활성화 여부만 조절함
		- 예시 : 결과 패널의 스테이지 클리어 / 실패 시 나타날 "임무 성공", "임무 실패"
	2. **상태가 많은 경우** : 1개의 TMP를 넣고, 해당하는 키를 다른 곳에서 찾아와서 넣는 방식.
		- 예시 : `StageSelectPanel`에서 표시될 각 스테이지의 정보와 세부 내용 TMP 영역. 스테이지가 늘어날수록 다 넣을 수는 없게 되니까.
### `KeyBinding` 중복 키 상황 처리
- 현재 : 중복되는 키의 버튼 색만 붉은 색으로 바뀜
- 변경 : 현재 변경 중인 키의 버튼 색도 붉은 색으로 표시되게끔 함
```cs
// KeyBindingSection의 중복 처리 부분
if (isDuplicate)
{
	// 현재 컨테이너의 색도 붉게 표시
	waitingContainer.SetButtonErrorColor();

	// 중복 버튼의 색을 붉게 표시
	if (duplicatedContainer == null) return;
	duplicatedContainer.SetButtonErrorColor();

	// 중복일 경우 다시 입력 대기
	// InputSystem.onAnyButtonPress.CallOnce(ctrl2 => OnAnyKeyPressed(ctrl2));
	RegisterKeyWait();
}
```

- 추가 변경 
	1. 중복 버튼의 위치로 스크롤 이동
	2. 중복되는 버튼에 진동 효과

#### 스크롤 이동 - KeyBindingSection
>[!note]
>- **이건 처리 못했음.** 
>- 현재 상황
>	- 윗쪽 컨테이너를 수정 중이고 아랫쪽 컨테이너가 중복될 때 스크롤 이동 X
>	- 아랫쪽 컨테이너를 수정 중이고 윗쪽 컨테이너가 중복일 때  스크롤이 이동하긴 하는데, 2번 클릭하면 원위치로 돌아감. 

#### 진동 효과 - KeyBindingButtonContainer
```cs
public void ShakeButton()
{
	RectTransform buttonRect = button.GetComponent<RectTransform>();
	Vector2 originalPos = buttonRect.anchoredPosition;
	
	if (buttonRect != null)
	{
		buttonRect.DOShakeAnchorPos(duration: 0.4f, strength: 8f, vibrato: 20, randomness: 90)
			.SetUpdate(true)
			.OnComplete(() => buttonRect.anchoredPosition = originalPos); 

		Logger.Log("버튼 흔들림");
	}
}
```
> - 이슈가 있었음 : 버튼 연타 시 버튼의 최종 위치가 계속 바뀜
> - 흔들리는 중에 `originalPos` 값이 저장되기 때문인 것으로 추측함. 재입력을 방지하는 시간을 추가해줌

```cs
// 중복 키 상황 처리
if (isDuplicate)
{
	isProcessingInput = true;
	float shakingTime = 0.4f;

	// 현재 컨테이너 색깔 처리
	waitingContainer.SetButtonErrorColor();

	// 중복 버튼 처리
	if (duplicatedContainer == null) return;
	duplicatedContainer.SetButtonErrorColor();
	duplicatedContainer.ShakeButton(shakingTime);
	ScrollToContainer(duplicatedContainer); // 해당 중복 버튼으로 컨테이너 위치 이동

	// 흔들리는 시간이 끝난 후 잠금 해제 + 재입력 대기
	DOVirtual.DelayedCall(shakingTime, () =>
	{
		isProcessingInput = false;
		RegisterKeyWait(); 
	});
}
// ...
```

```cs
	Key key = GetKey(action);

	// "`"의 경우 그냥 ToString이면 "backquote"로 나가므로 별도 처리
	if (key == Key.Backquote)
		return "`";
	else if (key == Key.Escape)
		return "ESC";
	// 숫자 키 `Digit0 ~ 9`처리
	// else if (key >= Key.Digit0 && key <= Key.Digit9)

	// 넘패드0 ~ 9 처리
	// else if (key >= Key.Numpad0 && key <= Key.Numpad9)
	//	return ((int)key - (int)Key.Numpad0).ToString();
	else 
		return key.ToString();
}
```
