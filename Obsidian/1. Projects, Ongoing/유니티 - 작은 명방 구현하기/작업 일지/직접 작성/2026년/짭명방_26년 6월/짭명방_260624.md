## 작업 내용
### PromotionPanel

#### 공격 범위 표시 부분 
- `TMP`의 오른쪽에 `AttackRangeBox`가 오도록 레이아웃 조절
	- 기존) `TMP`의 `RectTransform`상의 너비가 고정이라서 글자가 `AttackRangeBox`를 침범하는 경우가 있었음

- `ContentSizeFitter`를 추가했는데, 글자 갯수마다 레이아웃이 서서히 달라지는 이슈가 있음. 최초에도 각각의 범위 계산이 제대로 이뤄지지 않는 듯.
![[Pasted image 20260624153030.png]]

- 레이아웃이 갱신되지 않는 상황은 크게 2가지임
	1. 언어 전환 후, 해당 패널에 처음으로 접근할 때
	2. 해당 패널을 띄운 상태에서 언어 전환을 했을 때
	- 그 외의 경우, 즉 같은 언어 상태에서 해당 패널에 2번 이상 접근했을 때는 정상적으로 레이아웃이 잘 나타남.

- 해결 
	1. `Localization` + `TMP` 레이아웃 반영까지 **2프레임을 기다림**
	2. 관련 요소들의 레이아웃 리프레쉬를 전체적으로 수행함

```
[구조]
attackRangeContents(Horizontal Layout Group)
- AttackRangeText(Content Size Fitter)
- AttackRangeBox(Content Size Fitter)
```
- 여기서 3가지 요소(텍스트, 박스, 전체 컨테이너)를 모두 새로고침해줌

```cs
private IEnumerator RefreshLayout()
{
	// Localization + TMP 렌더 완료까지 2프레임의 여유를 확보함
	yield return null; 
	yield return null;

	// TMP 자체 갱신
	attackRangeText.ForceMeshUpdate();

	// 자식 -> 부모 순으로 갱신
	LayoutRebuilder.ForceRebuildLayoutImmediate(attackRangeText.rectTransform);
	LayoutRebuilder.ForceRebuildLayoutImmediate(attackRangeVisualContainer);
	LayoutRebuilder.ForceRebuildLayoutImmediate(attackRangeContainer);
}
```
> 이러한 새로고침을 `OnLocaleChanged` 및 `UpdateUI`에서 모두 진행하면 됨
#### 기타 수정 사항
- ~~`SkillDetailText`에 `ScrollRect` 추가~~
	- 언어 전환 시 스크롤이 나타나지 않는 이슈가 있음. 레이아웃을 고쳐봐도 안돼서, 그냥 `TMP`의 `Auto Size`로 해결

- `SkillDetailText`에 `Localization` 변환이 적용되지 않는 문제 해결

### NotificationToast 관련
- 광클 방지 - 쿨다운 추가
	- 직전에 클릭된 메시지가 같은 토스트일 때만 동작함

```cs
// 광클 방지
private string lastMessage = string.Empty;
private float lastNotificationTime = -Mathf.Infinity;
private bool IsOnCooldown() => Time.time - lastNotificationTime < notificationCooldown;
private void RecordLastNotificationTime() => lastNotificationTime = Time.time;

// 알림 띄우는 메서드의 시작 조건을 추가
if (IsOnCooldown()) return;
RecordLastNotificationTime();
```

> 원래는 4번째 토스트부터 레이아웃이 이상해져서 그걸 바꾸려고 했는데, 지금 정도의 쿨다운을 잡아두면 4번째 토스트가 등장하지 않는다. 쿨다운만으로 충분할 듯.
