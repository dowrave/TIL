## Skill Localization 계속

### ShieldSkill
- 얘만 `StatModifierBuff` 값이 이상하게 설정되어 있었다. 기준값은 0으로 설정.
	- 예시 : `0.3`이면 `+30%`으로 들어가는 방식. `Modifier`들의 합연산으로 버프가 이뤄진다.

```cs
protected override Dictionary<string, object> BuildDescArgs()
{
	return new Dictionary<string, object>
	{
		{ "duration", duration },
		{ "defenseModifier", statMods.defenseModifier * 100},
		{ "shieldAmount", shieldAmount}
	};
}
```

### DoubleShotSkill
- 여기도 `DamageMultiplier`라는 스탯이 `1.2`라는 값으로 들어가 있었다. 이것도 `20%` 증가한 공격력임을 나타내기 위해, `DamageModifier`로 변경하고 `0.2`로 수정
- `duration`이 들어가는 부분은 전부 `{duration:plural:second|seconds}`을 적용해줌
```cs
protected override Dictionary<string, object> BuildDescArgs()
{
	return new Dictionary<string, object>
	{
		{ "duration", duration },
		{ "damageModifier", damageModifier * 100}
	};
}
```

> `duration`이 있는 부분은 영어 번역 시 뒤에 `{duration:plural:second|seconds}`로 들어가게끔 세팅
> - 번역은 제미나이 돌리는 중

### SmashSkill
- 특정 공격마다 강화된 공격이 나가는 스킬. 이름 고민이 잠깐 됐는데 그냥 진행함
```cs
protected override Dictionary<string, object> BuildDescArgs()
{
	return new Dictionary<string, object>
	{
		{ "damageMultiplier", damageMultiplier * 100 },
		{ "attackType", LocalizationManager.GetAttackTypeName(attackType) }
	};
}
```

### StatModifierWithOnHitSkill => WeaponEmpowerSkill로 이름 변경
- 다른 거랑 볼 때마다 헷갈림;; 
- 이 네이밍이 좀 어렵다.
- 공격력 옵션을 강화하고, 공격 타입을 바꾸고, 스턴을 부여하는 스킬이므로 "무기 강화" 혹은 "마법 부여" 정도가 적절해보임.
- 내용은 어제 했던 건데, 이름이 `MagicalSlashSkill`이랑 겹쳐서 헷갈린다. 얘도 나중에 수정해야할 듯. 
### CostRecoverySkill
- 뱅가드가 먼저 나가서 전장을 파악한다 = 배치 코스트를 확보한다는 개념으로 접근함 
	- **`선행 정찰`** 이라는 이름을 붙였음
```cs
protected override Dictionary<string, object> BuildDescArgs() 
{
	return new Dictionary<string, object>
	{
		{"costRecoveryAmount", costRecoveryAmount}  
	};
}
```

## 이슈 수정
### KeyBindingSection의 초기화 기능 동작하지 않음
- 이게 동작하지 않는다기보다는 팝업의 새로고침이 안 이뤄지는 듯?
```cs
    public void ResetKeyBindings() 
    {
        currentBindings = defaultBindings;
    }
```
리셋 메서드인데 따로 초기화에 대한 부분은 없음

- 컨테이너 단위로 구현되고 있기 때문에 여기서 이벤트를 발생시키고 각 컨테이너가 구독해서 수정하는 식으로 진행

```cs
// KeyBindingManager.cs
public void ResetKeyBindings() 
{
	Logger.Log("[KeybindingManager]ResetKeyBinding 동작");
	currentBindings = defaultBindings;
	ResetKeyBinding?.Invoke();
}
```

```cs
// KeyBindingButtonContainer.cs
private void Start()
{
	if (GameManagement.Instance.KeyBinding != null)
	{
		keyBinding = GameManagement.Instance.KeyBinding;
		keyBinding.ResetKeyBinding += OnResetKeyBinding;
	}
}

private void OnResetKeyBinding() => SetCurrentKeyBindingText();

private void SetCurrentKeyBindingText()
{
	string keyName = keyBinding.GetKeyString(keyAction); 
	SetButtonText(keyName); 
}

// 메서드 이름 변경 : Reset -> CancelEditing
public void CancelEditing()
{
	isEditing = false;
	SetCurrentKeyBindingText();
	SetButtonNormalColor();
}

private void OnDestroy()
{
	// ...
	keyBinding.ResetKeyBinding -= OnResetKeyBinding;
}
```

- 추가로, **`ResetKeyBindings`에 저장하는 코드가 없어서 `SetKeys()`을 이용하게 수정**(내부에 저장로직까지 포함)
```cs
public void ResetKeyBindings() 
{
	foreach (KeyAction action in Enum.GetValues(typeof(KeyAction)))
	{
		if (action == KeyAction.None) continue;
		SetKey(action, defaultBindings[action]);
	}

	ResetKeyBinding?.Invoke();
}
```


>[!note]
>- 이전에 `JsonUtility`를 사용했던 저장 구조가 신경쓰이기 시작했는데... 일단은 `PlayerPrefs`만으로 거의 다 할 수 있으니 이렇게 진행하는 걸로.
