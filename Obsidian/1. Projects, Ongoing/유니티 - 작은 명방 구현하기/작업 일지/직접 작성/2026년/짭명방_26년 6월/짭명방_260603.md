## 추가 구현 : 인게임에서 언어 변경
- 변경 로직 자체는 `LocalizationSettings.SelectedLocale` 값을 변경하는 것으로 가능

1. `Language`라는 `enum` 타입을 정의하지 않아도 됨. **`Locale`로 처리 가능**
2. "현재 사용 중인 언어"는 `PlayerPrefs`로 관리된다.  `Unity.Localization.SelectedLocale`이라는 키값으로 관리됨.
	- 저 값을 직접 건드릴 필요는 없다. **실사용은 `LocalizationSettings.SelectedLocale`을 통해 읽고 쓰기만 하면 됨.** 저장도 자동으로 된다. 
3. 현재 사용 중인 언어들은 이전에 `Project Settings > Localization`에서 등록했던 값들이다. 


### 구현
- `LocalizationManager`
```cs
private List<Locale> availableLocales = new();

private IEnumerator Start()
{
	// Localization 초기화 대기 (비동기)
	yield return LocalizationSettings.InitializationOperation;

	availableLocales = LocalizationSettings.AvailableLocales.Locales;
}

// 언어 설정
public void SetLocale(int index)
{
	if (index < 0 || index >= availableLocales.Count) return;

	// 현재 사용 중인 언어를 등록하고 저장함
	LocalizationSettings.SelectedLocale = availableLocales[index];
}

// 현재 언어 인덱스
public int GetCurrentLocaleIndex()
{
	return availableLocales.IndexOf(LocalizationSettings.SelectedLocale);
}
```
> 참고 : `IEnumerator`로 `Start`를 구성하면 유니티에서 자동으로 코루틴으로 처리해줌

#### TMP_Dropdown 관련
- 추가 이슈 : 드롭다운으로 나타나는 메뉴가 너무 작음

```
TMP_Dropdown
  └─ Template          ← 드롭다운 열렸을 때 나타나는 패널
       └─ Viewport
            └─ Content
                 └─ Item          ← 각 항목의 프리팹
                      ├─ Item Background
                      ├─ Item Checkmark
                      └─ Item Label    ← 텍스트
```
> - `Item`의 `Rect Transform`을 수정하면 개별 아이템의 서식 설정 가능
> - 텍스트 크기는 `Item Label`로 수정

- 좀 만져봤는데 계속 헷갈려서 원리부터 알고 들어감

- `TMP_Dropdown`
	1. `Template`을 통째로 복사, `Item`을 `options` 수만큼 복제함
	2. 각 `Item`에 텍스트  / 체크마크를 세팅함
	3. 닫히면 복사본 전체를 삭제함

- `Content`의 항목 배치 원리
	- 코드로 직접 `Item`의 위치를 계산해서 배치함. (`Vertical Layout Group`을 사용하지 않음)
	- **`Item`의 `RectTransform` 높이를 기준으로 순서대로 쌓음**

---
- `Template`의 높이 : 복사본에는 적용되지 않음
	- 레이아웃이 계속 이상하게 나와서 그거 처리하느라 시간이 좀 걸리는 중
	- 암만 봐도 스크립트 단위로 처리하지 않아도 될 것 같은 기초적인 문제로 보이는데, 다른 방법이 안 보이니까 그냥 스크립트 단위로 작업함

이 부분은 높이값을 만지면서 테스트해봤는데 `Item`들과 `Viewport`가 깔끔하게 맞아떨어지지 않는다. 내부적인 로직이라 따로 알 방법은 없어보임.(찾으라면 찾을 순 있겠지만..)

```cs
public class OthersSection : MonoBehaviour
{
    [Header("Language Markdown Configs")]
    [SerializeField] private TMP_Dropdown languageDropdown; 
    private const float ItemHeight = 60f;
    private const int MaxVisibleItems = 3;

    private IEnumerator Start()
    {
		// ..
		
        // 언어 드롭다운 버튼 구현
        OrganizeLanguageDropdown();
    }

    // 드롭다운을 펼쳤을 때 나타나는 레이아웃이 이상해서 직접 코드로 넣음
    private IEnumerator AdjustDropdownList()
    {
        yield return null; // Dropdown List 생성 대기

        // Dropdown List 탐지
        Transform dropdownList = languageDropdown.transform.Find("Dropdown List");
        if (dropdownList == null) yield break; 

        RectTransform dropdownListRect = dropdownList.GetComponent<RectTransform>();
        Transform content = dropdownList.Find("Viewport/Content");
        RectTransform contentRect = content.GetComponent<RectTransform>();

        // 요소들의 높이값 변경
        float targetHeight = ItemHeight * MaxVisibleItems;
        dropdownListRect.sizeDelta = new Vector2(dropdownListRect.sizeDelta.x, targetHeight);
        contentRect.sizeDelta = new Vector2(contentRect.sizeDelta.x, targetHeight);


        // 각 요소의 높이 조절

        // 사용 불가) item 말고도 RectTransform이 있는 모든 요소를 수집함
        // RectTransform[] itemRects = dropdownList.Find("Viewport/Content").GetComponentsInChildren<RectTransform>();

        List<RectTransform> activeItemRects = new List<RectTransform>();

        // Content의 바로 아래 자식들만을 순회함
        for (int i = 0; i < content.childCount; i++)
        {
            Transform child = content.GetChild(i);

            // 하이어라키 상에서 활성화되어 있는 요소만 리스트에 추가
            if (child.gameObject.activeInHierarchy)
            {
                activeItemRects.Add(child.GetComponent<RectTransform>());
            }
        }
        
        for (int i = activeItemRects.Count - 1; i >= 0; i--)
        {
            RectTransform itemRect = activeItemRects[i];

            // 아래에서부터 각 Item 높이의 절반만큼 올라가는 방식
            // 예시 : ItemHeight = 60이고 총 3개 있을 경우, 
            // 2번 인덱스의 Y 위치는 60 / 2 + 60 * (3 - 1 - 2) = 30
            // 이런 원리로 0번은 150, 1번은 90, 2번은 30의 높이를 갖는다.
            float positionY = (ItemHeight / 2) + ItemHeight * (activeItemRects.Count - 1 - i); 
            itemRect.anchoredPosition = new Vector2(itemRect.anchoredPosition.x, positionY);
        }
    }

    private void OrganizeLanguageDropdown()
    {
        List<Locale> availableLocales = LocalizationSettings.AvailableLocales.Locales;

        // 드롭다운 항목 구성
        languageDropdown.ClearOptions();
        languageDropdown.AddOptions(
            availableLocales.ConvertAll(l => l.LocaleName)
        );

        // 초기값 : 현재 언어
        languageDropdown.SetValueWithoutNotify(GameManagement.Instance.LocalizationManager.GetCurrentLocaleIndex());
        languageDropdown.onValueChanged.AddListener(GameManagement.Instance.LocalizationManager.SetLocale);

        // 드롭다운 오브젝트에 클릭 감지 컴포넌트 추가...는 되어 있음
        DropdownClickDetector clickDetector = languageDropdown.GetComponent<DropdownClickDetector>();
        clickDetector.onClicked += () => StartCoroutine(AdjustDropdownList());  
    }
}
```

- 마지막으로 나타나는 언어 이름을 네이티브로 설정하는 코드를 추가
```cs
private void OrganizeLanguageDropdown()
{
	languageDropdown.AddOptions(
		// 모국어 직접 표기
		availableLocales.ConvertAll(l =>
		{
			if (l.Identifier.CultureInfo != null)
			{
				// 한국어, English, 日本語
				return RemoveBrackets(l.Identifier.CultureInfo.NativeName);
			}
	
			// 없다면 기본 이름(코드와 언어명이 영어로 표시됨)
			return l.LocaleName;
		})
	);
}

private string RemoveBrackets(string input)
{
	if (string.IsNullOrEmpty(input)) return input;

	// 공백(0개 이상) + 여는 괄호 + 아무 글자 + 닫는 괄호
	string pattern = @"\s*\(.*?\)";

	return Regex.Replace(input, pattern, "").Trim();
}
```


#### 해결
- `StackTraceUtility.ExtractStackTrace()` : `Claude`에게 물어보니까 **같은 프레임에서 즉시 `false`가 나오는 상황이므로, `OnDisable`에 찍어서 어디서 호출됐는지 추적하는 게 가능**하다고 함
```cs

```

- `PopupBase.Awake()`에 문제가 있었다.
```cs
protected virtual void Awake()
{
	if (closeButton != null) closeButton.onClick.AddListener(Close);
	if (backgroundButton != null) backgroundButton.onClick.AddListener(Close);

	gameObject.SetActive(false);
}
```
>-  최초에 오브젝트가 비활성화되어 있기 때문에, 활성화되는 순간 Awake - OnEnable - Start 순으로 코드가 동작, 바로 비활성화된다는 것

저 코드를 넣었던 이유는 최초에 해당 팝업이 비활성화된 상태로 나타나게 하기 위함이었다. 대안이 있을까?

1. 인스펙터에서 오브젝트를 비활성화 - 가장 단순하지만 실수 가능성도 있음
2. `CanvasGroup`으로 숨김 - 최초에 활성화된 상태니까 `Update` 등은 계속 돌아감
3. **매니저에서 일괄 비활성화** - 다른 오브젝트에서 관리되는 개념, 단 매니저에 직접 등록할 필요가 있음

1번이 제일 심플하다. 근데 **내 경우는 `MainMenuPopupManager`에서 팝업들을 일괄적으로 관리하고 있기 때문에 3번을 바로 적용**할 수 있겠음.

스테이지 씬에서도 쓰는 팝업이다. 이 경우는 1번처럼 처리.
