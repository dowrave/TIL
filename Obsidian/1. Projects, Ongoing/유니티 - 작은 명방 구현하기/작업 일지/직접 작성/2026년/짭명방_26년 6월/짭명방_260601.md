## 이슈 수정

### 튜토리얼 : 다음 `TutorialData`로 넘어가지지 않는 현상 수정 및 재실행 시의 구현
- 아마 `StageId` 값을 고치면서 발생한 문제로 보임 - `String` 값을 직접 참조했고, 그 값이 변경되었기 때문

- 튜토리얼에 관한 메서드임에도 `StageManager`에 관련 코드가 있다. 수정 중.

#### 튜토리얼 중에 게임이 꺼졌을 경우에 대한 생각 좀
-  튜토리얼 중에 중단했음을 알리고, **튜토리얼을 다시 진행할 것인지에 대한 질문**을 함

- 그러면 경우는 크게 2가지임
1. `MainMenuScene`에서 계속 되는 튜토리얼(1, 3)
2. `StageScene`에서 계속되는 튜토리얼(2)

**1번의 경우는 단순히 해당 튜토리얼의 처음부터 다시 재생**되게끔 하면 되는데
2번을 어떻게 할지가 고민이었다.

- 일단은 이렇게 가보기로 함 : 1번째 튜토리얼에서 편성하는 4명의 오퍼레이터 배치까지는 자동으로 진행되게 하고, 스테이지 진입 버튼만 누르게끔 하면 되지 않을까?

- 그러면 `TutorialData`를 하나 더 만들어야 할 것 같다.

#### 구현
- 말로 일일이 정리하기 좀 복잡하다. 과거의 코드를 보니까 한눈에 이해하기 어렵게 되어 있음; ~~더럽다는 뜻이다.~~

- ~~`OnSceneLoaded`~~ `Start`에서 튜토리얼이 진행되고 있었는지 여부를 체크하고 계속할지 여부를 물어본 다음, 완료된 튜토리얼 인덱스를 체크해서 그 다음 튜토리얼부터 재시작되게 했음
- 스테이지 씬이 진행되다가 종료됐을 경우는 스테이지 씬으로 다시 안내할 필요가 있다. 그래서 스쿼드 편성은 자동화하고 유저는 단순히 편성 패널로 들어가서 스테이지 진입 버튼만 클릭하도록 **대체 `TutorialData`** 를 만들었다.
	- **진행 중인 튜토리얼들은 인덱스로 관리**하고 있기 때문에 대체 튜토리얼을 진행하고 **"현재 진행 중인 튜토리얼 = 0번 인덱스"** 로 해놓으면 0번 튜토리얼을 진행하는 것과 동일함.

- 이외에도 `PlayerDataManager`에 있어야할 것과 `TutorialManager`에 있어야 할 정보들이 섞인 듯 해서 수정

## 지식이 늘었다
### 필드 + 데이터 이중 관리 
- `현재 튜토리얼이 진행되고 있는가`를 관리하는 필드는 런타임의 필드와 데이터로 모두 저장하고 있었음
	- 필드의 경우 세터 프로퍼티를 이용해서 필드 값이 변할 때 데이터도 함께 저장되는 구성이면 ㅇㅋ
```cs
private bool _isTutorialRunning = false;
public bool IsTutorialRunning
{
	get => _isTutorialRunning;
	set
	{
		_isTutorialRunning = value;
		if (GameManagement.Instance.PlayerDataManager != null)
		{
			GameManagement.Instance.PlayerDataManager.SetIsTutorialRunning(value);
		}
	}
}

// PlayerDataManager.cs
public void SetIsTutorialRunning(bool status)
{
	playerData.isTutorialRunning = status;
	SavePlayerData();
}
```

이렇게 구현할 경우 문제가 될 수 있는 부분은 초기화다. 초기화는 자체적으로 어떤 값을 갖게 하는 게 아니라, `PlayerDataManager`에서 읽어오는 형태로만 이뤄져야 함.

```cs
private void Start()
{
	playerDataManager = GameManagement.Instance.PlayerDataManager;

	// 현재 튜토리얼이 진행 중인지부터 확인
	// 필드에 값을 직접 넣는다
	_isTutorialRunning = playerDataManager.GetIsTutorialRunning();

	// ...
}
```
