#### 로딩 화면의 텍스트 로컬라이징
- 이 부분은 로컬라이징이 안 되어 있었음.
```
스테이지 로딩중 : {progress * 100:F0}%
```
> - 기존 텍스트의 형태가 이랬는데, 스마트 스트링은 자체 포맷팅이 가능하다.
> - 연산은 지원하지 않으니까, 스크립트 단위에선 100을 곱한 값을 progress에 넣고, 테이블에는 {progress:F0}으로 넣어두면 됨.

```cs
// LoadingScreen.cs
public void UpdateProgress(float progress)
{
	loadingSlider.value = progress;
	if (progress >= 1f)
	{
		progressText.text = LocalizationSettings.StringDatabase.GetLocalizedString(
			LocalizationManager.UI.TableName,
			LocalizationManager.UI.LoadingStatusComplete
		);
	}
	else
	{
		progressText.text = LocalizationSettings.StringDatabase.GetLocalizedString(
			LocalizationManager.UI.TableName,
			LocalizationManager.UI.LoadingStatusInProgress,
			arguments : new { progress = progress * 100 }
		);
	}
}
```
> 로딩 진행 중, 로딩 완료 에 대한 키값은 각각 `UI_LOADING_STATUS_IN_PROGRESS`, `UI_LOADING_STATUS_COMPLETE`

### `SkillIconBox` : 비활성화 후에 `SelectedLocaleChanged`의 호출에 반응하는 현상
- `Initialize()`에서 이벤트 구독, 이벤트 해제는 `OnDisable`로 옮김.
- 그래도 해결이 안되네? 

#### 상황
1. 메인메뉴 씬에서는 **언어 전환에 맞춰서 잘 동작**함
2. **스테이지 씬에서만 해당 컴포넌트의 오류가 발생**함. **씬 전환 과정에서 `SelectedLocaleChanged`에 대한 구독 해제가 제대로 되지 않았다**는 의미임
3. 이 오류가 나타날 때가 있고 아닐 때가 있음. 오퍼레이터의 갯수?

```cs
private void OnDisable()
{
	ResetSkillIcon();
	LocalizationSettings.SelectedLocaleChanged -= OnLocaleChanged;
	// Logger.Log("OnLocaleChanged 해제됨");
}

void OnDestroy()
{
	button.onClick.RemoveAllListeners();
	LocalizationSettings.SelectedLocaleChanged -= OnLocaleChanged;
}
```
> - `OnDisable`에만 있었는데, `OnDestroy()`에도 추가해봄
> - 근데 `OnDisable`은 `OnDestroy` 이전에 호출되는 개념이기도 함;

#### 해결
- `SkillIconBox.Intialize()`에 언어 전환 이벤트를 구독하는 부분이 있음
```cs
    public void Initialize(OperatorSkill skillData, bool showDurationBox = false, bool showSkillName = false)
    {
	    // ...
	    
	    LocalizationSettings.SelectedLocaleChanged += OnLocaleChanged;
    }
```
> - `SkillIconBox`의 초기화는 **이 컴포넌트의 비활성화 없이 다시 이뤄질 수 있었다.** 예를 들어서 한 오퍼레이터를 클릭한 상태에서 다른 오퍼레이터를 클릭했을 때, 상태 초기화 없이 바로 스킬 아이콘 박스의 초기화가 다시 실행됐고, 그래서 **같은 이벤트에 구독이 여러 번 들어가게 되었던 것**

그래서 저 이벤트를 등록하는 부분을 `OnEnable`로 옮겨서 해결했다.
```cs
private void OnEnable()
{
	ResetSkillIcon();
	LocalizationSettings.SelectedLocaleChanged += OnLocaleChanged;
}
```

`OnEnable`을 쓰고 싶지 않다면, `Initialize`에서 구독 해제를 한 번 하고 등록하는 패턴도 가능.


- 이 경우는 데이터가 오는지 여부에 관계 없이 언어에 따른 데이터 전환 메서드가 실행됨을 감안하면 **`Start / OnDestroy` 쌍도 괜찮아 보임**
	- 실제 변경 여부는 **사용하는 메서드 안에서 특정 데이터의 유무에 따라 실행을 결정하는 패턴**이면 되니까.
	- 실제로 옮겼다.

## 지식이 늘었다

### 클래스에서 사용하는 구조체의 구현 위치
```cs
// 1. 클래스 밖에서 구현하기
public struct WaveLayerData { ... }
public class WaveLines : MaskableGraphic { ... }

// 2. 클래스 내에 중첩해서 구현하기
public class WaveLines : MaskableGraphic
{
    public struct WaveLayerData { ... }  // 클래스 안
}

// 3. 별도 파일로 분리하기
// WaveLayerData.cs
public struct WaveLayerData { ... }
```

1. 클래스 밖에서 구현하기
	- 같은 파일에 있지만 중첩되지 않음
	- `WaveLayerData`가 `WaveLines` 이외의 곳에서도 독립적인 의미를 가질 수 있을 때 적합
	- 그러나 같은 파일에 있다는 것 자체가 둘의 관련 있음을 암시
	- 단점 : 파일명과 파일 내용이 완전히 일치하지 않게 됨

2. 클래스 내에서 중첩 구현
	- **`WaveLayerData`는 `WaveLines` 이외의 곳에서는 의미가 없음**을 구조적으로 표현
	- 외부에서 `WaveLines.WaveLayerData`로 접근하므로 소속이 명확
	- 단점 : 외부 접근이 장황해짐. 사용하는 곳이 많아지면 다르게 구현할 필요성 O

3. 별도 파일로 분리
	- 여러 클래스가 동등하게 소비하는 경우
	- 직렬화 / 저장이 필요한 경우
	- 규모가 커서 파일 하나에 두기 부담스러운 경우

### 이벤트 구독 / 해제 패턴
- 일반적으로 **`Start`와 `OnDestroy`** 가 한 쌍, **`OnEnable`과 `OnDisable`** 이 한 쌍이다.
- 그런데 `Initialize`와 `OnDisable`의 쌍을 맞추고 싶은 경우도 있을 수 있음
	- 이 경우 `Initialize`가 다시 동작할 때, `OnDisable`을 거친 다음(혹은 상태 초기화가 되었는지 등등)에 동작이 이뤄지는지를 봐야 함
- 이것저것 고려해야 하니 복잡해지는데, 이 경우 `Initialize`에서는 한 번 구독 해제를 한 다음 다시 구독하는 패턴을 권장한다.
```cs
LanguageManager.OnLanguageChanged -= UpdateText;
LanguageManager.OnLanguageChanged += UpdateText;
```

- 참고 : 만약 **같은 이벤트에 같은 컴포넌트가 구독을 여러 번 했다면, 해제도 같은 횟수로 이뤄져야 한다.** 안 그러면 **Destroy된 컴포넌트가 이벤트 구독을 참조하고 있다는 오류**가 발생함
