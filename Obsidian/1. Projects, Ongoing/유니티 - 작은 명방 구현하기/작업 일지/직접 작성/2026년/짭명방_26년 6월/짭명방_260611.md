## Skill Localization 적용하기
### Skill Desc(ription)에 관해
- 스킬마다 필요로 하는 데이터가 다름

>[!question]
>- 스킬 설명에 들어가는 파라미터 이름, 즉 변수 이름은 통일해서 쓰나? 스킬마다 구분해서 쓰나?
- **의미 있는 변수 이름을 사용하는 걸 권장**
	- `{value}, {value2}`같은 표현을 쓰는 경우, 값이 여러 개가 되면 헷갈림 + 문맥 파악도 어려워짐

아래는 관련 예시이므로 참고

| 상황      | 변수명 예시                              |
| --------- | ---------------------------------------- |
| 피해량    | `{damage}`, `{minDamage}`, `{maxDamage}` |
| 쿨타임    | `{cooldown}`                             |
| 지속시간  | `{duration}`                             |
| 범위      | `{range}`, `{radius}`                    |
| 스택/횟수 | `{stackCount}`, `{hitCount}`             |
| 배율/비율 | `{multiplier}`, `{critChance}`           |

### 전체적인 구조

- 공통 부모 클래스
```cs
// 부모 클래스
public string GetDesc()
{
	var localizedString = new LocalizedString(LocalizationManager.Skills.TableName, DescKey);
	var args = GetDescArgs();
	if (args != null)
	{
		localizedString.Arguments = args;
	}

	return localizedString.GetLocalizedString();
}

protected object[] GetDescArgs()
{
	var args = BuildDescArgs();
	return new object[] { args };
}

// 실제 Arguments을 구성하는 메서드
protected virtual Dictionary<string, object> BuildDescArgs() => null;
```

- 자식 & 손자 클래스
```cs
// 자식 - StatModifierSkill.cs
protected override Dictionary<string, object> BuildDescArgs() => new Dictionary<string, object>
{
	{"duration", duration},
	{"healthModifier", modifiers.healthModifier * 100},
	{"attackPowerModifier", modifiers.attackPowerModifier * 100},
	{"attackSpeedModifier", modifiers.attackSpeedModifier * 100},
	{"defenseModifier", modifiers.defenseModifier * 100},
	{"magicResistanceModifier", modifiers.magicResistanceModifier * 100},
	{"attackType", modifiers.attackType},
};

// 손자 - StatModifierWithOnHitSkill.cs
protected override Dictionary<string, object> BuildDescArgs()
{
	// StatModifierSkill의 구조를 가져옴
	var args = base.BuildDescArgs();

	args["stunDuration"] = stunDuration;
	args["stunChance"] = stunChance * 100;

	return args;
}
```

> - 생각만 해둘 것 
> - 1. `GetDescArgs()`의 호출이 빈번할 경우 캐싱해두는 게 좋다.
> 	- 지금 방식은 호출될 때마다 딕셔너리를 새로 만들고 있기 때문.
> 	- 빈번하지 않으니 넘어감.
> - 2. **스킬의 레벨업이 있다면, 런타임 중에 실제로 갖고 있는 값들에 대한 스킬 클래스도 별도로 구현해야 한다**. 여기선 구현하지 않았으니까 넘어감. 
> 	- 내 프로젝트의 `Operator`로 치면, `OwnedOperator`은 레벨 정보만 저장되는 개념이고, 런타임 중에 레벨 증가에 따른 스탯 상승량이 반영되는 방식임. 이거랑 비슷해 보인다.
> - 이런 캐싱해두는 값들은 SO에 저장하는 건 좋지 않다 : 런타임 상태를 보관하는 곳은 아님
### ArcaneField
>[!question]
>- 변수 설정은 0 ~ 1 사이의 `float` 값인데, 실제 반영은 `%` 단위라면?
- **변환은 코드 단위**에서 하면 되고, **어떤 변수를 사용하는가를 테이블에 표시하는 건 그대로 변수 이름을 사용**하면 됨
	- 예를 들어서 내부에 저장되는 변수 이름이 `tickDamageRatio = 0.5f`라고 해서, `stringTable`에서 `tickDamageRatioPercent`라고 **변수 이름을 바꿔서 넣을 필요는 없다**는 것.

- 이런 식으로 넣어주면 된다.
```cs
new Dictionary<string, object>
{
	{ "duration",        _duration             },
	{ "damageInterval",  _damageInterval        },
	{ "tickDamageRatio", _tickDamageRatio * 100 }, // 변환을 여기서
	{ "slowAmount",      _slowAmount * 100      }  // 변수명은 표시 기준
}
```

#### AttackType 관련
- 상태에 따라 값이 달라져야 하고, 그게 또 언어마다 달라져야 함
- 그렇기에 `modifiers.attackType` 보다는 공격 타입에 해당하는 키값을 가져와야 함
- 아직 공격 타입에 해당하는 키값을 구현하진 않았기 때문에, 그것부터 시작

- 쉬고 왔다. 계속함.
1. `Common` `StringTable`을 생성
	- `Common` : 여러 테이블이나 시스템에서 사용될 거라고 예상되는 경우
	- 처음부터 `Common`을 고려할 필요는 없다. 어떤 곳에서 쓰이다가 다른 곳에서도 쓰일 것 같으면 그때 옮기면 됨

2. `LocalizationManager`에서 연결
```cs
    public static class Common
{
	public const string TableName = "Common";
	
	public const string attackTypePhysical = "COMMON_ATTACK_TYPE_PHYSICAL";
	public const string attackTypeMagical = "COMMON_ATTACK_TYPE_MAGICAL";
	public const string attackTypeTrue = "COMMON_ATTACK_TYPE_TRUE";
}

public static string GetAttackTypeName(AttackType type)
{
	var key = type switch
	{
		AttackType.None => string.Empty, // 바꾸지 않는 경우가 있어서 오류를 발생시키지 않도록 설정
		AttackType.Physical => Common.attackTypePhysical,
		AttackType.Magical => Common.attackTypeMagical,
		AttackType.True => Common.attackTypeTrue,
		_ => 
		{
			#if UNITY_EDITOR 
			Logger.LogWarning("[LocalizationManager] 처리되지 않은 AttackType : {Type})");
			#endif
			return string.Empty;
		}
	};

	return new LocalizedString(Common.TableName, key).GetLocalizedString();
}
```

3. 사용
```cs
protected override Dictionary<string, object> BuildDescArgs() => new Dictionary<string, object>
{
	// ...
	{"attackType", LocalizationManager.GetAttackTypeName(modifiers.attackType)},
}
```

```
{stunDuration} {stunDuration:plural:second|seconds}
```

### 1. Switch 문 
```cs
var key = type switch
{
	AttackType.None => string.Empty, // 바꾸지 않는 경우가 있어서 오류를 발생시키지 않도록 설정
	AttackType.Physical => Common.attackTypePhysical,
	AttackType.Magical => Common.attackTypeMagical,
	AttackType.True => Common.attackTypeTrue,
	_ => 
	{
		#if UNITY_EDITOR 
		Logger.LogWarning("[LocalizationManager] 처리되지 않은 AttackType : {Type})");
		#endif
		return string.Empty;
	}
};
```
> 여기 `switch`문 관련
> - `_ => {}`문이 없을 때에는 경고문구가 나옴 

```
The switch expression does not handle some values of its input type (it is not exhaustive) involving an unnamed enum value. For example, the pattern '(AttackType)4' is not covered.
```
- `C#`의 스위치 문은 **모든 경우가 처리된다는 보장**이 필요함
- `_ => {}` 문이 없다면, `switch`문은 `enum` 타입에 해당하는 4가지 경우만이 들어 있고, 그것들을 제외한 경우는 아예 빠져있는 셈이 된다.
- `_ => {}`문은 기본 케이스임. 

```cs
// Switch 문(Statement) - default 사용
switch (type)
{
	case:
		return a;
	default:
		return string.Empty;
}

// Switch 표현식(Expression) - _ 사용
var key = type switch 
{
	AttackType.Physical => Common.attackTypePhysical,
	_ => string.Empty;
}
```

### 2. string.Empty의 string이 뭐임?
- **`System.string`의 `별칭(Alias)`** 이다.
- **`string.Empty == String.Empty == ""`**

- `string`은 C# 언어에서 제공하는 소문자 키워드이고, `String`은 `.NET`의 실제 클래스 이름이다.
```cs
string  ==  System.String
int     ==  System.Int32
bool    ==  System.Boolean
float   ==  System.Single
```

- 코드에서는 `string`으로 쓰는 게 관례이다. `String`은 `using System`이 필요하고 사용하기에도 더 번거롭기 떄문.
