## 역할 설명 위치
- 만들어놓고 보니까 역할 설명하는 곳이 하나도 없음
- 이거도 2가지 아이디어가 떠오름

1. `SquadEditPanel`이나 `OperatorInventoryPanel`에 직군의 역할을 간단하게 설명한 패널 구현하기
2. 각 오퍼레이터의 세부 정보에 태그 형식으로 역할 추가하기

- 1번으로 진행함. `TopArea`에서 가장 오른쪽에 나타나게 하면 될 듯
- 이 과정에서 `SquadEditPanel`의 오른쪽에 있는 `SquadEditButtonsContainer`는 별도의 항목으로 분리
	- 기존에는 `TopArea` 내부의 컴포넌트였는데, `SquadEditPanel`의 하위 오브젝트로 이동시켰음. `TopArea`의 변동 내역에 따라 위치에 영향을 받기 때문에 수정함

- 옮기는 과정에서 두 `ExpandableButton`의 상호작용 이슈(한 버튼이 클릭되면 다른 버튼은 접혀야 함)가 있어서 그거 수정 중
	- 이거 `UnityEvent`로 구현했어서 보이지 않았다. 인스펙터에 연결되어 있어서..
	- 프로젝트의 전체적인 구현과 비슷하게 `Action`을 이용해서 수정하겠음

```cs
// ExpandableButton.cs

// 기존 구현
// [Header("Events")]
// public UnityEvent firstClickEvent; // 1번째 클릭 시의 이벤트
// public UnityEvent secondClickEvent; // 2번째 클릭 시의 이벤트

// 각각 n번째 클릭됐을 때 이벤트가 발생함.
public Action<ExpandableButton> onFirstClicked;
public Action<ExpandableButton> onSecondClicked;

// SquadEditPanel.cs

// expandableButton을 클릭했을 때 다른 버튼은 확장 상태를 해제함
public void SetTheOtherExpandableButton(ExpandableButton clickedButton)
{
	if (clickedButton == squadBulkEditButton)
	{
		resetSquadButton.SetIsExpanded(false);
	}
	else
	{
		squadBulkEditButton.SetIsExpanded(false);
	}
}    
```

- 밥 먹으러 가기 전 기록
	- 2번째 클릭 이벤트까지 연결해야 함
	- 각 기능은 `MainMenuManager.HandleSquadBulkEditButton`, `MainMenuManager.HandleResetSquadButton()`에 있음
	- 처리 완료
