## 배속시 사운드 처리
- 순간적으로 나는 소리들은 크게 상관이 없는데, 몇 초 동안 나야 하는 소리 같은 데서 어색함이 보임
	- 예를 들어서 파티클이 떨어지는 시간은 1배속 기준으로 3초임. 2배속이면 1.5초인데, 소리는 3초를 기준으로 만들었음.
	- 이 경우 2배속 시에는 어떻게 처리해야 할까?
### 방법
1. 배속에 맞춰서 `AudioSource.pitch`를 조절하는 방법
	- 길이도 자동으로 조절됨
```cs
audioSource.pitch = Time.timeScale;
```

2. 피치를 고정하고, 수동으로 중단하는 방법
```cs
float adjustedLength = clipLength / Time.timeScale;
audioSource.Play();
Invoke(nameof(StopSFX), adjustedLength);
```
> `Invoke` : 유니티 내장 함수, 지정 시간 후에 메서드를 호출

> 대충 생각나는 게임들을 얘기해보면
> - 리버스: 1999는 위 방식 
> - 명방은 아래 방식

> 나도 아래 방식으로 테스트해봄

- **위 방식은 `StopSFX`가 파라미터를 받으면 사용 불가능하다.** 이떄는 코루틴을 써야 함.
	- `Source`를 여러 개 관리할 경우는 코루틴밖에 없다고 봐야 할 듯...

```cs
// 현재 게임 속도에 따른 클립 재생 시간
float gameSpeed = StageManager.Instance.IsSpeedUp ? 2f : 1f;
float adjustedLength = clip.length / gameSpeed;

// 재생 속도에 따른 멈추는 시간 설정
StartCoroutine(StopSourceAfterDelay(availableSource, adjustedLength));
```
`StageScene`일 경우엔 이 코드가 제대로 작동하지만 `MainMenuScene`일 때는 아니라서 관련 처리 추가
### 테스트 후
- 애매하게 자르는 게 더 어색한 느낌이 있음 - 소리가 뚝 끊김
- 그러면 이런 방향으로 가는 게 나을 듯 : **폭발 소리가 나는 시점에 떨어지는 소리를 멈추게 하기**

```cs
// 스킬 진행 중 로직에 추가
AudioSource particleDropSource = null;

// 낙하 SFX 재생 - 재생 중인 오디오 소스를 얻음(SoundManager의 내용도 수정)
if (_skill.ParticleDropSFX != null)
{
	particleDropSource = GameManagement.Instance.Sound.PlaySpecialSFX(_skill.ParticleDropSFX);
}

// 폭발 시점 - 오디오 소스 실행 종료
if (particleDropSource != null)
	GameManagement.Instance.Sound.StopAudioSource(particleDropSource);
```

- **이렇게 하는 게 더 자연스러움**

### 보스 스킬 글로벌 쿨다운 적용 안되는 현상
- 2가지 스킬 중, 하나를 썼다면 글로벌 쿨다운 후에 다른 스킬을 써야 하는데 곧바로 스킬이 나가는 현상이 있음
```cs
    private float _currentGlobalCooldown; // 공통 쿨다운 : 스킬이 2개 있는데, 두 스킬을 연속적으로 쓰는 걸 방지하기 위함
    private float currentGlobalCooldown;
```

- 한 필드를 이중으로 관리하고 있었음. 언더스코어가 들어간 버전으로 통일.
- 근데 그래도 스킬이 나간다.

- 일단 이중으로 처리되는 구조 수정
```cs
// Update에 들어가는 템플릿 메서드. 스킬 처리를 추가했다. 
protected override void OnUpdateAction()
{
	bool skillUsed = _skillController.OnUpdate();
	if (skillUsed) return;  

	base.OnUpdateAction();
}

// protected override void UpdateAllCooldowns()
// {
//     base.UpdateAllCooldowns();
//     _skillController.UpdateAllCooldowns();
// }
```
> `base.OnUpdateAction` 내부의 `_skillController.OnUpdate()`에 구현하면 될 요소를 굳이 `UpdateAllCooldowns`라는 별도의 메서드에 구현하고 있었다.
