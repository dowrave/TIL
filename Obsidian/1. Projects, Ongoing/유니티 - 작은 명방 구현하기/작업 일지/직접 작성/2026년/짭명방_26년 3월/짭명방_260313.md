## 기타 이슈 수정

### EnemyBoss
- 컨트롤러 분리 이후로 `EnemyBoss`를 테스트한 적이 없는 듯
- 테스트 환경 만들고 쫙 진행함

1. EnemyBoss가 움직이지 않는 문제 -> 완료
2. 근거리 스킬은 쓰는데 원거리 스킬을 쓰지 않는 문제 -> 진행 중
	- 이게 길어지고 있다. 일단 `mainTarget`의 대상은 잡히는데, 스킬의 시전 조건인 `CanActivate`에서 문제가 있는 듯?
	- `OperatorsInSkillRange` 로직을 봐야겠다.
```cs
public override bool CanActivate(EnemyBoss caster) => caster.OperatorsInSkillRange.Count > 0;
```
> `OperatorsInSkillRange`에 들어가는 조건이 콜라이더에 닿느냐 여부이기 때문에, 실질적인 문제는 콜라이더의 범위가 제대로 설정되지 않았다 <--에 있다고 보임

- 범위 추가하고 테스트했는데도 안됨
	- 여전히 `CanActivate`가 `false`로 나오고 있음

- 그러면 `OnTriggerEnter`가 멀쩡하게 동작하고 있는가를 체크 
	- 컴포넌트가 빠졌거나 비활성화된 건 아님
	- 잘 동작함(Update에 의해 동작하는 다른 디버깅 로그랑 겹치면 잘 보이지 않았음)

- 스킬 범위 내의 `Operator` 들에도 잘 추가됨

#### 원인 발견
- 부모 컨테이너 `EnemyBoss`에 있는 `OperatorsInSkillRange`라는 프로퍼티의 접근 대상이 `_skill.OperatorsInSkillRange`가 되어야 하는데 그냥 자기 자신의 `_operatorsInSkillRange`로 설정되어 있었음. 
- 컨트롤러로 분리하는 과정에서 정리하지 않고 지나가서 발생한 이슈로 보인다. 

#### 이슈 외의 추가 처리
- 대상이 죽었을 때 `skillRange`에서 제거하는 로직 필요
```cs
public void OnDeployableDied(UnitEntity unit)
{
	if (unit is Operator op)
	{
		_operatorsInSkillRange.Remove(op);
		op.OnDisabled -= OnDeployableDied;
	}
} 
```
> 처음엔 이렇게 작업했는데 그냥 `DeployableUnitEntity` 클래스 전체에 대해 스태틱 이벤트 `UnDeployed` 시점에 처리해도 될 것 같아서 그렇게 작업함

- 원거리 스킬 시전 중에는 멈춰야 하는데 움직이고 있음
	- `MovementController`에서 `_isWaiting` 필드는 있는데 실제로 사용을 안 하고 있었음

- 일단 중지, 체크할 거 2개
1. 위에서 `_operatorsInSkillRange`에서 정상적으로 제거되고 있는가
2. 폭발 후 공격 범위 VFX가 타일에 나타나지 않는 현상

#### operatorsInSkillRange 관련
```cs
Operator.OnUndeployed += OnOperatorUndeployed;
```
> - 여기서 `OnUndeployed` 이벤트는 `Operator`의 상위 클래스 `DeployableUnitEntity`의 이벤트임
```cs
public void OnTargetEnteredSkillRange(DeployableUnitEntity target)
{
	if (target is Operator op && op.IsDeployed && !_operatorsInSkillRange.Contains(op))
	{
		_operatorsInSkillRange.Add(op);
		Logger.LogFieldStatus(_operatorsInSkillRange.Count);
	}
}

public void OnTargetExitedSkillRange(DeployableUnitEntity target)
{
	if (target is Operator op && op.IsDeployed && _operatorsInSkillRange.Contains(op))
	{
		_operatorsInSkillRange.Remove(op);
		Logger.LogFieldStatus(_operatorsInSkillRange.Count);
	}
}

#endregion

// 배치 판정 해제 (=퇴각 | 사망) 시 스킬 범위에서 제외
public void OnOperatorUndeployed(DeployableUnitEntity deployable)
{
	if (deployable is Operator op && _operatorsInSkillRange.Contains(op))
	{
		_operatorsInSkillRange.Remove(op);
	}
} 
```
> 이런 식으로 진입, 이탈, 퇴각/사망 처리 시에 동작시켜봤다. 가장 마지막 메서드의 경우 `Operator`가 사망했을 때 동작하지 않는 이슈가 있음.

- 수정
```cs
// 스태틱 이벤트가 선언된 클래스에 대해서 등록
// 형 변환은 메서드 내부에서 처리
DeployableUnitEntity.OnUndeployed += OnOperatorUndeployed;
```

> 그래도 해결이 안 되네? 
- 아예 `UnitEntity.OnDeathStarted`로 뺐음. 유닛의 사망 로직 처리 시에 동작하기 때문
```cs
public void OnOperatorUndeployed(UnitEntity unit)
{
	if (unit is Operator op && _operatorsInSkillRange.Contains(op))
	{
		_operatorsInSkillRange.Remove(op);
		op.OnDeathStarted -= OnOperatorUndeployed;
	}
} 
```
> 이 이벤트는 스태틱 기반이 아니라 인스턴스 기반이므로 처리도 살짝 바뀜 
> 	- 클래스 전체에 대한 처리가 아니라 유닛이 등록되고 해제되는 시점에 이벤트 구독/해제를 처리하는 방식으로 변경

- 그래도 안됨 ㅡㅡ
	- 사망 시 이벤트 발생은 정상적으로 처리되고 있음
	- 이벤트를 받는 쪽에 문제가 있는 듯...
	- **이 아니다.** 마지막 메서드(사망 시 처리)에 범위 내의 적 카운트하는 로깅이 안 되어 있었음 ㅋㅋㅋㅋㅋ
	- 로깅해보니까 사망 이벤트 잘 받고 리스트 처리도 잘 됨. 끝
