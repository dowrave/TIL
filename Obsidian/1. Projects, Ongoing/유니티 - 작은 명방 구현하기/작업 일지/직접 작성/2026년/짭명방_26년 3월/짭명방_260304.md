- 이전에 리팩토링하면서 세부 요소들은 전부 컨트롤러로 빼뒀는데, 이 과정에서 온히트 SP 회복 로직이 누락된 것으로 보임
- 이참에 구조 자체도 살짝 손보기로 함

#### 수정
1. `OperatorSkill`의 SP 회복 타입을 `enum`으로 추가
```cs
[Tooltip("SP 회복 로직")]
public OperatorSPRecoveryType spRecoveryType;

// 별도 파일
public enum OperatorSPRecoveryType
{
    None,
    Auto, // 자동 회복(시간에 따라 올라감)
    OnHit // 공격 시 회복
}
```
> 모든 `OperatorSkill` SO 필드 수정

2. `OpAttackController.cs`의 공격 후 스킬 회복 로직 수정

// 공격 후 SP 회복 로직
if (_owner.CurrentSkill.spRecoveryType == OperatorSPRecoveryType.OnHitRecover &&  // 스킬이 공격 시 SP 회복
	!_owner.IsSkillOn && // 스킬 off 상태
	spBeforeAttack != _owner.MaxSP) // SP가 최대값이 아닐 때

### 바리케이드 배치 시 대각선 이동 로직 정상 작동 X
```
ㅁㅁ
ㅁX

ㅁ : 타일 / X : 지나갈 수 없음
```
> - 여기서 `X`가 언덕 / Empty 일 때는 잘 동작함
> - 그런데 `Barricade`인 경우에는 동작하지 않음
- 즉 `ㅁ`만을 따라서 이동해야 하는데, `X`가 있음에도 대각선 이동을 한다, 그게 게임 상에서 어색해보이는 면이 있다 - 타일과 `Enemy`가 겹치기 때문

#### 원인
```cs
// PathfindingManager.cs

// 대각선 이동
if (x != 0 && y != 0)
{
	// 대각선 이동 시, 양쪽 타일 모두 걸을 수 있어야 함
	Tile? SideA = currentMap.GetTile(tile.GridPosition.x + x, tile.GridPosition.y);
	Tile? SideB = currentMap.GetTile(tile.GridPosition.x, tile.GridPosition.y + y);

	if (SideA != null && SideB != null && SideA.TileData.IsWalkable && SideB.TileData.IsWalkable)
	{
		neighbors.Add(neighbor);
	}
}
```
- 경로 계산 로직 중 일부
- 타일의 타일 데이터만을 체크하고 있기 때문에 이 현상이 발생하고 있음
- 타일 데이터가 아니라, 타일 자체가 이동 가능한지 여부를 검사하면 된다. `Barricade`는 `Tile.IsWalkable`을 바꾸기 때문임.

#### 수정
- 일단 `TileData` 자체에도 `IsWalkable`이 있음
	- 기존 게임을 따라하는 과정에서 각 타일의 속성을 정의했기 때문임
- 그러면 `Tile.IsWalkable` 필드는 크게 아래 요소들이 모두 반영되어야 함
	- `TileData` 자체에 대한 판정
	- 인게임에서 변하는 판정

- `Tile.cs`
```cs
public void SetIsWalkable(bool isWalkable)
{
	// 타일 데이터 자체가 이동 불가능한 경우에는 수정되지 않음
	if (!TileData.IsWalkable) return;

	_isWalkable = isWalkable;
}
```

- `PathfindingManager.cs`
```cs
if (neighbor != null && neighbor.IsWalkable)
{
	// 대각선 이동
	if (x != 0 && y != 0)
	{
		// 대각선 이동 시, 양쪽 타일 모두 걸을 수 있어야 함
		Tile? SideA = currentMap.GetTile(tile.GridPosition.x + x, tile.GridPosition.y);
		Tile? SideB = currentMap.GetTile(tile.GridPosition.x, tile.GridPosition.y + y);

		if (SideA != null && SideB != null && SideA.IsWalkable && SideB.IsWalkable)
		{
			neighbors.Add(neighbor);
		}
	}
	// 직선 이동
	else
	{
		neighbors.Add(neighbor);
	}
}
```

## 효과음 구현
### SlashSkill
#### 초반 계획
- 얘는 소리를 좀 합성해야 할 듯
- 스킬
	- 1번째 타격은 즉발
	- 0.3초 대기
	- 이후 3연타가 들어가며, 연타 간의 간격은 0.15초

#### 방향 선회
- 처음엔 스킬 1개에 1개의 효과음을 그대로 대응하는 방식을 생각했는데
- 1번 찌르고 3번 벤다는 느낌이면 그냥 **개별 효과에 개별 SFX를 넣는 게 훨씬 좋아보임**
- 스킬 사양에 변동이 생길 때에 추가적인 작업을 할 필요가 없기 때문임
	- 예를 들어 1번째 타격 이전에 선 딜레이가 있다든가, 간격의 딜레이가 변화한다든가. 
- **스킬에 그대로 맞추는 경우보다는 따로따로 작업해둔 방식이 추가 작업이 필요하지 않음**

- 따라서 이 스킬을 위한 소리는 2가지로 압축할 수 있음
	- 찌르는 소리 하나
	- 베는 소리 하나

### 참고) 바리에이션 관련
