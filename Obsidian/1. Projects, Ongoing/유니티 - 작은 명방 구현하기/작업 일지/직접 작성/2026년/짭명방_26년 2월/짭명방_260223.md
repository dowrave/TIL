## 스킬 - 버프 연계 관련
- 현재 상황 및 필요 사항
- 스킬을 통해 버프가 걸리는 방식을 사용 중
- 스킬이 해제될 때, 함께 해제되게 하고 싶은 버프가 있음
	- 지속시간이 있는 경우는 `duration`값을 함께 전달하는 방식으로 싱크를 맞출 수 있음
	- **지속시간이 무한인 경우가 문제**인데, 버프에서 스킬이 해제되었음을 알 수 있어야 함
		- 이렇게 간다면 위의 `duration`을 전달하는 방식도 스킬에 연동되게끔 하는 식으로 바꿀 수 있음
- 한편, 스킬의 지속시간에 관계없이 유지/해제되는 버프들도 존재함. 이 버프들은 자체적인 지속시간을 가지며, 스킬을 추적할 필요는 없음.

>[!note]
>1. 버프에서 스킬의 상태를 추적하는 로직을 구현해야 함
>2. 추적 여부는 버프마다 달라질 수 있음

> 구현은 직접 해 봄. 

### StatModifierSkill(SO)
```cs
protected StatModificationBuff statBuffInstance;
```
- 여기서 이런 필드를 갖고 있는 게 문제가 될 수 있다. SO는 애셋으로 존재하기 때문에 저 인스턴스를 여러 오퍼레이터가 공유한다면 버그가 발생할 수 있다. 에디터에서도 오류 가능성이 있음.
- 물론 `null`처리로 초기화를 하고 있지만, 런타임에만 동작하는 요소는 런타임 컴포넌트에서 관리하는 게 좋다.

### StatModificationBuff
```cs
public class StatModificationBuff : Buff
{
	// (부모)Buff의 필드
	public OperatorSkill SourceSkill { get; protected set; }

    public StatModificationBuff(float duration, StatModifierSkill.StatModifiers mods)
    {
        this.duration = duration;
        buffName = "Stat Boost";
        modifiers = mods;
    }
}
```
> - 이 프로젝트에서는 `SourceSkill`이 `OperatorSkill`이어도 크게 상관은 없다. `Enemy`에게 `Buff` 관련 시스템이 적용되지 않기 때문이다.
> - 하지만 보통 이런 시스템은 모든 유닛에 공통적으로 적용되는 개념이니까 수정할 필요가 있어보인다.

### 제네릭 공변성
- `OperatorSkill`의 조상은 `UnitSkill<TCaster> where TCaster : UnitEntity`이다. 
- 그렇다고 저걸 `UnitSkill<UnitEntity>`로 변경할 수는 없다. 

```cs
// UnitSkill<UnitEntity>로 정의할 수는 있는데
public CannotAttackBuff(float duration, UnitSkill<UnitEntity> sourceSkill)
{
	this.buffName = "Cannot Attack";
	SourceSkill = sourceSkill;
	this.duration = duration;
}

public class ArcaneFieldSkill : ActiveSkill
{
	// 여기서 ArcaneFieldSkill은 UnitEntity<Operator>을 상속받는 요소이며, Operator는 UnitEntity의 자식 클래스
	// 그러나 이렇게 사용할 수 없음 
	CannotAttackBuff _cannotAttackBuff = new CannotAttackBuff(duration, this);
}
```
- 왜 사용할 수 없는지는 [[Csharp - 제네릭 공변성(Covariance)]] 참조. 암묵적 형변환이 불가능하기 떄문임.

- 해결법은 간단함 - `UnitSkill<Tcaster>` 위에 비제네릭 클래스 `UnitSkillBase: ScriptableObject`을 만들고, `UnitSkill<Tcaster>: UnitSkillBase where Tcaster: UnitEntity`로 구현하면 됨

### 처리 과정
1. `Buff`에는 `UnitSkillBase SourceSkill`이 있음. 자신을 생성한 스킬이며, `null`이어도 무방함.
2. (일단은) `ActiveSkill.OnEnd()`에 아래처럼 구현함
```cs
public override void OnEnd(Operator caster)
{
	// 이 ActiveSkill과 함께 해제되어야 하는 요소들이 들어감
	caster.RemoveBuffFromSourceSkill(this);
}
```
> - 이전에 `ArcaneFieldSkill`을 구현할 때, 캐스터의 공격 불가 기능을 구현하기 위해 추가했던 요소였다. 지속시간이 끝나면 공격 해제 버프가 취소되는 방식. 
> - `OnEnd`는 `Operator.SkillController`에 있으며, 어떤 스킬이든 종료되는 시점에 실행되는 메서드이다. 

3. `BaseSkill`에 영향을 받아야 하는 `Buff`라면 `UnitSkillBase`를 받도록 하고, 아니라면 구현하지 않는다.
	- 이거는 다양한 상황이 생길 수 있음. 똑같은 종류의 `Buff`이더라도 스킬을 따라갈 수도 있고 아닐 수도 있음
	- 그래서 생성자에 할당하는 `UnitSkillBase`는 `nullable` 필드로 구현하는 게 맞겠지만, 간결함을 위해 버프마다 필요하거나 필요하지 않다고 정해놓고 들어가겠음
```cs
public StatModificationBuff(float duration, StatModifierSkill.StatModifiers mods, UnitSkillBase sourceSkill)
{
	this.duration = duration;
	buffName = "Stat Boost";
	modifiers = mods;
	SourceSkill = sourceSkill;
}
```

## ActiveSkill도 수정
- 기존에 `float duration` 필드만 갖고 3가지 상황을 나눴음
	- 즉발
	- 지속시간 있음
	- 무한 지속

- 지속시간이 있는 경우는 큰 상관이 없는데, 즉발과 무한 지속을 어떻게 처리할지가 헷갈림
- 그래서 `enum SkillDurationType`을 추가하고 이것에 따라 분기를 설정하는 것으로 처리함
```cs
public enum SkillDurationType
{
	None,
	Instant, // 즉발
	Timed, // 지속 시간 있음
	Infinite // 무한 - 별도의 종료 조건 존재
}
```

- 스킬의 실행은 `OpSkillController`에서 처리함
```cs
if (skill.durationType == SkillDurationType.Instant)
{
	ExecuteInstantSkill(skill);
	
}
// 지속 시간이 있는 스킬
else if (skill.durationType == SkillDurationType.Timed || skill.durationType == SkillDurationType.Infinite)
{
	StartDurationSkill(skill);
}

private void StartDurationSkill(ActiveSkill skill)
{
	// 스킬 켜졌을 때의 VFX 활성화
	PlayDurationVFX(skill);
	
	// 참고) Shield의 경우 추가 VFX가 있는데 이걸 어떻게 처리할지 생각 필요

	// 코루틴 시작
	if (_activeSkillCoroutine != null)
	{
		_owner.StopCoroutine(_activeSkillCoroutine);
	}

	// 유한 지속
	if (skill.durationType == SkillDurationType.Timed)
	{
		_activeSkillCoroutine = _owner.StartCoroutine(Co_HandleDurationSkill(skill));
	}
	// 무한 지속(일단 - 탄환을 다 쓰면 풀림)
	else
	{
		_activeSkillCoroutine = _owner.StartCoroutine(Co_HandleInfiniteSkill(skill));
	}
}
```

#### 무한 지속 스킬 처리
```cs
private IEnumerator Co_HandleInfiniteSkill(ActiveSkill skill)
{
	skill.OnActivated(_owner);

	while (true)
	{
		if (_owner == null)
		{
			CleanupSkill();
			yield break;
		}

		// 스킬 틱 호출
		skill.OnUpdate(_owner);

		yield return null;
	}

	// ★도달 불가
	skill.OnEnd(_owner);
	CompleteActiveSkill(skill);
}
```
- `OpSkillController` 자체로는 종료가 불가능한 상황이다. 무한 루프이기 때문.

- 대신 `AmmoBasedActiveSkill`을 보면
```cs
	public override void OnActivated(Operator op)
	{
		// 스탯 변경 버프 생성
		StatModificationBuff statBuff = new StatModificationBuff(statModifier, this);

		// 공격 횟수 세는 버프 생성
		AttackCounterBuff attackCounterBuff = new AttackCounterBuff(maxAmmo, this);

		// 공격 횟수 세는 버프 종료 시 실행될 함수 지정
		// OnEnd는 Operator를 받고 콜백은 인자가 없으므로 람다식으로 감싸서 호출한다.
		attackCounterBuff.OnRemovedCallback += () => OnEnd(op);

		// 실제 버프 추가
		op.AddBuff(statBuff);
		op.AddBuff(attackCounterBuff);
	}

	public override void OnEnd(Operator caster)
	{
		base.OnEnd(caster);
	}
```
> **`attackCounterBuff`의 탄환 갯수를 트리거로 해서 스킬이 종료되게끔 해야 한다.**  버프를 트리거로 해서 종료되는 기능 자체는 이미 구현이 되어 있지만, 그게 `OpSkillController`까지 영향을 줘야 함.

- 일단 전체적인 구현은 거의 마무리가 됐음

```cs
if (originalAttackType != AttackType.None)
{
	owner.AttackType = originalAttackType;
}
```
