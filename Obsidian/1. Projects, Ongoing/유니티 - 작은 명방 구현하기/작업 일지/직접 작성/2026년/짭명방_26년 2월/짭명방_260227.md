## Barricade 배치 후에도 기존 경로가 수정되지 않는 현상
>[!done]
>- 2개의 버그가 겹쳐 있었다.
>1. `PathIndicator`의 `PathController` 정리 메서드가 `OnDiable`이라는 오타로 되어 있어서 구독 해제가 제대로 되지 않았음
>	- **에디터를 아예 멈췄다가 실행해도 이벤트가 구독이 남아 있는 문제가 발생할 수 있다**는 것도 배움
>2. 오브젝트를 만들고 비활성화하는 과정에서 `PathController`의 `Barricade`에 대한 구독 및 해제가 한꺼번에 이뤄졌고, `Initialize`로 실사용하는 시점에서는 구독이 되어 있지 않았음
>	- 2번 문제 자체는 심플한데, 1번 이슈가 2번 이슈를 숨겨주는 형태라서 까다로웠다. 디버깅 로그에서 뜬금없이 `PathIndicator`에 대한 게 나와서 겨우 알았다.
>
>- 해결
>1. `PathIndicator`의 오타 수정
>2. `PathController`의 `Iniialize` 시점에 이벤트를 구독하고, 유니티의 `OnDisable`에 실행되는 `CleanUp`에서 정리

- 이미 스폰된 적에 한해서만 이렇게 동작한다. 
	- 새로 생성된 적은 바리케이드를 피해서 돌아가는 경로를 설정함.

### 로깅 & 문제 발견
```
현재 노드 인덱스 : 9 / 전체 인덱스 갯수 : 10
```
> 예상보다 현재 인덱스가 훨씬 빠르게 돌아가는 게 문제인 듯

하지만 `_currentPathNodeIndex` 값 자체는 정상적으로 업데이트되고 있음.

- 별도의 테스트 환경에 대해서는 다른 오류가 발생하기도 했다.
```cs
InvalidOperationException: currentPathPositions가 비어 있음
```
바리케이드를 깐 시점에 발생하는 오류임

### 접근
1. 바리케이드를 깐 시점에 발생하는 오류인데, `_currentPathPositions`이 비어 있다?
```cs
if (_currentPathPositions == null || _currentPathPositions.Count == 0)
{
	Logger.LogError("_currentPathPositions에 필요한 정보가 없음");
	return false; 
}
```
> - 일단 `InvalidOperationException` 기반의 코드를 수정했음
> - 에러가 발생한 곳을 이상하게 가리키는 이슈가 있었다.

그런데 `_currentPathPositions`이 초기화되는 곳을 잘 모르겠음
```cs
protected void OnBarricadePlaced(Barricade barricade)
{
	Logger.Log("OnBarricadePlaced 동작");
	if (barricade.CurrentTile != null &&
		barricade.CurrentTile.EnemiesOnTile.Contains(_owner))
	{
		SetTargetBarricade(barricade);
	}
	else if (IsPathBlocked(_currentPathIndex)) // <-- 여기서 오류 발생 중
	{
		FindPathToDestinationOrBarricade();
	}
}
```
> `OnBarricadePlaced`는 이벤트를 받아서 곧바로 동작하는 메서드이고 다른 값을 수정하지 않는데 왜 `_currentPathPositions`가 초기화되는지를 모르겠다. 

### 상황 1

    {

테스트 환경에서 `PathIndicator`가 나타나지 않고 `Enemy`에 대해 테스트하는 상황임에도 이런 오류가 발생했음.

```
MissingReferenceException: The object of type 'PathIndicator' has been destroyed but you are still trying to access it.
Your script should either check if it is null or you should not destroy the object.
```
- 이전 환경에서 `PathIndicator` 객체가 파괴됐음에도 **이벤트 구독이 해제되지 않아서 발생하는 현상임**
- 유니티 **에디터에서 아예 게임을 멈췄다가 다시 재생하더라도 이런 이슈가 발생할 수 있음.** 

#### 해결
```cs
private void OnDisable()
{
	if (_path != null)
	{
		_path.OnDisabled -= ReturnToPool;
		_path.Cleanup();
	}
}
```
> - `OnDisable`이나 `OnDestroy`에 이런 식으로 클린업 메서드를 달아주면 됨.
> - `PathIndicator`의 정리 코드 자체는 있었다. 
> 	- 에디터에서 "이건 실행되지 않음"이라고 뜨길래 '? 잘 짰는데 또 IDE에서 뭔가 찐빠가 난 듯'하고 넘겼다. 
> 	- 변수로서 분명히 쓰이는데 쓰이지 않는다고 표시되는 경우가 종종 있었기 때문에, 이번에도 비슷한 이슈인가 하고 넘겼다.
> 	- 근데 `OnDisable`이 아니라 `OnDiable`로 써놔서 그런 거였다. ㅋㅋ...

> **완전히 엉뚱한 오류가 튀어나왔는데, 이게 생각보다 단서가 됐다.**
### 상황 2 - 원인 발견 및 해결
- 생각해보니 `PathIndicator`는 모든 맵에서 공통으로 사용하기 떄문에, 오브젝트 풀을 만드는 과정에서 생성됐겠다.

- `PathIndicator`의 기존 생성자 구조는 아래와 같았다.
```cs
public PathController(IMovable owner)
{
	_owner = owner;

	// 이벤트 구독
	Barricade.OnBarricadeDeployed += OnBarricadePlaced;
	Barricade.OnBarricadeRemoved += OnBarricadeRemovedWithDelay;
}
```
> 이게 `Enemy`나 `PathIndicator`가 생성되는 시점의 `Awake`에서 동작한다.

**`상황 1`을 해결하니까, `Enemy`의 경로 변경 로직이 동작하지 않았다.** 바리케이드를 깔았음에도 아무런 디버깅 로그가 올라오지 않았다. 즉, **기존 오류 로그는 `PathIndicator`에 대한 오류 로그였다.**

그러면 왜 여기서는 이벤트 구독이 되어 있지 않았을까?

- `상황 1`의 `OnDisable` 부분은 `Enemy`에도 똑같이 구현되어 있다.
- 이 프로젝트의 스테이지 씬에서 사용되는 모든 오브젝트는 로딩 과정에서 만들어진다. 
- 기존 구조를 보면 아래의 흐름이다.
	- 오브젝트가 만들어질 때, `_owner.Awake`에서 생성자 동작, `Barricade`에 대한 이벤트 구독
	- **비활성화되면서 `OnDisable`이 동작, `Barricade`에 대한 이벤트 해제**
	- **실제 사용을 위한 초기화 시점, 다시 활성화됨 : 동작 없음**
	- 구독이 해제되었기 때문에 바리케이드의 이벤트를 받아도 동작이 없던 것이다.

#### 해결
- 이벤트를 구독하는 부분을 `Initialize`로 옮겨주면 된다.
