## 효과음 겹칠 때 볼륨 감소(Voice Scaling)
- 일반적으로 `재생되는 효과음 갯수 = n`이라고 하면 **개별 소스의 볼륨을 `1 / sqrt(n)`으로 줄인다고 한다.**
	- 볼륨이 에너지 기준으로 합산되기 때문에 이런 방식을 쓴다고 한다. 
- 재생되는 전체 오디오 소스에 적용해야 함.
- 유니티 - `Audio Mixer - Duck Volume` 이펙트라든가, `FMOD / Wwise` 등의 미들웨어에서 통합적으로 관리할 수도 있다고 함.
- 이 프로젝트에선 직접 구현해봄. 어렵지 않기 때문에..
```cs
// PlaySFX 맨 밑에 추가

// 재생 중인 효과음 관리
_playingInstances[clip].Add(availableSource);

// 전체 볼륨 조절(같은 소리가 여러 개 재생되는 경우에 대한 처리)
float volume = 1f / Mathf.Sqrt(_playingInstances[clip].Count);
foreach (var source in _playingInstances[clip]) 
{
	source.volume = volume; 
}
```

## 사운드 밀리는 이슈
- 적이 이미 죽었는데 사운드가 또 발생하는 상황에 대한 문제가 어제부터 있었다. 
- 상황을 재현하기가 어려워서 처리하기가 어려운 이슈이기도 했음.

- 디버깅해본 결과, 이미 죽은 적에 대해서도 사운드가 재생되고 있을 가능성이 있음
```cs
// ArcaneFieldController.cs
protected override void ApplyPeriodicEffect()
{
	foreach (var target in affectedTargets.Keys)
	{
		if (target != null && target is Enemy enemy && caster != null)
		{
			AttackSource attackSource = new AttackSource(
				attacker: caster,
				position: transform.position,
				damage: caster.AttackPower * tickDamageRatio,
				type: caster.AttackType,
				isProjectile: true,
				hitEffectTag: hitEffectTag,
				showDamagePopup: false
			);

			GameManagement.Instance.Sound.PlaySFX(hitSFX);

			enemy.TakeDamage(attackSource);
		}
	}
}
```
- 여기서 조건문을 보면 `target`에 대해선 `null`체크만 하고 있다.
- 프로젝트 중간에 유닛 관리를 오브젝트 풀링으로 바꿨다. 그래서 **죽은 유닛은 비활성화**된다.
- 하지만 **비활성화됐다는 게 `null`이 됐다는 건 아니다.** 

그래서 `affectedTargets`을 선정하는 로직에 `!target.gameObject.activeInHierarchy` 조건문을 추가했다. 애초에 거기서 넘어온 거 자체가 문제이기 때문.
```cs
protected virtual bool IsTargetInField(UnitEntity target)
{
	// 조건 추가
	if (target == null || !target.gameObject.activeInHierarchy) return false;

	Vector2Int targetPos = MapManager.Instance!.ConvertToGridPosition(target.transform.position);
	return skillRangeGridPositions.Contains(targetPos);
}
```

수정하고 2번 더 테스트해봤는데 비슷한 상황이 발생하지 않았음. 일단 이 정도로 봉합하고 또 발생하면 그 때 처리하는 걸로.

## `caster` 사망 시에도 효과가 남는 문제
- `caster` 비활성화 이벤트를 받아서 스스로의 동작을 멈추게 해야 하는데 그런 로직이 없었음
```cs
// 공통 부모인 FieldEffectController에 구현
// UnitEntity.OnDisabled를 받아 동작함
protected virtual void OnCasterDisabled(UnitEntity entity)
{
	CleanUpAndReturnToPool();
	caster.OnDisabled -= OnCasterDisabled;
}
```

### Operator 재배치 타일 초기화 이슈
- 재배치 시에 "`newTile`이 없거나 기존 타일과 동일함" 발생
- 저 상황이 발생하는 상황이 특정 상황 외의 나머지라서 좀 까다롭다.
- 일단 `CurrentTile = null` 처리가 없어서 이것부터 시작해봄
- 근데 해결됐네? 굿..
- 코드를 다시 봐도 왜 `CurrentTile`이 초기화되지 않은 게 오류를 발생시켰는지는 모르겠다.
```cs
Vector3 position = _owner.transform.position;
Tile? newTile = MapManager.Instance!.GetTileAtWorldPosition(position);

if (newTile != null && newTile != CurrentTile)
{
	CurrentTile = newTile;
}
else
{
	Logger.LogError("newTile이 없거나 기존 타일과 동일함");
	return;
}
```

다만 위에서도 말했듯 직관성에 이슈가 있으니 코드는 아래처럼 수정함. 
```cs
if (newTile == null)
{
	Logger.LogError("타일을 찾을 수 없음");
}

if (newTile != CurrentTile)
{
	CurrentTile = newTile;
}
```
