## BGM 볼륨 관련
- 헤드폰을 쓰고 모니터링할 때는 BGM이 좀 크다는 느낌인데, 스피커로 체크할 때는 오히려 작다는 느낌이 들었다.
- 그래서 다른 게임들하고도 비교해봄. (스피커 세팅을 일정하게 맞춘 상황에서의 설정값)
	- 뮤제닉스 : 디폴트 마스터 볼륨이 70%. 100%으로 잡았을 때는 배경음의 요소들이 확실하게 들림.
	- 엔드필드 : 디폴트 마스터 볼륨이 100%. 조용한 BGM(제강호)은 잘 들리지 않지만 전투 상황에서의 BGM은 살짝 조용한 느낌이지만 잘 들리고, 효과음은 뚜렷하게 들림

- 마스터 볼륨을 80% 정도로 설정하고 BGM 파일들을 2dB 올린 버전으로 테스트해봤음
	- 내 프로젝트에서는 **BGM에 다른 효과음이 묻히는 경향이 있다.** 그래서 그냥 원래 버전으로 작업함.

## 효과음 동시 재생하다가 1개 재생될 때 볼륨 문제
>[!note]
>- 상황
>- `AudioClip`의 길이는 1초가 조금 넘고, `ArcaneFieldSkill`의 틱 간격은 0.5초
>- 2명을 동시에 공격하다가 1명만 남는 상황일 때, 이전의 효과음이 릴리즈만 남은 상황에서 **새로운 효과음을 1개 재생하는 상황임에도 클립이 여전히 2개로 체크**됨(둘 다 재생 중이기 때문)
>- 따라서 **새로 재생된 소리는 `1 / 루트2`의 볼륨**으로 들린다. 
>- **이게 인게임에서 티가 난다.**

> 위 상황을 어떻게 관리해야 하는가..에 대한 이슈. 

### 방향
- `특정 시간 간격`을 기준으로 그 안에 재생된 소리는 풀에 남겨두고, 시간이 지나면 풀에서도 제외하는 방식을 적용해봄
- 예를 들어 0초에 2개의 음원이 재생됐고 0.5초에 1개의 음원을 새로 재생해야 하는 상황이라면, **0.4초 쯤에 이전 2개의 음원은 "현재 재생 중인 풀"에서 제외**하는 방식임
	- 여기서 **제외하는 방식을 어떻게 처리할지**도 이슈겠다.

### 구현
- 제외하는 방식 처리 : 이전에도 2개를 초과하는 소리가 재생되면 해당 소리를 끊고 즉시 풀로 반환하는 식으로 처리했음. 여기서도 같은 방식을 이용함.

```cs

// 동시 재생 관련
private Dictionary<AudioClip, List<AudioSource>> _playingInstances = new(); // 실행 중일 때 각 글로벌 큐에서 여기로 이동
private Dictionary<AudioSource, float> _playStartTimes = new(); // 각 효과음이 시작될 때 시간을 등록
private float MAX_INSTANCE_PER_CLIP = 2;

// 효과음 실행 전, 큐 요소들 정리
private void ManageSFXQueue(AudioClip clip)
{
	if (!_playingInstances.ContainsKey(clip))
	{
		_playingInstances[clip] = new List<AudioSource>();
	}
	var clipAudioSources = _playingInstances[clip];

	// 임시 기준
	float poolExceptionTime = 0.4f;

	for (int i = clipAudioSources.Count - 1; i >= 0; i--)
	{
		AudioSource currentSource = clipAudioSources[i];
		
		// 만료(소리가 나고 있지만 체크될 필요는 없는 경우)
		bool sfxExpired = Time.time - _playStartTimes[currentSource] >= poolExceptionTime;

		// 재생 종료(실제로 재생이 끝난 경우)
		bool sfxFinished = !currentSource.isPlaying;

		if (sfxExpired || sfxFinished)
		{
			// 재생 중인 소리 멈추고 볼륨 원상 복구
			currentSource.Stop();
			currentSource.volume = 1f;

			// 관리 데이터 정리
			_playStartTimes.Remove(currentSource); // dict는 Remove(key)로 해당 요소 한꺼번에 제거 가능
			clipAudioSources.RemoveAt(i); // 현재 리스트에서 제거
			_sfxGlobalQueue.Enqueue(currentSource); // 글로벌 큐로 반환
		}
	}

	// 최대 재생 수 초과 시 가장 오래된 소스 강제 종료
	if (clipAudioSources.Count >= MAX_INSTANCE_PER_CLIP)
	{
		AudioSource oldest = clipAudioSources[0];

		oldest.Stop();
		oldest.volume = 1f; // 볼륨 원상 복구

		_playStartTimes.Remove(oldest);
		clipAudioSources.RemoveAt(0); // ex) [A, B, C] => [B, C]의 형태로 A만 제거
		_sfxGlobalQueue.Enqueue(oldest);
	}
}
```
> - 실행 전에 요소들을 정리하는 메서드인 `ManageSFXQueue`에서,  `sfxExpired`와 간련된 요소를 추가했다.

>[!note]
> - **제거를 위한 반복은 역순으로 구성**한다. `Remove()` 등의 제거 메서드는 인덱스를 바꾸기 때문에, 거꾸로 내려오는 게 안전하다. 
> - **딕셔너리에서의 `키, 밸류` 쌍 제거는 `Remove(키)`로 진행**한다.

- 잘 동작함. 마음에 걸리는 지점이라면 `0.4초`라는 지금의 필요에 의해서만 설정한 값이 있다. 크게 거슬리는 지점이 없다면 이대로 두겠음.
