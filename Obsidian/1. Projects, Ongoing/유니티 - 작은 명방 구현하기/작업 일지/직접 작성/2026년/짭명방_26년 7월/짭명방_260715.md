## 다른 상태 구현에 관해
- 위에서 `Stun`, `SkillCast`, `Death` 등에 대한 얘기가 나왔다.
- 애니메이션이 있다면 100% 구현하는 게 맞음. **이 프로젝트는 애니메이션은 딱히 없으니 구현하지 않아도 되겠다...는 느낌은 있지만 구현해보기로 함.**

1. **기존 시스템과의 융화**
	- `Stun` 
		- 이미 버프 시스템에서 처리하고 있던 부분
		- 상태가 버프 시스템을 구독하게 한 다음, 디버프가 풀리면 다시 `Move`나 `Combat`으로 복귀시키는 방식이 좋을 듯
	- `Death`
		- 유의할 점은 터미널 상태라는 점이다. 즉, 여기 돌입하면 다른 상태로 전환될 수 없다.
		- "탈출 불가"를 명시하는 구조로 만들지 단순 컨벤션으로 처리할지?

2. **우선순위**
	- 동시에 여러 조건을 만족했을 때의 우선순위가 필요하다.
	- 일반적으로 `Death > Stun > SkillCast > Combat > Move` 같이 처리되는 게 맞음
		- 추가로 `SkillCast` 중에 `Stun`이 들어왔다면 "스턴에 의해 보스의 스킬을 끊을 수 있는가" 같은 요소도 생각할 수 있음

3. **각 상태가 관리할 데이터 / 생명 주기**
- `Enter` : 진입 시 1회 처리
- `Update` : 매 프레임 처리
- `Exit` : 이탈 시 정리
	- 상태 전환 시 정리해야 할 리소스가 있을 수 있다. 예를 들어, `SkillCast`에서 코루틴이 돌아가고 있었다면 `Combat`으로 전환될 때 코루틴을 멈춰야 함.

### State에서 관리해야 하는 요소
- 판단 기준 : **이 값이 진행을 멈췄을 때, 그 동작은 버그인가 의도인가?**(스스로 물어보셈)
	- 버그 - 상태와 무관하게 항상 갱신되어야 하므로, 객체 레벨에서 관리
	- 의도 - 어떤 컨트롤러가 동작하지 않는 것 자체가 의도적으로 배척한 것, 상태에서 관리

- 예시(내 프로젝트 코드임)
```cs
protected override void Update()
{
    if (!IsUpdateValid()) return; 
    base.Update(); // 버프 효과 갱신
    UpdateAllCooldowns(); // 공격 쿨다운 등
    if (HasRestriction(ActionRestriction.CannotAction)) return;
    if (AttackDuration > 0) return;
    stateMachine.Update();
}
```
> - `UpdateAllCooldowns` : 쿨다운은 상태와 관계없이 시간이 지나면 줄어들어야 함. 상태와 무관한, 시간 경과에 따른 데이터.
> - 버프 지속시간 갱신 : 위와 마찬가지
> - 위 요소도 객체에서 **자신의 데이터를 갱신 -> 자신의 상태를 처리**하는 구조로 잡혀 있음. 

- `State`가 관리해야 하는 것
- 예를 들어 이동 상태라면, `이동 동작 실행, 모델 회전` 등은 현재 상태가 이동일 때에만 의미가 있는 동작이다. 이동 상태인데 `공격 컨트롤러`를 호출하지 않는 건 의도적으로 그렇게 설계한 거임.
- 즉, **`State`는 "모드에서 어떤 컨트롤러의 업데이트를 호출하고, 그 결과에 따라 어떻게 전환할 것인지"** 정도만 처리하면 충분하다.

- 즉, **같은 데이터를 두 곳에서 각자의 목적으로 확인하는 것이며, 완전히 똑같은 일을 두 곳에서 중복해서 처리하는 게 아니다.** `State`와 `Controller` 각각이 자신의 이유로 호출하는 것이다.

#### 2. 스킬 사용 조건에 관해
- `HasUsableSkill.cs`
```cs
// 현재 사용할 수 있는 스킬이 있는가를 판단함
public bool HasUsableSkill()
{
	if (_currentGlobalCooldown > 0f) return false;

	// 저지당한 상태에선 원거리 스킬을 사용할 수 없음
	List<EnemyBossSkill> skillsToCheck = (_owner.BlockingOperator != null) ? meleeSkills : rangedSkills;

	foreach (EnemyBossSkill skill in skillsToCheck)
	{
		if (skillCooldowns[skill] <= 0 && skill.CanActivate(_owner))
			return true;  
	}
	
	return false;
}
```
**여기 자체에 근거리일 때와 원거리일 때의 조건**을 넣으면 된다. 기존 `skill.CanActivate()`는 정말 해당 스킬만의 요건을 넣으면 되고, 근거리일 때 사용할 수 있는 스킬의 공통 조건이 있고 원거리 스킬일 때 사용할 수 있는 스킬의 공통 조건이 있기 때문임.
- 기존에는 `skill.CanActivate()` 내부에 근거리 & 원거리 스킬의 조건이 들어가 있어서 적어둔 부분.

- `HasUsableSkill`도 아래처럼 수정
```cs
public bool HasUsableSkill()
{
	if (_currentGlobalCooldown > 0f) return false;

	// 저지당한 상태에선 근거리 스킬만 사용할 수 있음
	bool isMelee = _owner.BlockingOperator != null;
	List<EnemyBossSkill> skillsToCheck = isMelee ? meleeSkills : rangedSkills;
	
	// 원거리인 경우, 타겟 유효성을 점검(근거리는 위에서 이미 점검함)
	if (!isMelee && _operatorsInSkillRange.Count == 0) return false;

	foreach (EnemyBossSkill skill in skillsToCheck)
	{
		if (skillCooldowns[skill] <= 0 && skill.CanActivate(_owner))
			return true;  
	}
	
	return false;
}
```

#### 3. EnemyBossSkill에서 castTime을 받아 시전 시간 동안 `SkillCast` 상태 유지시키기
- `TryUseSkill()`의 시그니처를 바꾼다.
	- `bool TryUseSkill()`이었는데, 이것만으로는 어떤 스킬이 발동됐는지 알 방법이 없다. `CastTime`을 받아내기 위해 `bool TryUseSkill(out EnemyBossSkill activatedSkill)`로 변경.
```cs
public bool TryUseSkill(out EnemyBossSkill activatedSkill)
{
	activatedSkill = null;
	
	// ...
	
		if (skillCooldowns[bossSkill] <= 0 && bossSkill.CanActivate(_owner))
		{
			bossSkill.Activate(_owner, mainTarget);

			// 쿨다운 설정
			_currentGlobalCooldown = COMMON_SKILL_COOLDOWN;
			skillCooldowns[bossSkill] = bossSkill.CoolTime;

			activatedSkill = bossSkill;

			return true;
		}
}
```

- 위 메서드는 `State`에서 사용, `_remainingCastTime`을 설정하는 데 사용된다.
```cs
public override void Enter()
{
	base.Enter();

	if (_enemySkillController.TryUseSkill(out EnemyBossSkill activatedSkill))
	{
		_remainingCastTime = activatedSkill.CastTime;
	}
	else
	{
		// 방어 케이스
		_remainingCastTime = 0f;
	}
}

public override void Update()
{
	if (_remainingCastTime > 0f)
	{
		_remainingCastTime -= Time.deltaTime;
		return; // 캐스팅 중일 때는 다른 행동 없이 대기
	}

	// 캐스팅 종료 / 실패 시 다음 상태 재판단
	if (_owner.Attack.TargetsInRange.Count > 0 && _owner.Attack.AttackCooldown <= 0f)
	{
		_fsm.ChangeState<EnemyAttackState>();
	}
	else
	{
		_fsm.ChangeState<EnemyMoveState>();
	}
}

public override void Exit()
{
	// 캐스팅 지속시간 + 코루틴이 있을 때 시전 중단 처리
	_remainingCastTime = 0f;
	
}
```

#### 4. 스킬 사용 중 정지 상태를 어떻게 바꿀 것인가
- 기존에는 `_owner.SetIsWaiting`, `_owner.SetStopAttacking`을 걸어두고 `yield return new WaitForSeconds()`으로 설정해서 동작을 막았음
- 근데 지금은 `State` 기반으로 전환됐기 때문에, 애초에 저 두 동작 자체가 실행되지 않음(`Move`나 `Attack` State로 전환하지 않는 이상)

- **일단 저 메서드들은 제거(상태 단위에서 자동으로 처리됨)**하는 것만으로 해결됨
