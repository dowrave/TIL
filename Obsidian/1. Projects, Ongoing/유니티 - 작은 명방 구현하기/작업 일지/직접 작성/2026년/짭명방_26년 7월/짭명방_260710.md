## 유한 상태 머신이란?
- `Finite State Machine(FSM)`
- **현재 하나의 상태만을 가지며, 조건에 따라 다른 상태로 전환된다.**
- 현재는 반드시 하나의 상태만을 가져야 한다. 

### FSM 여부에 따른 차이
- 쓰지 않았을 때 
```cs
void Update()
{
    if (hp <= 0)
    {
        Die();
        return;
    }

    if (player == null)
    {
        Patrol();
    }

    if (player != null)
    {
        Chase();
    }

    if (distance < attackRange)
    {
        Attack();
    }
}
```
> 각각이 어떤 의미를 가진 상태인지에 대해서도 생각해야 함. 


### 상태(State)
- **현재 객체가 무엇을 하고 있는가?**
- 예를 들어 `Idle, Move, Attack, Dead` 등이 있음. 
- 그리고 **각 상태는 `상태에 들어올 때, 상태 중일 때, 상태에서 나갈 때` 해야 하는 일이 다르다.** 
	- 대부분 **`Enter, Update, Exit`라는 3가지 함수로 구현**한다.

- 예를 들어 `Move -> Attack`이 된다면 아래처럼 처리한다.
```
Move.Exit()
Attack.Enter()
Attack.Update()
Attack.Update()
Attack.Exit()
```

### 상태 전환 함수(Change State)
- FSM에서 가장 중요한 함수.
```cs
ChangeState(new AttackState());
```

위처럼 구현한다면, 내부적으로는 이런 처리가 된다.
```
현재 상태 Exit() -> 새 상태 생성 -> 새 상태 Enter() -> 새 상태 Update() -> ...
```

### FSM의 구성 요소
- **객체는 자신이 어떤 상태인지만 갖고 있다.**
```
Enemy.StateMachine.CurrentState
```

- 그리고 실제 행동은 각 상태가 결정한다.
```
IdleState
MoveState
AttackState
DeadState
```
## 내 프로젝트에 반영하기
> FSM을 직접 구현해본 적이 없으니, 직접 타이핑하는 방향으로 작업함

### 기본 틀 개요
- `EnemyStateMachine`
	- `State` 전환 함수
	- `Update()` : 부모 `MonoBehaviour`의 `Update`에서 얘를 호출
- `EnemyState`
	- 공통 부모를 구현, 각 타입을 `enum`으로 정의할 필요는 없음.
	- **`MonoBehaviour`을 상속받지 않음.** 
		- **판단 기준 : 유니티에 컴포넌트로 넣는지 여부.** 
		- 따라서 `EnemyStateMachine`도 상속받지 않는다. 다 `Enemy`의 부속품이니까.

### 기본 틀 구현
- `EnemyStateMachine.cs`
	- 모든 상태를 `Dictionary<Type, EnemyState>`로 갖게 함
	- 상태 변화 함수는 제네릭으로 구현. `ChangeState<EnemyState>()`로 호출하면 됨.
```cs
using System;
using System.Collections.Generic;

public class EnemyStateMachine
{
    private Enemy _owner;
    private EnemyState _currentState; 
    private Dictionary<Type, EnemyState> _states = new();

    public EnemyState CurrentState => _currentState;

    public EnemyStateMachine(Enemy owner)
    {
        _owner = owner;

        _states[typeof(EnemyMoveState)] = new EnemyMoveState(this);
        _states[typeof(EnemySpawnState)] = new EnemySpawnState(this);

        // 최초 State는 Spawn으로 처리
        ChangeState<EnemySpawnState>();
    }

    public void ChangeState<T>() where T : EnemyState
     {
        _currentState?.Exit();
        _currentState = _states[typeof(T)];
        _currentState.Enter();
    }

    public void Update()
    {
        _currentState.Update();
    }
}
```
- `typeof()`과 `System.Type` 클래스에 대한 설명은 별도로 분리 : [[Csharp - typeof(), System.Type 클래스]]


- `EnemyState.cs` (모든 적의 상태의 부모가 되는 추상 클래스)
```cs
public abstract class EnemyState 
{
    protected EnemyStateMachine _stateMachine;

    // 이 클래스는 추상 클래스이므로, 외부에서 만들 일이 없음
    // 대신 자식에서 사용하기 때문에 protected
    protected EnemyState(EnemyStateMachine stateMachine)
    {
        _stateMachine = stateMachine;
    }

    public virtual void Enter() { }
    public virtual void Update() { }
    public virtual void Exit() { }
}
```

- 자식 클래스에서는 아래처럼 구현, 필요한 메서드는 오버라이드해서 사용.
