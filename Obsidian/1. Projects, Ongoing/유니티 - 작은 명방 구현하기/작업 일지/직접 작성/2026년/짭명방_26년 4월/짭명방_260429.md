## 버튼에 커맨드 할당하기
- 정확히는 "기능에 커맨드를 할당하기"이다.
- 어제 작업한 내용 중, 키를 변경하는 부분이 있었다. 따라서 변경된 키를 반영하는 방법이 필요함.

>[!note]
>- 바인딩된 키를 읽어와서, 각 기능에 할당하는 로직이 필요함
>- 이전에 구현했던 치트키 `F1`도 바인딩 매니저에 포함시키는 쪽으로 진행
### InputCommandManager : 커맨드 할당 등록 및 실행
```cs
using UnityEngine;
using System.Collections.Generic;
using UnityEngine.InputSystem;

public class InputCommandManager : MonoBehaviour
{
    private Dictionary<KeyAction, System.Action> commands = new();

    // 커맨드 할당 등록
    public void Register(KeyAction action, System.Action command) => commands[action] = command;

    private void Update()
    {
        foreach (var (action, command) in commands)
        {
            Key key = GameManagement.Instance.KeyBinding.GetKey(action);
            if (Keyboard.current[key].wasPressedThisFrame)
            {
                command.Invoke();
            }
        }
    }
}
```

#### 키 등록 과정 중
- 대부분의 기능은 `StageManager`에 있어서 상관없었음
- 근데 로비로 돌아가기 기능은 `StageUIManager.InGameTopButtonContainer` 내부에 있었다. 게임을 멈추고(StageManager) 로비로 돌아가기 패널을 표시(StageUIManager)하는 기능임.
- `StageUIManager`로 빼두는 시도를 해봤는데, 참조가 이상하게 꼬이는 느낌이 든다. **하위 컴포넌트의 메서드가 상위 컴포넌트의 메서드를 실행시키는 구조가 뭔가 이상했음**

- 이 경우, 버튼을 클릭했을 때 실행되는 메서드를 `StageManager`에 두고, `StageManager`에서는 게임을 멈추는 자기 기능만 수행하고 이벤트를 발생시킨 다음, `StageUIManager`에서 이벤트를 받아 패널을 띄우는 방식으로 처리함
	- 즉 `InGameTopButtonContainer`가 `StageManager`에 구현된 메서드를 실행시키기만 하면 됨. 싱글턴의 기능을 실행시키는 것이므로 이벤트로 구현할 필요도 없다.

```cs
inputCommandManager.Register(KeyAction.ChangeSpeed, HandleTimeScale);

private void HandleTimeScale()
{
	if (_currentGameState == GameState.Battle)
	{
		if (_slowState) 
		{
			Time.timeScale = 0.2f;
		}
		else
		{
			Time.timeScale = _isSpeedUp ? 1.5f : 1f; 
		}
	}
	else
	{
		Time.timeScale = 0f;
	}
}
```
> - 연결시킨 메서드가 "현재 상태에 따른 재생 속도"를 조절하기 때문에 아무 변화가 없었음
> - 버튼에 연결된 메서드와 동일하게 `ToggleSpeedUp`으로 수정(`IsSpeedUp` 상태를 변경)

2. ESC키를 눌러서 패널이 떴을 때, 다른 단축키의 동작 막기
- `InputCommandManager`에서 활성화/비활성화 상태랑 메서드만 추가해주면 됨
- 패널이 등장했을 때 비활성화, 사라졌을 때 활성화
- 근데 처음은 잘 동작하는데 2번째부턴 동작하지 않는 이상한 현상이 있다. 로그를 찍어봐도 `InputCommandManager`가 이상하진 않음.
- 정확히는 **2번째 등장했을 때부터는 스페이스바가 동작**하는 현상임
	- 유니티에선 `Spacebar / Enter`로 포커스된 요소를 동작시키는 기본 설정이 있다. 단축키를 막았음에도, 그 단축키는 내가 구현한 요소들에만 적용되기 때문에 유니티의 디폴트 설정으로 동작하는 요소가 막히지 않았던 것
	- 그래서 패널을 비활성화할 때 버튼의 포커스를 해제하는 기능을 추가해준다.
```cs
// ConfirmationReturnToLobbyPanel.cs
private void OnDisable()
{
	RemoveOnClickEventListeners();

	StageManager.Instance!.SetGameState(previousState);
	GameManagement.Instance.InputCommand.SetEnabled(true); 
	EventSystem.current.SetSelectedGameObject(null); // 버튼 포커스 해제(재활성화 시 Spacebar로 동작하는 현상 억제)
	Logger.Log($"패널 비활성화");
}
```
