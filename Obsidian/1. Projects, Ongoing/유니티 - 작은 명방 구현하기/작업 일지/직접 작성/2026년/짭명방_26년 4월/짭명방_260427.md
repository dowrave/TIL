## 키 매핑 & 변경 인터페이스 구현
- UI는 `OptionPopup`으로 통합하고, `PopupBase`를 상속받는 `OptionPopup`을 만듦
- 스테이지에서의 키 매핑만 구현
	- `ESC` : 로비로 돌아가기
	- `Spacebar` : 일시정지
	- `Tab` : 속도 전환

- 일단 `OptionPopup`을 옵션 설정으로 변경, 내부 탭으로 설정 섹션을 바꾸는 작업부터 시작.

- 탭 클릭으로 섹션 설정 변경하는 작업은 완료

- 일단 오늘은 `KeyBinding` 부분의 레이아웃까지만 작업하고 내일 스크립트로 연결함

### 지식이 늘었다
- 이전에 반복문을 구현하는 과정에서 `i`라는 반복문 자체의 인덱스를 쓰는 대신, 반복문 내에서 로컬 변수를 새로 만들어서 할당하는 부분이 있었다.
```cs
for (int i = 0; i < tabEntry.Count; i++)
{
	int index = i; // 클로저 캡처 버그 방지용(이유는 후술)
	tabEntry[i].button.onClick.AddListener(() => SelectTab(index));
}
```

>[!note]
>- 아래 내용은 위 코드 아래에 주석으로 달아둠.
>1. 리스너는 파라미터가 없는 메서드만을 등록할 수 있음. 그래서 바로 쓸 수 없고, 람다 함수를 쓴다.
>2. `tabEntry[i]`는 즉시 평가되므로 각각 올바른 버튼에 접근함
>3. 람다 함수 내의 index는 실제 버튼이 클릭되는 시점에 읽힌다. i로 넣을 경우 반복문이 끝난 시점에 실행되므로, `SelectTab(lastIndex)`가 실행됨
>4. 일반적으로 함수가 종료된 후 지역 변수는 사라지지만, 람다가 캡처한 변수(클로저)는 예외다. 
>5. 람다 함수가 람다 외부 변수(위 경우 `index`)를 참조하게 된다면 이 변수는 힙에 별도 객체로 저장되어서 람다가 살아있는 동안 유지된다.
