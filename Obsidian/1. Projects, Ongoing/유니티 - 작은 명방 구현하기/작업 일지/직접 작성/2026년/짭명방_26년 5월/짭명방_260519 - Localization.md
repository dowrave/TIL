
## 어제의 미해결 이슈
>[!note]
>- 언어를 변경할 때, `TMP`에서 폰트 설정을 어떻게 할 것인가? 

- 어제는 딱 결론이 나오는 게 없었다. 언어를 바꿨을 때 모든 씬의 모든 TMP 값을 일괄적으로 바꿔야 한다고 생각했었음.
- 오늘 접하면서 든 생각 : **`GetText()`에서 현재 설정된 언어에 맞춰서 폰트를 설정하고 텍스트도 설정하는 식으로 작업하면 되지 않을까?** 즉 그때그때 가져오는 방식이다. `LocalizationTable`도 애셋 테이블로 있음
	- 근데 지금 방식은 `GetText()`라는 걸 쓰지 않음 - 인스펙터에서 할당하고 있음

- 일단 **`Fallback`, 즉 현재 폰트에 없을 때 다른 폰트를 탐색하는 방식으로 시도해 봄**

## 폰트 이슈 해결(Fallback)
- `Fallback` : 대비책이라는 뜻
	- 프로그래밍 전반에서 많이 쓰이는 개념이라고 한다. 난 처음 들음.

- `TMP Font Asset`에는 `Fallback Font Assets`을 설정하는 필드가 있다. 현재 넣으려고 하는 글자가 폰트 아틀라스에 없을 때, 다른 폰트를 검색해서 찾아오는 개념임

- 내 경우, `SUITE` 폰트를 사용하고 있다.
![[Pasted image 20260519175137.png]]
> - `Fallback Font Assets`에 일본어 폰트를 집어넣으면 됨
> - 영어는 대부분의 폰트에 들어 있으니까 별도로 설정할 필요는 없다
> 	- `TMP`에서 글자 간격을 일일이 조절했던 게 있긴 한데, 일단 여기서는 그냥 넘어감

- 이렇게 설정하고 다시 어제 막혔던 이슈였던 일본어가 나타나지 않는 문제를 다시 테스트

## TMP 기본 설정 변경

### 기본 폰트 변경 - SUITE -> NotoSans

#### 이유
- `SUITE`의 영어 폰트가 이쁘지 않다는 인상이 있었음
- `NotoSansJP`에 영어랑 일본어 폰트가 다 있어서 이것만 두고 `SUITE`을 `Fallback`으로 둬도 되겠지만, 그냥 구조적으로 깔끔하게 `NotoSans`를 기본으로 잡고 Fallback으로 일본어랑 한국어를 별도로 두는 게 더 낫겠다. 

#### 과정
- `Google Fonts`에서 `Noto Sans`을 받아서 애셋으로 만들고 `Fallback`으로 `Suite`와 `NotoSansJP`로 설정.
	- 별도로 아틀라스를 만들 필요는 없음(SDF 파일을 만들 때 포함되는 과정)
	- 일본어 버전에서 포함했던 이유는 글자 수가 많아서 기본 아틀라스 설정으로는 잘리는 글자가 발생하기 때문.

#### 앞으로 추가할 TMP의 기본 설정 변경
- `Edit > Project Settings > TextMeshPro`에서 `Default Font Asset`을 `NotoSans`로 설정
- 이미 생성된 오브젝트들의 설정은 직접 바꿔줘야 한다고 함

#### 프리팹이랑 씬에 있는 `TMP`는 다 바꿔놨음
- 하이어라키에서 `t:textmeshprougui`로 검색하면 쫙 나옴
