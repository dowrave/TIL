## KeyBindingSection에 스크롤 뷰 구현

### Scroll View 복습
- (이전에도 다룬 적 있는 것 같지만) `Scroll View`의 구조는 아래와 같다.
```
Scroll Rect
- Viewport (Mask, Image) // 화면에 보여질 영역
  - Content // 전체 내용물
- Scrollbar
	- Sliding Area
	  - Handle
```
- 유니티에서 직접 Scroll View를 만들 때의 구조.
1. `Scrollbar`를 쓰는 경우, `Scroll Rect`의 자식으로 `Viewport`와 `Scrollbar`가 함께 오도록 하는 구조가 권장된다.(일부 `Visibility`에서 해당 설정이 강제됨)
2. `Viewport`의 경우 `Mask` 컴포넌트가 필요하며, `Mask` 컴포넌트는 `Image` 컴포넌트를 필요로 한다. 

- 이외에도 스크롤 바에 대해서 더 정리해둘 필요가 있겠음
	- 뭐 어찌저찌 잘 됐다. 이번엔 `Scrollbar` 부분이 배경이고 `Handle`이 스크롤 영역의 크기인데, 이 경우는 `Scrollbar`가 투명한 게 더 어울렸음.

### 기타 수정 사항
- `Scroll Rect`의 구조로 고쳤기 때문에 [[짭명방_260504]]에 했던 항목 생성 자동화가 생성되는 위치도 `Content`로 수정해야 함
```cs
Transform contentTransform = GetComponent<ScrollRect>().content.transform;
```

- 초기화 버튼도 구현. 이건 프리팹에 구현은 해두되, 가장 마지막에 와야 해서 스크립트도 추가로 들어감.
```cs
// Awake에서 설정
resetButtonContainer.transform.SetAsLastSibling(); // 가장 마지막에 배치되도록 수정
resetButton.onClick.AddListener(OnResetButtonClicked);
```

#### Horizontal Layout Group 관련 이상한 현상
- 자동화된 각 항목에 들어가는 요소 레이아웃에 살짝 문제가 생겨서 `Horizontal Layout Group` 기반으로 수정하는 과정에서..

```
Horizontal Layout Group
- TMP
- Empty(간격용)
- Button
```

여기서 `Horizontal Layout Group`을 켜면 아래처럼 정체 불명의 흰색 이미지가 나타난다.
![[Pasted image 20260506164508.png]]

- 발생 원인 : `Horizontal Layout Group`에 `Image`가 있다면, 하위 레이아웃을 조절하는 과정에서 해당 이미지가 나타날 수 있다.
	- 레이아웃을 껐을 때 나타나지 않았던 이유도 궁금하긴 한데 지금은 그거까지 들어갈 건 아닌 것 같다. `Image`를 비활성화하는 식으로 수정.
