## Special 믹서에 해당됐던 내용들 처리

### 기존 `Special` 믹서 의 제작 의도와 구현 방식
- 다른 BGM, SFX의 소리를 모두 낮추고 `Special`에 해당되는 소리만 들리게 하기
	- 보스의 원거리 스킬 시전 시 파티클이 떨어지는 과정에서의 소리를 없앴다.
- `Special` 믹서를 별도로 만들고, 해당되는 소리만 이 믹서에 할당하는 방식으로 처리했었음. 보스가 스킬을 시전했을 때 나머지 믹서들의 소리를 0으로 만들었다가 과정이 종료되고 나면 다시 소리가 커지는 방식.


###  fmod에서 비슷하게 구현하기
- `Special` 서브 그룹을 만들고 거기에 사용됐던 이벤트(원래는 음원)들도 해당 믹서로 옮겨줌
- 이 과정에서 `Snapshot` 얘기가 나와서 배우고 넘어가겠음

### FMOD Snapshot 공부
- **어떤 상태로 만들라는 명령**을 담은 프리셋
- 믹서에 일시적으로 씌우는 레이어로, **FMOD는 기본 믹서 상태를 항상 별도로 유지한다.** 
	- `Snapshot`이 적용됐다가 해제되더라도 원래의 세팅 값을 별도로 기억하는 코드를 짤 필요는 없다.

- 이 부분은 아예 공식 문서의 스냅샷을 참고로 AI에게 요약을 시켰다. 이미지는 직접 윈도우 캡처 도구로 잡아서 넣은 것.

#### `Scope` 
- 여기에 포함된 프로퍼티에만 영향을 준다. 
![[Pasted image 20260514153621.png]]
> 스튜디오에서는 건드린 값은 자동으로 `Scope In`되며(붉은 부분, 원래 값으로 되돌려 놓더라도 붉은색이면 스코프에 잡힌 상태), 위 이미지의 연두색 점선으로 쳐진 부분은 스코프에 포함되지 않은 값임

#### Overriding & Blending
- 볼륨 프로퍼티의 경우에만 `Blending`은 더하는 방식이고, 나머지는 다 덮어씌운다.

#### 복구 매커니즘
- 스냅샷이 종료되면 FMOD는 스코프 내부의 프로퍼티를 믹서에 설정된 기본 값으로 되돌린다. 

#### 페이드 처리
- `AHDSR Modulator`를 `Intensity`로 적용하는 방식으로 FMOD는 권장하고 있음
	- `AHDSR`은 엔벨로프의 그 5개를 의미함. `어택, 홀드, 디케이, 서스테인, 릴리즈`
- 유니티에서 코루틴으로 볼륨을 조정해도 되긴 하지만, 이 방식이 코드 상 `start/stop`만 호출해도 되는 간단한 방식이다.

### 실제 구현

#### FMOD Studio : 스냅샷 설정
![[Pasted image 20260514154224.png]]
- `Mixer`에서 `BossSkillDucking`이라는 스냅샷을 만들고, `VCA`로 `BGM, SFX` 값을 `-40dB`로 낮춤
	- 조절값은 테스트하면서 수정할 예정, VCA나 그룹 믹서 중 하나만 수정하면 될 듯
	- `Ducking` : 어떤 오디오 신호가 나올 때 다른 오디오 신호를 줄이는 방식.
		- 사이드 체인이랑 비슷한 개념이다. 
		- 차이점이라면, 사이드 체인은 줄어드는 정도가 기준이 되는 신호의 영향을 받는 개념, 지금 구현하는 덕킹은 줄어드는 양을 개발자가 정하는 개념임

- `AHDSR` 설정법
![[Pasted image 20260514162515.png]]
> 설정에서 약간 헤맸다. 이것저것 테스트하고 정리함.
> 1. 최종적으로 설정되기 원하는 값을 `Scope`로 잡아둔다.
> 2. 하단의 `Intensity -> Add Modulation -> AHDSR`을 설정하면 오른쪽의 엔벨로프가 나타난다. 
> 	- `Intensity`는 1번에서 설정한 값을 얼마만큼 적용할 것인가에 대한 지표이다.
> 	- 예를 들어서 `-15dB`로 설정했고, `Intensity`에서 50%을 잡아뒀다면 실제로 스냅샷을 실행했을 때 떨어지는 값은 `-7.5dB`가 되는 개념.
> 	- 그래서 일반적으로 `Intensity`를 건드릴 일은 잘 없지 않을까 싶다. 내려가길 원하는 값은 1번에서 다 세팅해둘 테니까.
> 3. `Event Editor`에서 `Snapshot`에 해당하는 이벤트를 실행시키고, 스냅샷을 켜고 끄면서 원하는 방식으로 잘 적용되는지 테스트.

- 코드에서의 호출법은 인스턴스 사용과 동일하다. 경로만 바꾸면 됨.
```cs
_snapshot = RuntimeManager.CreateInstance("snapshot:/BossSkillDucking");
_snapshot.start();

_snapshot.stop(FMOD.Studio.STOP_MODE.ALLOWFADEOUT);
_snapshot.release();
```

그러면 위의 `set3DAttributes`도 필요 없어지는 듯.

### 스냅샷 추가 설정 - 로우 패스 필터
- 위에선 단순히 볼륨만 낮추는 식으로 구현했는데, 이것 이상으로 **다른 소리를 "먹먹하게" 나오게 하고 싶다.**
- 이 경우는 고음을 깎는 `LPF(로우 패스 필터)`를 적용한다는 듯. 자세한 건 이전에 공부한 적 있어서 생략함.

#### `fmod`에서 적용하는 방법
1. 각 그룹`BGM, SFX`에서 하단의 `Master Track`의 좌측 부분에 `Add Effect > FMOD Deprecated > FMOD Low Pass`를 클릭

2. 디폴트 설정을 관리하는 부분이므로, `Cutoff`는 최댓값으로 잡아서 어떤 주파수도 걸러내지 않게 설정

![[Pasted image 20260514224530.png]]
> `Mixing Desk` 상태에서 각 그룹을 클릭하고, 로우패스 필터를 넣었을 때의 설정. 아래의 4번이랑 비교해보면 됨.

3. 이 상태에서 **`Snapshots` 탭의 원하는 스냅샷을 클릭하면 믹서 탭이 해당하는 스냅샷에 대한 설정 창이 뜸**
![[Pasted image 20260514224326.png]] -> ![[Pasted image 20260514224338.png]]
> 이 부분에서 특히 헤맸다. 지금 설정하는 부분이 기본 설정인지 스냅샷 설정인지 헷갈렸기 때문인데, 스냅샷 탭을 누른 상태면 스냅샷 설정을 하고 있는 거임.

4. 스냅샷에 대한 설정창이 뜬 상태에서 다시 `Routing`의 그룹을 클릭해보면, 스코프에 잡히지 않은 값은 점선으로 나타나는 창이 나타남.
![[Pasted image 20260514224143.png]]
- 이 상태에서 `Cutoff`값을 원하는 값으로 수정하면, 스냅샷을 켰을 때 원하는 값 이상의 주파수는 필터링이 적용되어 먹먹한 소리로 나온다. 

>[!note]
>- 이렇게 고음을 깎아내면 전체적인 소리의 크기가 줄어들기 때문에, 이전에 설정했던 스냅샷의 볼륨은 덜 깎아내도 되게 된다.

#### 추가 참고
- 위에서 `Routing`의 각 그룹을 클릭하는 방식으로 진행했는데, 스냅샷 자체에서 진행할 수 있을 것 같다.
![[Pasted image 20260514225239.png]]
1. 스냅샷에 잡혀 있는 그룹들을 클릭
2. 하단의 `Group` 왼쪽 부분에서 `Add Effect > FMOD Deprecated > FMOD Low Pass`으로 필터 추가
3. 스냅샷을 끄고, `Routing`의 각 그룹에서 `Cutoff`값을 최대로 설정
4. 스냅샷 탭이 켜진 상태에서 필터링되길 원하는 주파수 `Cutoff`를 설정
