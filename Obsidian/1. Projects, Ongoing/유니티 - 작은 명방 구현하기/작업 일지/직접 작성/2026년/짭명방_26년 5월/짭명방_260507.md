## 단축키 할당 버튼 상/하단에 시각화
- 관련 컴포넌트를 하나 만들어야 할 듯

- 대충 개요 작성
	- `Image`랑 `TMP`가 들어간 컴포넌트
	- 연결된 `KeyAction`에 대한 필드도 하나 필요(어떤 `Key`인지 가져와야 하니까)
	- 스테이지 시작 시 `Initialize()` 등의 메서드로 애니메이션 동작
		- 구체적인 애니메이션 
			- 부모버튼의 위치에서 시작, 가려진 채로 출발
			- 특정 거리만큼 이동(인스펙터에서 할당 : 속도, 거리) 후 해당 위치에서 페이드아웃 후 비활성화
	- 이후에도 특정 단축키를 클릭해서 관련 요소를 모두 볼 수 있게 하는 기능도 추가 예정
		- 단축키의 동작은 단순히 비활성화된 위치에서 나타났다가 사라졌다가 하는 기능. 이건 나중에 구현하면 됨

### 초안
```cs
using UnityEngine;
using DG.Tweening;
using TMPro;

public class KeyBindDisplay : MonoBehaviour
{
    [Header("References")]
    [SerializeField] private TextMeshProUGUI buttonLabelText; 
    [SerializeField] private KeyAction action;

    [Header("Animation Configs")]
    [SerializeField] private float movementDuration;
    [SerializeField] private float moveDownDistance;
    [SerializeField] private float fadeOutDuration;
    [SerializeField] private Ease moveEase = Ease.OutQuad;
    [SerializeField] private Ease fadeEase = Ease.InQuad;

    private CanvasGroup canvasGroup;
    private Sequence animationSequence;
    private RectTransform rectTransform;

    private void Awake()
    {
        rectTransform = GetComponent<RectTransform>();
        canvasGroup = GetComponent<CanvasGroup>();
    }

    public void Initialize()
    {
        buttonLabelText.text = GameManagement.Instance.KeyBinding.GetKeyString(action);

        // 버튼의 현재 위치에서 하단으로 이동 후 사라지는 로직
        canvasGroup.alpha = 1f;

        animationSequence?.Kill();
        animationSequence = DOTween.Sequence().SetUpdate(true);

        animationSequence
            // 현재 앵커 Y좌표에서 moveDownDistance만큼 아래로 moveDownDuration만큼 이동
            .Append(rectTransform.DOAnchorPosY(rectTransform.anchoredPosition.y - moveDownDistance, movementDuration).SetEase(moveEase))

            // fadeOutDelay만큼 대기
            .AppendInterval(fadeOutDelay)

            // 위 동작 후(Append), fadeOutDuration만큼 알파값을 0으로 페이드아웃함
            .Append(canvasGroup.DOFade(0f, fadeOutDuration).SetEase(fadeEase))
            
            // 동작 완료 후 오브젝트를 비활성화함
            .OnComplete(() => gameObject.SetActive(true));
    }

    private void OnDisable()
    {
        animationSequence?.Kill();
    }
}
```
- 구체적인 애니메이션 로직 구현은 클로드의 도움을 받았다. 어떤 식으로 동작하는지 정도는 남겨놓고 싶어서 주석으로 표시해둠.

> 원래 `AppendInterval`은 없었는데, 테스트해보니까 보여줄 부분은 뚜렷하게 보여줘야 하므로 대기 시간을 상대적으로 길게 잡고, 사라지는 시간은 짧게 잡는 게 맞는 것 같다.


- 그러면 이런 초안을 어떻게 활용할지도 고민해봐야겠다.
- 중앙 매니저를 두는 방식 vs 개별 프리팹을 따로 넣어두는 방식 사이에서 고민 중.
	- 버튼의 크기가 다르다. 4개는 같은데 1개가 다름.
	- 저 다른 버튼은 게임 중에 움직일 수 있다.
	- 이 요소(디스플레이)는 버튼에 가려져야 한다. 
- 위치를 추적하는 기준이라면 자식 오브젝트로 두는 게 편한데, 그러면 "가려져야 한다"는 조건이랑은 또 맞지 않게 되는 게 있어서 좀 복잡하다.

private void RegisterShowingKeyBinding()
{
	if (!isKeyBindingRegistered)
	{
		GameManagement.Instance.InputCommand.RegisterHold(KeyAction.ShowKeyBinding, ShowKeyBindings, HideKeyBindings);
		isKeyBindingRegistered = true;
	}

private void ShowKeyBindings()
{
	foreach (var entry in buttonKeyActionEntries)
	{
		if (entry.action != KeyAction.None)
		{
			entry.display.gameObject.SetActive(true);
		}
	}
}

private void HideKeyBindings()
{
	foreach (var entry in buttonKeyActionEntries)
	{
		if (entry.action != KeyAction.None)
		{
			entry.display.gameObject.SetActive(false);
		}
	}
}

## InputCommandManager에 "버튼을 누르는 동안"에 동작하고, "버튼을 떼면" 취소되는 기능 추가
- 기존의 단축키는 `wasPressedThisFrame`을 `Update`마다 점검했음 
- "누르는 동안에만 활성화" 기능을 쓰려면 별도의 커맨드를 추가해줘야 함

```cs
private Dictionary<KeyAction, (System.Action onPress, System.Action onRelease)> holdCommands = new();

public void RegisterHold(KeyAction action, System.Action onPress, System.Action onRelease)
    => holdCommands[action] = (onPress, onRelease);

private void Update() 
{
    foreach (var (action, hold) in holdCommands)
    {
        Key key = GameManagement.Instance.KeyBinding.GetKey(action);
        if (Keyboard.current[key].wasPressedThisFrame)
            hold.onPress.Invoke();
        else if (Keyboard.current[key].wasReleasedThisFrame)
            hold.onRelease.Invoke();
    }
}
```
> "누르는 동안에만 유지"되는 기능을 구현하겠다면, 이전의 `wasPressedThisFrame`과 `wasReleasedThisFrame`을 함께 사용하면 됨. 이를 위해 눌렀을 때`onPress`와 뗐을 때`onRelease` 각각 동작할 메서드를 따로 구현해서 등록하면 된다.

## OptionPopup 단축키 추가
- 스테이지에서만 동작(메인메뉴에 있어도 상관은 없겠다)
- 기본은 `O`인데, 유의할 점은 팝업이 등장했을 때 팝업을 끄는 단축키도 동일한 키로 구현해야 한다는 것. 

- 처음에는 이런 식으로 시도했음
```cs
inputCommandManager.Register(KeyAction.ShowOptionPopup, StageUIManager.Instance.ShowOptionPopup);
```

- 클로드에게 물어보니, 아예 등록할 때부터 팝업의 상태에 따라 동작을 다르게끔만 설정하면 된다는 듯. 람다함수로 구현하면 된다. 
```cs
inputCommandManager.Register(KeyAction.ShowOptionPopup, () => 
{
	// 팝업의 활성화 여부 체크
	bool isOpen = StageUIManager.Instance.OptionPopup.ActiveSelf;	
	if (!isOpen)
		StageUIManager.Instance.HideOptionPopup();
	else
		StageUIManager.Instance.ShowOptionPopup();
})
```
> 여기서 람다 함수는 **`inputCommandManager`의 `commands` 딕셔너리가 살아있기만 하면 유지된다.** 
> 어쨌든 여기서의 포인트는 **활성화 여부에 따른 동작을 따로 구현할 필요 없이 함수 하나에 넣어서 다 처리해버릴 수 있다**는 것.

- `OptionPopup`은 메인메뉴와 스테이지 모두 있기 때문에 메인메뉴 씬에서도 단축키 연결
```cs
// MainMenuManager
// 팝업의 위치가 다르다
private void RegisterCommands()
{
	GameManagement.Instance.InputCommand.Register(KeyAction.ShowOptionPopup, () =>
	{
		bool isOpen = PopupManager.Instance.OptionPopup.gameObject.activeSelf;
		if (!isOpen)
			PopupManager.Instance.ShowOptionPopup();
		else
			PopupManager.Instance.HideOptionPopup();
	});
}

// OnDestroy
if (GameManagement.Instance != null)
{
	GameManagement.Instance.UserSquadManager.OnSquadUpdated -= OnSquadUpdated;
	GameManagement.Instance.InputCommand.UnregisterAll(); // 단축키 할당 해제
}
```

### 이슈 2 - 팝업이 켜졌을 때 InputCommand를 비활성화함
- 팝업이 나타날 때는 단축키가 동작하지만, 나타난 후에는 `inputCommand`의 모든 동작을 막음
- `OptionPopup`에 대한 키만 동작시키고 싶다.

- 이 경우, **`InputCommandManager`에서 화이트리스트를 만들어야 한다**고 한다. 즉 모두 막은 다음 특정 단축키만 허용하는 방식. 

```cs
// InputCommandManager.cs
private HashSet<KeyAction> whitelistedActions = new();

public void SetEnabled(bool enabled, params KeyAction[] whitelist)
{
	isEnabled = enabled;
	whitelistedActions.Clear();
	foreach (var action in whitelist)
	{
		whitelistedActions.Add(action);
	}
} 

private void Update()
{
	// !isEnabled 조건이 원래 최상단에 있었는데 이걸 아래 2개로 나눔
	
	foreach (var (action, command) in commands)
	{
		Key key = GameManagement.Instance.KeyBinding.GetKey(action);

		// 비활성화됐을 때에는 화이트리스트에 있는 액션만 수행
		if (!isEnabled && !whitelistedActions.Contains(action)) continue;

		if (Keyboard.current[key].wasPressedThisFrame)
		{
			// Logger.Log($"현재 입력된 키 : {Keyboard.current[key]} / command 실행됨");
			command.Invoke();
		}
	}

	foreach (var (action, hold) in holdCommands)
	{
		Key key = GameManagement.Instance.KeyBinding.GetKey(action);

		if (!isEnabled && !whitelistedActions.Contains(action)) continue;

		if (Keyboard.current[key].wasPressedThisFrame)
			hold.onPress.Invoke();
		else if (Keyboard.current[key].wasReleasedThisFrame)
			hold.onRelease.Invoke();
	}
}
```

- 그리고 `OptionPopup`에서 `SetEnabled`을 아래처럼 수정만 해주면 됨
```cs
GameManagement.Instance.InputCommand.SetEnabled(false, KeyAction.ShowOptionPopup);
```
#### 지식이 늘었다 파트 : params 타입에 대해
```cs
public void SetEnabled(bool enabled, params KeyAction[] whitelist)
```

- **`param` 키워드 : 가변 인자**를 받을 때 사용한다. 배열 타입 앞에 붙여서 배열을 만들어서 전달하는 대신, **인자를 나열하는 방식으로 전달**할 수 있다. 
```cs
// params 인자가 없을 때 : 배열을 만들어서 전달해야 함
KeyAction[] keyActions = new KeyAction[] {
	KeyAction.ShowOptionPopup, 
	// ...
}
SetEnabled(true, keyActions)

// params 인자가 있을 때 : 아래 4개 모두 사용 가능
SetEnabled(false, KeyAction.ShowOptionPopup);
SetEnabled(false, KeyAction.ShowOptionPopup, KeyAction.ShowKeyBinding);
SetEnabled(false);
SetEnabled(false, new KeyAction[] { KeyAction.ShowOptionPopup });
```
