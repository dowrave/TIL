## BGM 전환 로직
### 기존 유니티 오디오 시스템에서의 구현 방식
- 2개의 음원을 동시에 재생시켜놓고, 한쪽만 뮤트를 시켜놓음
- 전환 상황이 발생하면, 뮤트되지 않은 쪽은 서서히 페이드아웃되고 이후에 뮤트된 쪽이 페이드인되면서 들어오는 방식

### `FMOD`에서의 구현 방식
- **1. 이벤트에 2개의 트랙을 달아둠.**
	- 기본 설정으로 보스 스폰 후에 재생될 BGM은 뮤트를 걸어둠

- **2. 이벤트에 파라미터 생성**
	- 이벤트 클릭 후, 상단 탭에서 `Create > Add Parameter(Ctrl + P)`으로 파라미터를 추가함
	- `Continous, 0 ~ 1, Local` 등으로만 일단 잡아놓음. 

![[Pasted image 20260515151428.png]]
> - 파라미터를 생성하고 우측을 보면 이렇게 파라미터의 이름과 게이지, 오른쪽의 체크박스가 나타나야 함
> - 그리고 이 이벤트에 대한 파라미터이므로 다른 이벤트를 클릭했을 때는 해당 파라미터가 나타나지 않음

- **3. 각 트랙의 볼륨에 대한 오토메이션을 생성함**
	- 트랙의 볼륨 게이지 우클릭 -> `Add Automation`

- **4. 우측 `Overview`의 파라미터 항목에 있는 체크박스를 클릭**

![[Pasted image 20260515151644.png]]

> 혹은 아래 이미지의 `Timeline`의 오른쪽 탭 클릭 > `Add Parameter Sheet`
![[Pasted image 20260515151712.png]]

- **5. 파라미터로 제어하는 볼륨 오토메이션 설정**
![[Pasted image 20260515151813.png]]
- 3번에서 볼륨에 대한 오토메이션을 생성했다면, 파라미터 시트에서도 볼륨에 대한 오토메이션을 설정할 수 있는 트랙이 나타난다.
- 내 경우, 기존에 재생되던 음악을 페이드아웃 한 후 새로운 음악을 페이드인하는 식으로 구현하기 위해 `0.5`까지는 기존 음악의 페이드아웃을, `0.5 ~ 1`은 새로운 음악이 페이드인되는 식으로 구현했음.

- **6. 유니티에서 코드로 제어**
	- **이제 파라미터의 값만 0에서 1로 올려주면 된다.** 

```cs
// SoundManager.cs

// PlayBGM
currentBGM = RuntimeManager.CreateInstance(bgmRef);
currentBGM.setParameterByName(BGMTransitionParam, 0f); // 파라미터가 없어도 그냥 재생됨
currentBGM.start();

public void SwitchTrack()
{
	if (_isSwitching) return;
	StartCoroutine(MoveParameter(from: 0f, to: 1f, transitionDuration));
}

private IEnumerator MoveParameter(float from, float to, float duration)
{
	_isSwitching = true;

	float elapsed = 0f;
	while (elapsed < duration)
	{
		elapsed += Time.unscaledDeltaTime;
		float t = Mathf.Clamp01(elapsed / duration);
		currentBGM.setParameterByName(BGMTransitionParam, Mathf.Lerp(from, to, t));
		yield return null;
	}

	currentBGM.setParameterByName(BGMTransitionParam, 1f);
	_isSwitching = false;
}
```

> 여기서 **`setParameterByName`은 파라미터가 없는 이벤트들에 대해서도 오류를 일으키지 않는다.** 그냥 쓰면 됨.

### 테스트
- `BGM` 전환 로직이 작동하지 않음 
	- `EventReference`로 바꾸는 과정에서 `StageData.alternativeBGM`이라는 필드가 필요 없어짐(하나의 이벤트에 2개의 트랙을 넣고 전환하는 식으로 바뀌었기 때문)
	- 기존 로직은 `alternativeBGM`의 `null` 여부를 체크했는데, `baseBGM` 여부로 변경하고 `StageData`의 `alternativeBGM` 필드를 제거함

- 재 테스트) BGM 전환 로직이 잘 동작함.
	- 단, 바뀐 BGM이 이전의 BGM에 악기를 추가한 형태인데 그게 크게 부각되지는 않고 있다. 아예 바뀐 BGM을 다시 만져봐야 할지도.

## 클릭 소리 안들림 
- `PlayButtonClick()`이 주석 처리되어 있었음.
- 버튼 클릭 소리는 공통으로 쓸 일이 많아서 `SoundManager`에 아예 별개의 메서드로 구현해뒀다. 
- 이외에도 기존 유니티 오디오 시스템을 기준으로 작성되어 있어서 주석처리했던 코드들을 모두 `FMOD` 기반으로 동작하도록 바꿔줬다.
- 스킬 등에 할당되어 있지 않았던 `FMOD Event`들도 다 연결해줌

## FMOD의 동시 재생 갯수 처리

> [!note]
> - 기존 코드의 경우, 동시에 재생될 수 있는 `SFX`의 갯수를 추적해서 최대 2개로 제한해뒀는데, FMOD에서 어떻게 관리되는지는 한 번 봐야 할 듯?
> - 크게 상관없으면 괜찮지만, `ArcaneFieldSkill` 같이 같은 소리가 여러 번 재생되어야 하는 경우에 위화감이 들었기 때문에 이 부분은 체크할 필요가 있겠다.

- 이거도 **스튜디오 레벨에서 처리**할 수 있다고 한다.
- 근데 AI가 던져준 얘기가 내가 보는 화면에선 없는 경우도 많아서 이것도 기능을 좀 찾아봐야 할 듯
- 일단 `Claude`는 `Polyphony`를 설정하라고 얘기해줬는데, **내가 검색해서 알아봤을 땐 `Max Instances`가 더 관련 설정에 가까워보인다.** 동시에 재생되는 인스턴스의 수를 제한하는 개념이기 때문. 

![[Pasted image 20260515164626.png]]
- `Event Editor`를 눌렀을 때 우측 하단의 `Event Macros` 설정. 
- `Max Instances`을 수정해줌. 
	- 문제가 됐던 부분이 `LightningHit`이어서 이것만 2로 잡아줬다.
	- 나머지 경우..는 이상한 부분만 수정하면 될 듯?
	- 디폴트 설정은 무한.

- 추가 ) `Stealing`의 경우 `Oldest, Quietest, Virtualize, None, Furthest` 등이 있다. 
	- `None`의 경우 소리가 거의 끝나가는 상황에서도 새로운 음원이 재생되지 않으므로 배제
	- `Oldest`가 기본 설정인데, 어떤 상황에서 팍 하는 소리가 들린다. 
	- `Quietest`도 테스트해봄 : 소리가 나야 하는데 나지 않는 경우가 있음
	- **으아아 겁나 복잡하다;** 가장 괜찮았던 게 `Oldest`라서 이걸로 잡고 넘어감

### 같은 효과음이 동시에 여러 개 재생될 때의 볼륨 처리
- 사실 이게 더 중요했다. `LightningHit`에서 신경쓰였던 부분이라면, 효과음이 겹쳤을 때 너무 커지고, 반대로 효과음이 겹치지 않게 됐을 때에 너무 작아지는 느낌이 있었기 때문.
- 이 부분은 그냥 `SFX Bus`에 리미터만 걸고 넘기겠음.

## 기타 수정 사항
### `StageScene`에서 단축키 클릭 시 버튼 클릭 소리 들리게 수정
- 버튼을 눌렀을 때 다른 피드백이 없는 게 이상해서 추가함
- 기존 코드
```cs
    // 단축키 - 기능을 연결하는 메서드
    private void RegisterCommands()
    {
        InputCommandManager inputCommandManager = GameManagement.Instance.InputCommand;

        inputCommandManager.Register(KeyAction.ReturnToLobby, RequestReturnToLobby);
        inputCommandManager.Register(KeyAction.ChangeSpeed, ToggleSpeedUp);
        inputCommandManager.Register(KeyAction.Pause, TogglePause);
        inputCommandManager.Register(KeyAction.CheatWin, GameWin);
        inputCommandManager.Register(KeyAction.ToggleStatisticPanel, StatisticsManager.Instance.TogglePanel);
        inputCommandManager.Register(KeyAction.ShowOptionPopup, () =>
        {
            bool isOpen = StageUIManager.Instance.OptionPopup.gameObject.activeSelf;
            Logger.Log($"옵션 단축키 클릭됨 : {isOpen}");
            if (!isOpen)
            {
                Logger.Log("단축키 - ShowOptionPopup");
                StageUIManager.Instance.ShowOptionPopup();
            }
            else
            {
                Logger.Log("단축키 - HideOptionPopup 동작");
                StageUIManager.Instance.HideOptionPopup();
            }
        });
    }
```

- 내부에 `inputCommandManager.Register`을 래핑하는 로컬 함수를 하나 추가해주면 됨
```cs
private void RegisterCommands()
{
	// ...

    void RegisterWithSound(KeyAction action, Action callback)
    {
        inputCommandManager.Register(action, () =>
        {
            SoundManager.Instance.PlayOneShot(SFXKey.ButtonClick);
            callback();
        });
    }
	RegisterWithSound(KeyAction.ReturnToLobby, RequestReturnToLobby);
}
```

```cs
{

	yield return new WaitForSecondsRealtime(blockDurationTime);
	blockingPanel.gameObject.SetActive(false);
	returnToLobbyText.text = "로비로 돌아가기";
}
```
