### 구현 과정
- 각 섹션에 들어가는 세부 컨테이너의 UI는 이미 구현되어 있으니까, 이를 떼내고 원래 섹션에서 관리하던 요소들도 일부 분리함
- 이 과정에서 각 컨테이너의 이름, 즉 `KeyAction`의 기능을 어떻게 넣을지가 좀 까다로웠다. `enum`으로 관리되고 있는 요소들이고, 이전에 `LocalizationManager`도 구현했던 게 떠올라서 반영했음.

```cs
// KeyBindingManager.cs --------
public enum KeyAction
{
    None,
    [KeyActionLabel("keybinding.returntolobby")]
    ReturnToLobby,
    [KeyActionLabel("keybinding.pause")]
    Pause,
    [KeyActionLabel("keybinding.changespeed")]
    ChangeSpeed,
    [KeyActionLabel("keybinding.cheatwin")]
    CheatWin
}

[System.AttributeUsage(System.AttributeTargets.Field)]
public class KeyActionLabelAttribute: System.Attribute
{
    public string LocalizationKey { get; }
    public KeyActionLabelAttribute(string localizationKey) => LocalizationKey = localizationKey;
}

// KeyBindingButtonContainer.cs --------
using UnityEngine;
using UnityEngine.UI;
using TMPro;
using System.Collections.Generic;
using UnityEngine.InputSystem;
using System.Reflection;

public class KeyBindingButtonContainer: MonoBehaviour
{
    [Header("References")]
    [SerializeField] private Button button;
    [SerializeField] private Image buttonImage;
    [SerializeField] private TextMeshProUGUI containerLabel;
    [SerializeField] private TextMeshProUGUI buttonText;
    [SerializeField] private KeyAction action; // PlayerPrefs에는 action.ToString()으로 전달해서 사용하면 됨  

    [Header("Color Configs")]
    [SerializeField] private Color normalColor = new Color(0.96f, 0.96f, 0.96f, 1);
    [SerializeField] private Color selectedColor = new Color(0.85f, 0.85f, 0.85f, 1);
    [SerializeField] private Color errorColor = new Color(1, 0.67f, 0.67f, 1);

    // action에 들어있는 항목들을 바탕으로 이 container를 초기화
    public void Initialize(KeyAction action)
    {
        this.action = action;

        // 컨테이너의 이름
        string labelText = GetLabel(action);
        containerLabel.text = labelText;

        // 버튼 이름
        ShowCurrentKeyBinding();
    }

    // Localization Key를 얻어서 표시
    private string GetLabel(KeyAction action)
    {
        var field = typeof(KeyAction).GetField(action.ToString());
        var attr = field?.GetCustomAttribute<KeyActionLabelAttribute>();
        return attr?.LocalizationKey ?? $"keybindings.{field.ToString().ToLower()}";
    }

    private void ShowCurrentKeyBinding()
    {
        IReadOnlyDictionary<KeyAction, Key> currentBindings = GameManagement.Instance.KeyBinding.CurrentBindings;
        buttonText.text = currentBindings[action].ToString();
    }

    private void OnEnable()
    {
        ShowCurrentKeyBinding();
    }

    public void SetContainerLabel(string label) =>  containerLabel.text = label;
    public void SetButtonText(string label) =>  buttonText.text = label;
    public void SetButtonNormalColor() => buttonImage.color = normalColor;
    public void SetButtonSelectedColor() => buttonImage.color = selectedColor;
    public void SetButtonErrorColor() => buttonImage.color = errorColor;
}
```
> 일단 대략적인 로직은 이렇고, 이제 버튼이 어떻게 관리되는지 정도만 생각해보면 될 듯

- 일단 더 작업했고, 아래 2개가 남았음
	- `enum` 타입에서 `None`으로 처리되는 부분에 대한 오류 처리
	- 버튼을 클릭해도 버튼 변경 로직이 제대로 처리되지 않음

- 다 처리 완료
	- 좀 헤맸던 부분은 `Section > Container > Button`으로 구조를 변경하면서, `Button`을 클릭했을 때 `Section`의 메서드가 실행되게 하는 것이었음
	- 처음엔 스택 오버플로우가 떴다. 
	- 구현은 `Button`의 `onClick`에선 `Container.OnClicked`라는 이벤트를 발생시키고, `Section`에서 `Container`를 만들 때 `Button`의 이벤트를 구독하는 방식
		- `Container`에서 중요한 건 아래 2개를 구분하는 것이다.
			- 이벤트를 발생시키는 메서드 (=`Container`에서 `Button`에 구독하는 메서드)
			- `Section`의 메서드가 실행되면서 `Container`에서 실행되기 원하는 메서드
				- 예시) 색깔 변경
			-  처음에는 순환 구조가 생겨서 스택 오버플로우가 떴음.

- 추가) 편집 중에 패널을 닫을 때, 편집 중인 상태를 해제해줘야 함
```cs
// InputSystem.onAnyButtonPress.CallOnce(ctrl => OnAnyKeyPressed(ctrl));
```
> 원래는 위 메서드로 실행시켜줬는데

```cs
	private IDisposable keyWaitHandle;

{
	// InputSystem.onAnyButtonPress.CallOnce(ctrl => OnAnyKeyPressed(ctrl));
	RegisterKeyWait();
}

private void RegisterKeyWait()
{
	keyWaitHandle = InputSystem.onAnyButtonPress.Call(ctrl =>
	{
		keyWaitHandle?.Dispose(); // Call 실행 후, 이벤트 구독을 해제
		keyWaitHandle = null;
		OnAnyKeyPressed(ctrl);
	});
}
```
> - 이렇게 바꿔줬다. `onAnyKeyPress` 자체에 `OnAnyKeyPress`를 할당할 수 없기 때문에, `System.IDisposable`을 이용해서 수동으로 해제를 시켜주는 방식. 

이미 중복된 키가 있어서 재실행되는 상황이나 패널을 닫는 `OnDisable` 상황에도 
```cs
keyWaitHandle?.Dispose(); // Call 실행 후, 이벤트 구독을 해제
keyWaitHandle = null;
```

위 코드로 처리하면 되겠다.
