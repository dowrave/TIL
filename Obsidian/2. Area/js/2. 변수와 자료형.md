## 입출력

### alert()
- 알림 창을 표시한다. 메시지나 변숫값이 표시된다.
```js
alert("안녕하세요?")

alert(10)
// 알림창에 10 표시

a = 10
alert(10)
// 알림창에 10 표시
```

### confirm()
- 단순한 내용 외에도 **확인과 취소 버튼**이 있어서 사용자가 어떻게 클릭했느냐에 따라 다르게 동작시킬 수 있다. 이를 `확인 창`이라고 한다.
```js
confirm("프로그램을 종료하시겠습니까?")
```
> 왼쪽 버튼을 누르면 True, 오른쪽 버튼을 누르면 False를 반환한다.


### prompt()
- 사용자가 간단한 **값을 입력**할 수 있는 창이다.
- 기본값을 지정하지 않으면 빈 텍스트로 표시된다.
```js
prompt("이름을 입력하세요.")
prompt("이름을 입력하세요.", "홍길동") // 입력창에 '홍길동'이 들어가 있음.
```
- 반환된 값을 사용할 수 있다.
- 입력하지 않고 `취소` 버튼을 누른다면, `Null`이 반환된다.

> 참고) 콘솔 창에 뜨는 `undefined`
> 실행한 명령어가 반환하는 값이 없을 때 뜬다. 앞에서 `console.log()`도 출력만 할 뿐 반환하는 함수가 아니었고, 여기서도 `alert()`의 경우 반환하는 값이 없기 떄문에 `undefined`가 뜬다.


### console.log()
- 프로그램 동작을 위해 확인하는 용도로 가장 많이 사용한다.
- 변수와 함께 내용을 연결할 수 있다.
	- 연결 연산자 `+`
	- `템플릿 리터럴`이라는 것도 있다. 나중에 설명한다,,,
```js
name = "김갑송"
console.log(name + "님은 js를 공부중입니다.") // 김갑송님은 js를 공부중입니다.
```

### document.write()
- 웹 브라우저창에 내용을 표시할 때 사용한다. 
- 실제 웹 브라우저창에 내용을 표시할 떄는 `DOM`을 사용하여 원하는 위치를 지정할 필요가 있지만, 아직 안배웠죠?
- 연결 연산자, 템플릿 리터럴 모두사용 가능하다.

## 변수

- 선언과 할당
```js
let a // 선언

a= 10 // 할당

let a = 10 // 선언 & 할당
```


### let
- 변수를 지정할 때 사용

### const
- 상수를 지정할 때 사용
- 값을 변경할 경우, 오류가 발생한다.

### var
- 에크마스크립트 2015 이전에는 `var`을 사용했다.
#### 변수 호이스팅
- `호이스팅Hoisting`을 조심해야 한다 : 호이스팅이란, 변수의 선언 부분을 스코프의 가장 윗부분으로 끌어올리는 것이다. 
```js
// 호이스팅 예시
var x = 10;
var sum = x + y;
var y = 10;
console.log(sum) // NaN
```
> `y`가 선언되지 않은 변수임에도 `sum` 변수는 `NaN`값이 되는데, 자바스크립트 해석기가 변수를 사용하기 전에 따로 기억해두기 때문이다. 
> 실제로 위 코드는 아래처럼 동작한다.
```js
var x = 10;
var y;
var sum = x + y;
var y = 10;
console.log(sum)
```
- 한편, 여기서 변수가 `let`으로 선언되었다면 `let sum = x + y;`에서 오류가 발생한다. 

#### 변수 재선언과 재할당
```js
function add(a, b) {
	return a+b;
}

var sum = add(10, 20);
console.log(sum); // 30
var sum = 100;
console.log(sum); // 100
```
- `var`로 선언된 변수는 재선언과 재할당이 가능하다.
- 그러나 코드가 길어질수록, 실수로 같은 이름의 변수를 할당할 위험이 있다. 이러한 위험을 방지하기 위해, `let`과 `const`는 변수 재선언을 허용하지 않는다. `let`은 변수이므로 재할당이 가능하지만 `const`는 상수이므로 재할당 또한 허용하지 않는다.

| 예약어 | 선언하지 않고 사용할 경우 | 재선언 | 재할당 |
| ------ | ------------------------- | ------ | ------ |
| var    | 오류 X(호이스팅)          | O      | O      |
| let    | 오류 O                    | X      | O      |
| const  | 오류 O                    | X      | X       |

## 자료형
- `원시 유형Primitive`과 `객체Object`로 나뉜다.
- `원시 유형`은 하나의 값만 저장하는 자료형이다. `숫자number, 문자열string, undefined, null, symbol 유형` 등이 있다.
- 원시 유형 외에는 모두 `객체`이다.

### typeof 함수
- 값이나 변수를 넣으면 어떤 자료형인지 알려준다. 파이썬의 `type()`과 동일한 역할.
```js
let data = 5
typeof(data) // number
```

### 숫자형Number
- **자바스크립트에서는 정수와 실수를 함께 묶어** 숫자형이라고 한다.
- 실수를 저장하는 방법이 자바스크립트에서는 다르기 때문에, 복잡하거나 정밀한 계산에서는 자바스크립트를 사용하지 않는 것이 좋다.

```js
type(3.145)
```

### 문자열String
- `'` 이나 `"`로 묶은 데이터를 의미한다. 
- `빈 문자열` : `''`이나 `""` 같이, 따옴표만 있고 내용이 없는 문자열.

#### 특수기호 표시하기
- 문자열에는 **이스케이프 문자나 특수 문자**가 포함된다.
- `이스케이프 문자` : 화면에 표시되지 않지만 줄바꿈, 탭처럼 문서에서 기능을 수행하는 문자.
	- 특수문자로 큰 따옴표를 표시하려면 `"`이 아니라 `\"`으로 백슬래쉬를 앞에 붙여줘야 한다.

| 이스케이프 문자 | 설명             |
| --------------- | ---------------- |
| `\ddd`          | 8진수 문자       |
| `\\xddd`        | 16진수 문자      |
| `\b`            | 백스페이스 문자  |
| `\f`            | 폼 피드 문자     |
| `\n`            | 줄바꿈 문자      |
| `\r`            | 캐리지 리턴 문자 |
| `\t`            | 탭 문자          |

#### 템플릿 리터럴Template Literal
- `템플릿 리터럴Template Literal`은 문자열과 변수, 식을 섞어 하나의 문자열을 만드는 표현 방식이다.
- 이전에는 문자열 부분을 큰 따옴표로 묶고 `+`을 사용해서 연결했는데, 변수나 식이 많아질수록 오타가 날 확률이 높아진다.
- 템플릿 리터럴 사용 예시. **백팃(시프트 뺀 물결표 키)으로 묶는다.** 파이썬의 `f-string`처럼 쓰는데, 변수 앞에 `$`이 온다는 차이만 있다.
```js
let name = '도레미'
let classroom = 201
console.log(name + "님, " +  classroom + "호 강의실로 입장하세요.") // 이전
console.log(`${name}님, ${classroom}호 강의실로 입장하세요.`) // 템플릿 리터럴

// 템플릿 리터럴은 줄바꿈도 그대로 적용된다.
console.log(
`${name}님, 
${classroom}호 강의실로 입장하세요.`)
```

### 논리형, 불리언형Boolean
- 참, 거짓을 표현하기 위한 유형. `true, false`이다.

#### truthy, falsy
- 일반 값에서도 `true로 인정할 수 있는 값`과 `false로 인정할 수 있는 값`이 있다.
- 이를 각각 `truthy`, `falsy`하다고 얘기하며, 일반적으로 `falsy`한 값 이외에는 모두 `truthy`한 값이라고 말한다.
```js
// falsy한 값들
0
""
NaN // Not a Number : 숫자가 아님 (변수가 선언만 된 상태에서 연산에 사용될 경우 NaN)
undefined
null0
```

#### undefined, null
- `undefined` : 값이 할당되지 않았을 때의 변수의 초깃값. **값이면서 자료형이다.**
```js
let userName
console.log(userName)
```

- `null` : 값이 없거나 유효하지 않은 값. 값이면서 자료형이다.
	- 이전에 프롬프트 창에서 `취소` 버튼을 클릭하면 `null`이 반환되었다. 프롬프트 창에 입력한 값을 가져오는 코드에서 그 값이 없기 때문에 `null`이 반환된 것이다.
- `null`을 사용자가 직접 할당할 수도 있다.
```js
let age = null;
```
> 이는 `age` 변수에 값이 할당되지만, 유효하지 않은 값이라는 의미가 된다.

- 자바스크립트 프로그램은 값의 유무룰 판단할 떄 변수의 값이 `null`인지 여부를 체크한다.

- 결론) `undefined`는 사용자가 실수로 값을 지정하지 않았을 때를 의미한다. `null`은 사용자가 일부러 유효하지 않은 값을 지정한 것을 의미한다.

### 객체
- **여러 개의 원시 유형을 하나로 묶어놓은 것. 배열이나 함수도 객체다.**

#### 객체 정의
- 객체는 하나의 변수에 다양한 정보가 포함된 자료형이다.
- `중괄호{}` 안에 모든 정보를 `키와 값`의 형태로 담는다. 여러 개의 `키와 값`을 담을 수도 있다.
```js
let gitBook = {
	title : "깃&깃허브 입문", // 문자열
	pubDate : "2019-12-06", // 문자열
	pages : 272, // 숫자
	finished = true // 논리
}
```
- 한줄로 표시해도 무방하다.

#### 객체 값에 접근하기
```js
gitBook.title
gitBook['title']
```
- 두 방식 모두 가능하며, 일반적으로 `gitBook.title`이라는 위의 방식을 더 많이 사용한다.

#### 배열
- 배열은 `대괄호[]`로 묶고 그 안에 값을 나열한다. 각 값은 `,`로 구분한다.
- 빈 배열 또한 배열이다.
```js
colors = ["red", "blue", "green"]

/*
(3) ['red', 'blue', 'green']

0: "red"
1: "blue"
2: "green"
length: 3
[[Prototype]]: Array(0)
*/
```
> 콘솔 창에서 배열을 확인했을 때, 왼쪽에 있는 숫자롤 `인덱스index`라고 한다.
> 대부분의 배열과 마찬가지로 `index`는 0부터 시작한다.

- `colors[i]`
- `colors.length` : **길이 값은 배열이 갖는 프로퍼티로 접근**해서 얻는다.

### 심벌Symbol
- 에크마스크립트 2015에 새롭게 추가된 **원시 유형의 자료형**으로, 
	- **유일성을 보장**한다.
	- **객체 프로퍼티의 키로 사용**할 수 있다.

- 오픈 소스를 가져와서 쓰거나, 다른 팀원이 만든 객체를 함께 사용할 때 이름의 충돌이 발생할 수 있는데, 이런 경우 키 이름을 심벌로 지정하면 서로 충돌하지 않는다.

#### 심벌 만들기
```js
let var1 = Symbol()
let var2 = Symbol()
var1 == var2 // false
```
- `Symbol()`로 만들며, 당연한 얘기겠지만 두 객체는 다르다.

- 심벌을 키로 사용하기 : `[키]`처럼 대괄호로 묶어서 표현한다.
```js
let id = Symbol()
const member = {
	name : "Kim",
	[id] : 12345
}
member // {name: "Kim", Symbol(): 12345}
member[id] // 12345
```
- **심벌을 키로 사용할 때는 `.`로 접근하지 않고, `[]`로만 접근**한다.

- 위 상황에서 `id`라는 값을 키로 삽입하면..
```js
member.id = 3456
member // {"name": "Kim", "id": 3456, Symbol(): 12345}
```
> - 기존의 **심벌은 유지되면서 새로운 키값이 추가**되는 것을 알 수 있다.


#### 심벌에 설명 추가하기
- 심벌을 사용한 부분이 `Symbol()`으로 표시되었다.
- 여러 개의 심벌을 사용했을 때 이들을 구분하기 위해 설명을 붙일 수도 있다.
```js
Symbol("설명")

// 예제에서
let grade = Symbol("grade")
member[grade] = "VIP"
member // {name: 'Kim', id: 3456, Symbol(): 12345, Symbol(grade): 'VIP'}
```

## 자바스크립트의 장점이자 단점 : 자료형 변환
- 자바스크립트는 프로그램 실행 중에 자료형이 변환되는 언어이다. 
	- 특히 자동으로 형이 변환될 때도 있다.
- 따라서, 자동으로 형이 바뀌는 경우를 미리 알아둘 필요가 있다.


### 자바스크립트 자료형의 특징
- **변수를 선언할 때 자료형을 따로 지정하지 않는다.** 

#### 강력한 자료형 체크와 느슨한 자료형 체크
- C언어, 자바는 변수 선언 시 자료형을 미리 결정한다. 이러한 방식을 `강력한 자료형 체크Strong Data Type Check`라고 한다.
```c
int num = 20
char *name = "John"
```

- 자바스크립트는 변수의 자료형을 미리 지정하지 않는다. 할당된 변수에 따라 자료형이 결정되는데, 이를 `느슨한 자료형 체크Weak Data Type Check`라고 한다. 
	- 편리하지만 체크나 일관성 면에서 단점이 있다.
```js
num = 20
num = "John"
```

#### 자동 형 변환
- **문자열을 사칙연산에 쓰면 자동으로 숫자형으로 변환**된다.
- **숫자와 문자열을 연결하면 숫자가 자동으로 문자열로 변환**된다.

- 예시
```js
let one = "20"
let two = 10

one + two // "2010" : two가 문자로 바뀜
one - two // 10 : one이 숫자로 바뀜
```
- `+`는 더하기 연산자, 연결 연산자 모두 쓴다. 따라서 **`+`의 앞뒤로 문자열이 있다면, 다른 쪽에 있는 숫자는 문자열로 바뀌게 된다.**
- 한편 **`-`이 사용되는 경우는 숫자밖에 없어서, 다른 쪽에 있는 문자열이 숫자로 바뀌어서 계산**된다.

### 직접 숫자형으로 변환하기
- 숫자가 아닌 값을 직접 숫자로 변환할 수도 있다.

#### Number() 함수
- 소괄호 내에 값을 넣어주면 숫자로 변환할 수 있다.
- `"10", null, undefined` 등 모든 자료형을 숫자로 바꿀 수 있는데, 그 규칙은 아래와 같다.

| 기존 유형     | 변환 결과              |
| ------------- | ---------------------- |
| true          | 1                      |
| false         | 0                      |
| 숫자          | 숫자                   |
| null          | 0                      |
| undefined     | NaN                    |
| 정수 문자열   | 정수(맨 앞의 0은 제거) |
| 실수 문자열   | 실수(맨 앞의 0은 제거) |
| 16진수 문자열 | 10진수                 |
| 빈 문자열     | 0                      |
| 기타          | NaN                    |
>- 기타 문자열은 "HI?" 같이 여러 개가 섞인 경우도 포함됨

#### parseInt(), parseFloat()
- 각각 소괄호 내의 값을 정수, 실수로 변환한다.
- 사용자 입력을 받아 10을 곱하는 프로그램을 만들어보자.
```js
let userInput = prompt("아무 숫자나 입력하세요.")
typeof(userInput) // 'string'
```

- 이 `userInput`값에 10을 곱하면 자동으로 숫자로 형변환되면서 10이 곱해지지만, 해당 변수는 여전히 `string`인 상태이다.
	- 즉, 연산에 사용될 때는 변수를 꺼내 형변환한 다음 사용하지만 변수 자체에 형 변환이 가해지는 건 아님
	- 책에는 `parseInput()`을 하면 변수 자체의 형이 바뀌는 것처럼 설명되어 있는데, **정확히는 `userInput = parseInput(userInput)`으로 변수에 재할당해야 해당 변수의 형이 바뀐다.**
	- 아니면 아예 처음에 `let userInput = parseInt(prompt("아무 숫자나 입력하세요."))`로 받아도 될 듯.

### 문자열로 변환하기
- 문자열이 아닌 값을 문자로 바꿀 때는 `ToString()`, `String()` 함수가 있다.

#### toString() 함수
- `null`, `undefined`를 제외한 나머지 자료형을 문자열 데이터로 변환한다.
```js
값.toString()
값.toString(기본값)
```
- 숫자를 문자열로 변환할 때, 기본값을 사용해 10진수인지 2진수인지 지정할 수 있다. 기본적으로 10진수로 변환하며, 2진수나 8진수로 바꾸려면 소괄호 내에 2나 8을 입력한다.

```js
num = 10 
isEmpty = true

num.toString() // '10'
num.toString(2) // '1010'
is.Empty.toString() // 'true'
```

#### String() 함수
```js
String(값)
```
- `null` , `undefined`까지 포함하여 문자열로 변환한다. 이 둘을 `'null', 'undefined'`로 변환하며, 나머지는 `toString()`과 동일함.

### 논리형으로 변환하기 : Boolean() 함수
- 아래 규칙에 따라 변환함

|           | true                  | false     |
| --------- | --------------------- | --------- |
| 숫자형    | 0이 아닌 값           | 0         |
| 문자열    | 빈 문자열 빼고 나머지 | 빈 문자열 |
| undefined |                       | undefined          |
